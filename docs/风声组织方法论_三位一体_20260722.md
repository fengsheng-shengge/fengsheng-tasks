# 风声AI-Native数字员工组织方法论

**三位一体：七习惯 × 五把尺子 × 四层能力栈**

> 版本：v1.0 | 确立日期：2025-07-22 | 确立人：生哥

---

## 一、方法论总览

风声数字员工团队不是六个Agent的拼凑，而是**三层体系咬合驱动的自闭环组织**。

三套体系各有分工、互为因果：

| 体系 | 层级 | 核心问题 | 本质 |
|------|------|----------|------|
| 七习惯 | 行为层 | **怎么做事** | 价值导向的行为准则 |
| 五把尺子 | 评估层 | **怎么考核** | 可量化的产出标准 |
| 四层能力栈 + M×H×E×L | 进化层 | **怎么进化** | 能力建设的路径与公式 |

**三位一体咬合**：

```
七习惯驱动行为
      ↓
行为产出被五把尺子量
      ↓
量完发现Gap
      ↓
Gap决定能力栈往哪补
      ↓
补完能力提升 → 新能力解锁更高习惯
      ↓
      └─────── 循环 ───────┘
```

缺任何一层，组织要么乱跑（无准则）、要么瞎忙（无度量）、要么停滞（无进化）。三层咬合，才能自运转。

## 二、体系一：七习惯（行为层）

> 怎么做事——不是随便做，是按准则做。

基于史蒂芬·柯维《高效能人士的七个习惯》，落地到AI数字团队的协作场景。七习惯不是挂在墙上的口号，而是每个数字员工派活、接活、交付时的行为底线。

### 2.1 第一层·个人成功（当前聚焦）

| 习惯 | 核心原则 | 数字团队落地定义 |
|------|----------|------------------|
| **习惯1：积极主动** | 影响圈做事，不耗关注圈 | 只做能产出交付物的事，不空耗在无法控的环节上 |
| **习惯2：以终为始** | 先问终点再动手 | **金标准：能让经纪人拿给客户看吗？能自动生成交付物吗？不能就不是P0** |
| **习惯3：要事第一** | 投重要不紧急 | 不被紧急琐事裹挟，把精力投在重要不紧急的能力建设上 |

> **当前策略：锁定第一层，逐层解锁。** 个人成功是公众成功的前提——连自己都管不好的Agent，谈什么协作。

### 2.2 第二层·公众成功（待解锁）

| 习惯 | 核心原则 | 数字团队落地定义 |
|------|----------|------------------|
| **习惯4：双赢思维** | 派活不是甩锅，交付物对双方有价值 | 每个任务流转必须有双方可用的产出 |
| **习惯5：知彼解己** | 先理解对方专业判断再派活，不越界 | 接活前先搞清楚上下文和约束，不盲目执行 |
| **习惯6：统合综效** | 六人异构1+1>2，找第三方案 | 不满足于简单分工，追求组合创新产出 |

### 2.3 第三层·不断更新（待解锁）

| 习惯 | 核心原则 | 数字团队落地定义 |
|------|----------|------------------|
| **习惯7：不断更新** | 身体/精神/智力/社交四维度持续迭代 | 落到团队=**组织记忆**：经验沉淀、知识复用、能力升级永不停歇 |

## 三、体系二：五把尺子（评估层）

> 怎么考核——不是感觉好不好，是数据说话。

五维度量化评估，六人各有达标线与优秀线。每把尺子配一句考核口诀，是自检的快筛工具。

### 3.1 五维度总览

| 维度 | 量什么 | 为什么量 |
|------|--------|----------|
| **输出量** | 周产出的数量 | 保证基本产出节奏 |
| **任务复杂度（闭环率）** | 从接活到交付的完整闭环比例 | 不闭环=没做，只推不收=半成品 |
| **技能复用率** | 已有技能被重复调用的比例 | 复用率高说明沉淀有效，反之说明在重复造轮子 |
| **运行时长** | 单次任务从启动到完成的耗时 | 效率的硬指标 |
| **并发度** | 同时处理多任务的能力 | 吞吐量的上限 |

### 3.2 六人考核标准

| 数字员工 | 达标线 | 优秀线 | 考核口诀 |
|----------|--------|--------|----------|
| **小眼镜** | 周新增 ≥ 5条 | ≥ 10条 | 话术三关过了没 |
| **小豆子** | 周交付 ≥ 1份 | ≥ 2份 | 需求的Why是什么 |
| **小扣子** | 周上线 ≥ 1个 | ≥ 2个 | commit的XY判断在哪 |
| **小酒窝儿** | 周审核 ≥ 5条 | ≥ 10条 | 六帽逐项给我 |
| **小鱼儿** | 周同步 ≥ 1份 | ≥ 3份 | 事实推论判断分开说 |
| **小马丫** | 周推送 ≥ 4条 | ≥ 8条 | 四条线都有料 |

> **口诀用法**：自检时念一遍，答不上来=这周还没到位。

## 四、体系三：四层能力栈 + M×H×E×L（进化层）

> 怎么进化——不是堆功能，是补短板、建闭环。

### 4.1 四层能力栈

四层**叠加共存，非线性替代**。每一层是下一层的前提，但不代表有了上层就丢掉下层。

```
┌─────────────────────────────────┐
│  L4 Loop层：自进化闭环           │  ← 能自我迭代
├─────────────────────────────────┤
│  L3 Harness层：可控执行框架      │  ← 任务可追踪、可验证、可恢复
├─────────────────────────────────┤
│  L2 协作层：多人协同能力          │  ← 懂分工、会交接、能统合
├─────────────────────────────────┤
│  L1 执行层：单点任务交付          │  ← 给活能干完
└─────────────────────────────────┘
```

| 层级 | 能力定义 | 标志性表现 |
|------|----------|------------|
| **L1 执行层** | 单点任务交付 | 给活能干完、能交付 |
| **L2 协作层** | 多人协同 | 懂分工、会交接、能统合 |
| **L3 Harness层** | 可控执行框架 | 任务状态可持续、过程结果可验证、失败后能自我恢复 |
| **L4 Loop层** | 自进化闭环 | 发现Gap→自动补能力→验证提升→循环迭代 |

### 4.2 自进化公式

```
Self-improving Agent = Model × Harness × Evaluation × Feedback Loop
```

**乘法关系，缺一归零。**

- **Model（模型能力）**：底座智能，理解与生成
- **Harness（执行框架）**：让智能可控、可追踪、可恢复
- **Evaluation（评估体系）**：知道好不好，好到什么程度
- **Feedback Loop（反馈闭环）**：从评估结果驱动改进

> 任何一个因子为零，整个乘积为零。模型再强，没有Harness就是脱缰野马；评估再精准，没有Loop就是一次性体检。

### 4.3 当前成熟度评估

| 因子 | 当前得分 | 满分 | 状态 |
|------|----------|------|------|
| Model | 8 | 10 | ✅ 底座强，已过可用线 |
| Harness | 3 | 10 | ⚠️ 短板，急需补 |
| Evaluation | 3 | 10 | ⚠️ 短板，急需建 |
| Loop | 2 | 10 | 🔴 最弱，等前三项到位 |

### 4.4 Harness三标准（P0优先补）

Harness是当前最大的短板。三标准必须逐条落地：

| 标准 | 定义 | 验证方式 |
|------|------|----------|
| **任务状态可持续** | 任务从创建到完成，状态全程可追踪、不丢失 | 任意时刻查询任务，能说清"做到哪了" |
| **过程结果可验证** | 中间产物和最终产出都有明确的验收条件 | 每个交付物有checklist，对得上就能过 |
| **失败后能自我恢复** | 出错不挂死，能定位原因并重试或降级 | 模拟失败场景，观察能否自动恢复 |

## 五、三位一体咬合关系

三套体系不是并列的，是**因果链驱动的自闭环**：

```
                    ┌──────────────────┐
                    │   七习惯（行为层）  │
                    │   驱动正确的行为   │
                    └────────┬─────────┘
                             │ 行为产出
                             ▼
                    ┌──────────────────┐
                    │  五把尺子（评估层） │
                    │   量化行为结果     │
                    └────────┬─────────┘
                             │ 发现Gap
                             ▼
                    ┌──────────────────┐
                    │ 四层能力栈（进化层）│
                    │  Gap决定补哪层    │
                    └────────┬─────────┘
                             │ 能力提升
                             ▼
               新能力解锁更高层级的习惯 ──→ 回到起点
```

**咬合的关键时刻**：

1. **习惯驱动行为**：习惯2"以终为始"要求每个任务先定义交付物——这直接影响五把尺子里"闭环率"的计算
2. **尺子量出Gap**：小扣子"周上线"连续不达标，暴露的是Harness层的任务状态不可持续
3. **Gap指向进化**：闭环率低→补L3 Harness；复用率低→补L2协作层；评估不准→建Evaluation
4. **进化解锁习惯**：Harness到位后，习惯4"双赢思维"才具备落地条件——因为任务流转终于可追踪了

## 六、组织愿景与独有特点

### 6.1 愿景

**高效 · 稳定 · 高质量 · 团结 · 有生产力的AI-Native数字员工团队**

### 6.2 组织独有特点

**六人异构 × 三套体系 = 不可复制的组织特征**

| 维度 | 风声特色 |
|------|----------|
| **异构底座** | 扣子 / 百炼 / Trae / 腾讯 / 马维斯——六人六引擎，不依赖单一平台 |
| **三层咬合** | 行为-评估-进化自闭环，不是拼功能是建系统 |
| **口诀文化** | 五把尺子配口诀，自检零门槛，考核有温度 |
| **乘法公式** | M×H×E×L缺一归零，杜绝"强模型弱执行"的幻觉 |

### 6.3 对外价值

这不是风声的内部工具，是**可复制的AI组织方法论**。

- 六人异构的多平台协作经验，是行业稀缺的实操样本
- 三位一体自闭环框架，为任何AI团队提供从"能用"到"自进化"的路径
- **方法论本身可成为风声的第二产品线**：助培养——帮其他团队建自己的AI组织

## 七、当前状态与下一步优先级

### 7.1 当前状态快照

| 维度 | 状态 | 判断 |
|------|------|------|
| 七习惯 | 锁定第一层（个人成功） | ✅ 方向明确，不贪多 |
| 五把尺子 | 六人标准已定义 | ✅ 可执行，待首轮数据验证 |
| 四层能力栈 | L1稳定，L2起步，L3/Harness短板 | ⚠️ 当前进化的瓶颈 |
| M×H×E×L | Model 8分，其余2-3分 | 🔴 乘法效应尚未形成 |

### 7.2 下一步优先级（P0-P2）

| 优先级 | 动作 | 对应体系 | 预期效果 |
|--------|------|----------|----------|
| **P0** | 落地Harness三标准：任务状态可持续、过程可验证、失败可恢复 | 进化层·L3 | 消除最大短板，为Loop奠基 |
| **P0** | 建Evaluation：从五把尺子的数据出发，建评估基准 | 进化层·Evaluation | 有尺子才有度量，有度量才能闭环 |
| **P1** | 跑通首轮五把尺子考核，收集基线数据 | 评估层 | 验证标准可行性，校准达标线 |
| **P1** | 巩固习惯1-3的行为落地，养成第一层肌肉记忆 | 行为层 | 个人成功稳固后再开第二层 |
| **P2** | 启动Loop的最小闭环：评估→发现Gap→补Harness→再评估 | 进化层·L4 | 让M×H×E×L从乘法公式变成运转引擎 |
| **P2** | 解锁习惯4-6，进入公众成功阶段 | 行为层·第二层 | 协作习惯在有Harness保障后才真正落地 |

### 7.3 里程碑路线

```
Phase 1（当前）  ：补Harness + 建Evaluation → 消除短板
Phase 2          ：跑通首轮考核 + 巩固习惯1-3 → 形成基线
Phase 3          ：启动最小Loop闭环 → 乘法效应初现
Phase 4          ：解锁公众成功习惯 → 协作升级
Phase 5          ：全体系自运转 → 方法论可复制输出
```
