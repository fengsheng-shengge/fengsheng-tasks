# 风声·箭头消除 + 知识植入 融合设计方案 v1.0

## 一、核心思路

**游戏是壳，内容是彩蛋。** 不做"教育游戏"，做"游戏里藏着知识"。

| 层 | 占比 | 目标 |
|---|---|---|
| 游戏层 | 90% | 吸引人玩、上头、留存 |
| 知识层 | 10% | 关卡间弹出，3秒可读，5秒可互动 |

## 二、游戏本体（箭头消除）

**核心规则：**
- 屏幕上铺满不同朝向的箭头 ⬆⬇⬅➡
- 点击箭头→沿自身方向飞出屏幕=消除
- 路径被其他箭头挡住=不能消除，需先消挡路的
- 清空所有箭头=通关

**关卡节奏：**
- 前10关：3×3网格，纯教学
- 11-30关：4×4，引入障碍
- 31-50关：5×5，双向箭头
- 50关后：6×6+，旋转箭头、传送门等

**单局时长：** 30秒-2分钟

## 三、知识植入机制

### 3.1 触发节奏

```
Level 1  → 纯游戏（教学关）
Level 2  → 纯游戏
Level 3  → 通关后弹出「知识闪卡」（3秒可读，点击继续）
Level 4  → 纯游戏
Level 5  → 通关后弹出「一问一答」（答对给游戏道具奖励）
Level 6  → 纯游戏
Level 7  → 纯游戏
Level 8  → 通关后弹出「知识闪卡」
...每3-5关穿插一次，不破坏节奏
```

**关键原则：**
- 不每关都弹，3-5关穿插一次
- 玩家可跳过，无惩罚
- 互动型题目答对给游戏奖励（提示道具/额外积分），答错不扣分
- 弹出时间控制在3-5秒可完成

### 3.2 三种内容形态

#### 形态A：知识闪卡（3秒读完）

纯展示，一张卡片一句话结论+一句话佐证。

```
┌─────────────────────────┐
│  💡 今日专业小贴士        │
│                          │
│  定金≠订金               │
│  一字之差，双倍返还变原额  │
│  退还，价值可能差10万      │
│                          │
│  《民法典》第587条         │
│                          │
│      [知道了] →          │
└─────────────────────────┘
```

#### 形态B：一问一答（5秒互动）

两选一，极简，答对给游戏道具。

答对后：

从真实案例提炼一句话takeaway。

## 六、小程序技术要点

- 游戏本体用Canvas渲染，轻量（<1MB）
- 知识卡片用小程序原生组件覆盖在Canvas上层
- 内容数据从云端JSON拉取，可热更新
- 用户答题数据本地缓存，后续可做"已掌握知识点"标记
- 道具系统与知识问答联动：答对→获得道具→降低游戏难度→正循环

## 七、与「法官来了」的关系

- 「法官来了」= 独立小游戏，纯问答，深度练专业
- 「箭头消除+知识植入」= 休闲游戏+轻度知识触达
- 两者可以共存：消除游戏吸引人打开，法官来了让人深度学习
- 消除游戏中答对的题目，解锁法官来了中的对应题目

## 八、不做的事

- ❌ 不做强制学习：不能"必须答对才能过关"
- ❌ 不做长篇内容：知识卡片超过2行就是失败
- ❌ 不做恐吓式内容：不说"不学就亏"，只说"知道这个帮到你"
- ❌ 不做内容前置：不在游戏开始前弹知识，只在通关后
