# 📱 风声微信小程序 · 产品设计规划

> v1.0 | 2026.6.24 | 小鱼儿主笔 | 待生哥确认

---

## 一、产品定位

**一句话**：把风声核心能力装进微信，让服务者在刷朋友圈的间隙就能提升自己。

| 维度 | 说明 |
|------|------|
| 用户 | 房产经纪人/装修顾问/物业管家等居住服务者 |
| 场景 | 碎片化学习+即时工具+社交分享 |
| 价值 | 降低使用门槛（不用打开电脑），缩短引流路径（抖音→微信→小程序3步到位） |
| 与主站关系 | 小程序是主站的**轻量入口**，不是替代品。深度功能导流主站，轻量功能小程序闭环 |

## 二、核心功能模块（MVP）

### P0 — 第一批上线（验证期）

| 模块 | 功能 | 对应主站产品 | 差异 |

| 🎯 客户解码器 | 4域26场景快速解码 | /reply/ | 小程序版精简为：选域→选场景→出话术，3步完成 |
| ⚡ 六维品质测评 | 6维度快速自检 | /quality-test/ | 小程序版：6题快速版（主站10题完整版），结果可分享 |
| 📖 业务字典搜索 | 词条搜索+详情 | /knowledge/ | 小程序版：搜索+热门词条+收藏，不做全量浏览 |
| 👤 个人中心 | 微信登录/测评历史/收藏 | 无 | 小程序专属 |

## 三、页面架构

```
小程序首页（Tab 1）
├── 今日推荐（轮播：热门场景/新词条/测评入口）
├── 快捷工具
│   ├── 🎯 客户解码器
│   ├── ⚡ 品质测评
│   └── 📖 业务字典
├── 热门场景卡片（场景→话术，点击即用）
└── 精选内容（短视频/长图文卡片）

├── 域选择（4域卡片）
├── 场景列表（筛选+搜索）
├── 解码结果（话术+应对策略）
├── 👍👎反馈 + 📤分享
└── ✨推荐：相关测评/关联词条

品质测评页面
├── 快速版（6题，2分钟）
├── 测评结果（雷达图+分数+建议）
├── 📤分享到朋友圈/群
└── 查看完整版→导流主站

业务字典页面
├── 搜索框（即时搜索）
├── 热门词条
├── 域分类浏览
├── 词条详情
└── 收藏+分享

个人中心（Tab 2）
├── 微信头像+昵称
├── 我的测评历史
├── 我的收藏
├── 学习统计（连续天数/完成数）
└── 关于风声
```

## 四、技术架构

```
┌─────────────────────────────────────┐
│           微信小程序前端              │
│  (WXML/WXSS/JS, 复用主站逻辑)        │
└──────────────┬──────────────────────┘
               │ HTTPS
┌──────────────▼──────────────────────┐
│         API网关层 (Cloudflare)       │
│  /api/v1/* → 统一认证+限流+路由      │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│         共享后端服务层                │
│  ┌───────────┐  ┌───────────────┐   │
│  │ 解码器API  │  │ 测评引擎API   │   │
│  │ (主站共用) │  │ (主站共用)     │   │
│  └───────────┘  └───────────────┘   │
│  ┌───────────┐  ┌───────────────┐   │
│  │ 知识底座API│  │ 用户数据API   │   │
│  │ (主站共用) │  │ (新增)        │   │
│  └───────────┘  └───────────────┘   │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│    D1数据库 (主站共用)               │
│  + 小程序用户表 (openid/unionid)     │
│  + 收藏表 / 测评记录表              │
└─────────────────────────────────────┘
```

**核心原则：API先行，三端共用**
- 解码器/测评/知识底座的后端逻辑抽成API
- Web端/小程序端/机构版共用同一套API
- 只有前端展示层不同

## 五、引流闭环

```
抖音视频 → 评论区置顶 → 公众号文章 → 文中小程序卡片 → 小程序体验
                                                        │
                                            ┌───────────┘
                                            ▼
                                    测评结果分享到朋友圈
                                            │
                                            ▼
                                    好友点击→打开小程序
                                            │
                                            ▼
                                    新用户注册→体验产品
                                            │
                                    ┌───────┴───────┐
                                    ▼               ▼
                              留在小程序        导流主站深度体验
                              (轻量闭环)       (完整功能+Agent培养师)
```

**关键路径**：抖音→小程序 3步可达（视频→公众号→小程序），比抖音→主站少2步

## 六、与主站/机构版的关系

| 维度 | 主站 | 小程序 | 机构版 |
|------|------|--------|--------|
| 用户 | C端服务者 | C端服务者（移动优先） | B端机构 |
| 登录 | 浏览器指纹 | 微信授权 | 企业账号 |
| 功能深度 | 完整版 | 精简版 | 完整版+管理后台 |
| 数据 | D1共享 | D1共享 | 私有部署/数据隔离 |
| 定价 | 免费+进阶+高阶 | 同主站 | 年授权/按坐席 |
| 引流 | 被动(SEO/搜索) | 主动(分享/公众号) | 直销 |

**数据打通**：小程序用户通过unionid与主站用户关联，统一看板统计。
