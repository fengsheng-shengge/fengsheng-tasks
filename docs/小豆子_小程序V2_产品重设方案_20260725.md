## 〇、重设核心：从「有什么」到「经纪人面对消费者，拿来就能用」

生哥判定 V2.0 半成品的根因——产品从「有什么数据」出发（首页 4 张卡 = 4 个已有数据模块），不是从「经纪人面对客户需要什么」出发。

本次重设的出发点更进一层：**站在经纪人视角，想清楚「这个工具到底能让我在客户面前拿出什么、直接服务消费者」**。不是给他一堆知识，而是给他「拿起来就能用」的方法和交付物。

重设四原则：

1. **场景驱动，不数据驱动** —— 入口是「我在服务谁、到哪一步」，不是「我有什么功能」
2. **道法术器四层交付，不跳层** —— 每个工具按 道(心里懂的)→法(当场走的)→术(嘴上说的)→器(递给客户的) 给，不把话术当方法、不把模板当答案
3. **器是可带走的交付物** —— 每个工具的终点，是一份能当场发给客户 / 给客户看的东西（测评卡、对比表、清单），这是「拿来直接服务消费者」的落点
4. **MVP 聚焦买房客户 × 3 场景** —— 先一个角色跑通闭环，再扩（量不是壁垒，触发机制才是）

## 一、价值主张（经纪人视角）

> **小豆子 = 你坐在客户对面时，掏出手机就能用的「专业外挂」。**
> 不是让你去学，是让你当场就用——懂客户、会说话、有依据、能交付。

对经纪人而言，每个工具回答四个问题：

| 经纪人心里在想 | 小豆子给 |
|---|---|
| 他到底什么意思？ | **道**：人性规律，先懂，才不慌 |
| 我下一步怎么做？ | **法**：当场动作步骤，照着走 |
| 我嘴上怎么开口？ | **术**：原话话术，直接说 |
| 我能给客户看什么？ | **器**：可带走的交付物，当场发 |

## 三、客户类型 × 阶段矩阵（信息架构底座）

| | 买房客户 | 租房客户 | 出售业主 | 出租业主 |
|---|---|---|---|---|
| **售前** | 需求解码→画像 | 需求解码→画像 | 房源估值→委托 | 房源估值→委托 |

**P2**：出租业主 + 售后深化

## 四、信息架构重设

### 4.1 首页 = 场景入口（重设核心）
两级选择，不堆功能卡：
- **第一级 选「服务谁」**：4 客户类型 chips，买房客户激活，其余标「即将开放」
- **第二级 选「到哪步」**：售前 / 售中 / 售后，按所选客户类型显示可用场景
- 点场景 → 进工具页

### 4.2 工具页 = 道法术器（经纪人当场可用）
进入某场景后，整页是该场景的 道 / 法 / 术 / 器 四层内容。顶部带「客户在场」场景卡，提醒此刻面对什么；每层可点「查依据」浮出相关词条；**器**层是成品交付物，可直接发客户。

### 4.3 词典 = 工具内「知识速查」浮层
词条不再独立作为首屏主推，而是工具页内「知识速查」浮层——工具讲到某点，点「查依据」浮出相关词条（自动调用，按 场景×阶段×道法术器 标签过滤）。**词典 Tab 仍保留全量浏览与收藏。**

### 4.4 导师 P0 不上，入口预留
导师 Tab 显示「专业导师即将开放」，不占 MVP 开发量。

### 4.5 Tab 结构（MVP 保留壳，重排优先级）
`首页(场景入口) | 词典(全量+收藏) | 导师(预留) | 测评(游戏化保留) | 我的(积分/贡献)`
说明：词典 Tab 与工具内浮层共存——Tab 用于主动浏览全量，浮层用于场景内按需调取。

## 五、道法术器四层框架（经纪人视角重解）

| 层 | 经纪人心里在想 | 给什么（拿来即用） | 不跳层红线 |
|---|---|---|---|
| **道** | 他到底什么意思 | 一句话规律 / 人性洞察，让你先懂、不慌 | 不直接给话术 |
| **法** | 我下一步怎么做 | 当场动作步骤 / checklist，照着走 | 不跳过方法直接给模板 |
| **术** | 我嘴上怎么开口 | 按场景的原话话术，直接说 | 不堆大杂烩话术 |
| **器** | 我能给客户看什么 | 可带走 / 可分享的交付物（测评卡·对比表·清单） | 不绑定小程序，能单独发客户 |

工具页四层 section 顺序固定：**道 → 法 → 术 → 器**，逐层展开；**器**层做成成品卡，带「发客户 / 复制」动作，是「拿来直接服务消费者」的落点。

## 六、3 个 P0 场景工具设计（每个先说「拿到手能用什么」）

> 高保真原型见 `小豆子V2重设_交互原型_20260725.html`。

### 6.1 客户解码（买房客户 · 售前）

> **🎯 经纪人拿到手，在客户面前能用什么**
> ① 3 分钟生成「客户画像卡」当场给客户确认需求；② 客户说「再看看」时，掏出「六维测评卡」用数据回应，不空口说服；③ 把决策变成可带走的「家庭版对比卡」发客户。

**道**（先懂，不慌）
- 客户说「再看看」通常不是真看，是三大顾虑：怕买贵 / 怕踩坑 / 怕后悔
- 决策是情绪曲线，不是理性打分；先处理情绪再处理事情

**法**（当场怎么走）
1. 听原话 —— 记录客户原话，不替他概括
2. 拆顾虑 —— 归类 价格 / 风险 / 决策
3. 对画像 —— 预算·家庭·时间窗 校准需求
4. 给标准 —— 用可验证标准（测评/数据）替代主观说服

**术**（嘴上怎么说）
- 说「再看看」→「理解，这房我帮您做了六维测评，关键几项都不错，您先看数据再决定」
- 说「有点贵」→ 先确认超预算还是觉得不值，再拆持有成本
- 说「要和家人商量」→ 把决策变成可带走的「家庭版对比卡」

**器**（直接发给客户的交付物）
- 需求挖掘七问模板（可分享客户填 / 经纪人自用）
- 客户画像卡（预算 / 家庭 / 时间窗 / 决策人）—— 当场生成、客户确认
- 六维测评卡（把「再看看」变成可看的数据，发客户）

**知识速查调用**：需求挖掘话术(术) · 客户画像维度(法) · 预算判断方法(法) · 决策情绪规律(道) · 六维测评标准(器)

---

### 6.2 带看服务（买房客户 · 售中）

> **🎯 经纪人拿到手，在客户面前能用什么**
> ① 带看前自动生成「带看清单 + 动线脚本」，照着走不漏点；② 带看中把客户原话顾虑对应到房间具体点，当场说；③ 带看后发客户一份「房源对比表」，让他带得走、记得住。

**道**（先懂，不慌）
- 带看不是展示房子，是管理客户决策体验；首看定调，复看定心
- 客户记不住参数，只记得「感觉被懂」

**法**（当场怎么走）
1. 带看前 —— 房源匹配 + 动线设计 + 痛点预设
2. 带看中 —— 动线引导 / 价值点植入 / 竞品对比
3. 带看后 —— 24h 内跟进 + 决策窗口管理

**术**（嘴上怎么说）
- 动线：「我们先看采光和动线，这是这套房最值的地方」
- 价值点：把客户原话里的顾虑，对应到房间具体点
- 竞品对比：客观维度对比，不贬低竞品

**器**（直接发给客户的交付物）
- 带看 Checklist（前/中/后）—— 经纪人照着走
- 房源对比表（客户可带走，记住「被懂」的感觉）
- 动线脚本（首看 / 复看 各一版）

**知识速查调用**：动线设计方法(法) · 价值点植入话术(术) · 竞品对比话术(术) · 带看 Checklist(器) · 房源匹配规律(道)

---

### 6.3 谈判斡旋（买房客户 · 售中）

> **🎯 经纪人拿到手，在客户面前能用什么**
> ① 客户压价时，掏「90 天成交锚定卡」用数据说话；② 把分歧拆成「可交换条件清单」发给双方，把拉锯变商量；③ 帮业主算「空置成本 vs 让价」，用账说话。

**道**（先懂，不慌）
- 谈判不是压价，是让双方都觉得赢；价格背后是心理账户
- 僵局多因信息不对等，不是脾气

**法**（当场怎么走）
1. 锚定 —— 先给合理参照系（成交数据 / 评估）
2. 条件交换 —— 价格换周期 / 家具 / 维修
3. 情绪管理 —— 暂停 / 换话题 / 给台阶
4. 僵局破局 —— 引入第三方标准（政策 / 数据 / 案例）

**术**（嘴上怎么说）
- 客户压价 →「我理解，我们拿 90 天成交数据说话，这个价在合理区间，但家具/周期可以谈」
- 业主不让步 → 帮业主看「空置成本 vs 让价」，用数据算账
- 双方拉锯 → 把分歧拆成可交换的条件清单

**器**（直接发给客户的交付物）
- 谈判筹码表（可交换条件库）
- 条件交换清单（客户/业主各一版，发双方）
- 价格锚定卡（90 天成交参照，当场给客户看）

**知识速查调用**：锚定方法(法) · 压价应对话术(术) · 条件交换清单(器) · 心理账户规律(道) · 90天成交数据(器)

## 七、词条调用映射（供 #186 小眼镜打标签）

### 7.1 三维标签体系
每个词条打三个标签，才能被工具精准调用：

| 维度 | 取值 |
|---|---|
| 客户类型 | 买房客户 / 租房客户 / 出售业主 / 出租业主 |
| 业务阶段 | 售前 / 售中 / 售后 |
| 内容层次 | 道 / 法 / 术 / 器 |

### 7.2 工具 → 词条映射（P0 三个场景）

| 工具 | 调用词条（客户类型=买房 · 阶段=售前/售中 · 层次） |
|---|---|
| 客户解码器 | 需求挖掘话术(术) · 客户画像维度(法) · 预算判断方法(法) · 决策情绪规律(道) · 六维测评标准(器) |
| 带看服务 | 动线设计方法(法) · 价值点植入话术(术) · 竞品对比话术(术) · 带看 Checklist(器) · 房源匹配规律(道) |
| 谈判斡旋 | 锚定方法(法) · 压价应对话术(术) · 条件交换清单(器) · 心理账户规律(道) · 90天成交数据(器) |

> 说明：上表为**框架示例**，小眼镜据此从实际 5800 词条（trade/policy/dimension 等域）给每条打三维标签；工具调用时按 `客户类型×阶段×层次` 过滤，只给当前场景相关词条，不 500 条全堆。
> 原则：词条不直接展示给用户，由工具自动调用；思考框架引用词条核心结论（一句话，点开看详情）。

### 7.3 调用接口（与终端记忆层对齐）
工具内「知识速查」浮层 = 小程序端最小化的 `/memory/trigger`：
```
请求：{ sceneId, stage, layer }  →  响应：{ oneLineAnswer, entries[] }
```
未来眼镜端直接复用同一触发逻辑，小程序先跑通闭环。
