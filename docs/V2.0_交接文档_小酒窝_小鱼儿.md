#### P0-02：行为埋点系统 ✅
- **文件**：`src/utils/tracker.js`（重写）
- **7 个埋点事件**：

| 事件名 | 定义 | 触发位置 |
|--------|------|---------|
| `content_click` | 点击率：看到内容→点击进入详情 | 首页场景卡、词典列表 |
| `content_expand` | 展开率：解锁案例/备忘 | 词条详情解锁操作 |
| `content_collect` | 收藏率：点击收藏 | 词条详情收藏 |
| `content_followup` | 追问率：贡献纠错/验证/提问 | 词条详情贡献区 |
| `trigger_input_use` | 触发入口使用率 | 首页输入条点击/焦点 |
| `trigger_match_success` | 触发匹配成功率 | 首页输入→匹配结果 |
| `trigger_to_detail` | 触发转化率 | 首页匹配后跳转详情 |
| `pageview` | 页面访问 | 所有页面 onShow |
| `mentor_message_sent` | 导师对话发送 | 导师页快捷/自由输入 |
| `paywall_shown` | 付费墙弹出 | 导师页配额耗尽 |
| `points_change` | 积分变动 | 首页答题获奖 |

- **存储**：MVP 阶段存本地 `uni.getStorageSync('fs_events')`，预留远程上报接口（`fengsheng.tech/api/event`，需后端配合）
- **接入页面**：首页、词典、词条详情、导师、domain-list

## 四、给小鱼儿的部署清单

### 4.1 部署步骤

```bash
cd fs-mini-program

# 拉取最新代码
git pull origin main

# 安装依赖（如有更新）
npm install

# 构建微信小程序
npm run build:mp-weixin

# 构建 H5（可选）
npm run build:h5
```

### 4.2 微信小程序上传

1. 打开**微信开发者工具**
2. 导入项目：`/workspace/fengsheng-tasks/fs-mini-program/dist/build/mp-weixin`
3. AppID：`wxd0f79d52a84ea4a8`（风声助手）
4. 检查无报错后，点击**上传**
5. 填写版本号：`2.0.0`，版本备注：`V2.0 P0+P1完整交付（触发入口+行为埋点+文案升级+时效标注）`

### 4.3 部署后检查项

- [ ] 微信小程序后台审核通过
- [ ] 首页"客户说了什么"入口可正常使用
- [ ] 行为埋点本地数据正常写入
- [ ] 登录页功能列表显示正确
- [ ] 词条详情时效标注（法律类词条）正常显示

### 4.4 注意事项

- **H5 端限制**：因 uni-app H5 端 `require()` 限制，大 JSON 文件（entries.json 500KB、search_aliases.json）无法在 H5 dev 模式动态加载，mp-weixin 不受影响，**最终用户使用不受影响**
- **埋点数据**：当前为本地存储，建议后续接入后端 `/api/event` 接口进行远程上报
- **法条版本**：合同法等已废止法条在 `legalVersion` 中已标注，需后续 entries.json 加 `lastVerified`/`legalStatus`/`legalVersion` 字段替代推断逻辑
