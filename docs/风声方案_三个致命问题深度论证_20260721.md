# 风声方案 · 三个致命问题深度论证

> 小眼镜 · 2026-07-21
> 每个问题用五步法走到底：事实→推断→反证→边界→结论

---

## 问题一：经纪人到底会不会用？

### 1. 事实层

**经纪人现在怎么获取专业信息？**

| 方式 | 使用频率 | 痛点 |
|------|---------|------|
| 问同事/店长 | 最高 | 同事也不一定对，法条更新了可能不知道 |
| 百度/微信搜 | 高 | 搜到的是营销号，法条可能过期 |
| 公司内训 | 低频 | 一年几次，内容泛，跟眼前客户的问题不搭 |
| 法律咨询 | 极低 | 成本高，一般不到打官司不找律师 |
| 什么都不查 | 最高 | 靠"感觉"说，说错了也不知道 |

关键事实：**经纪人目前最大的信息来源是"同事"和"感觉"，不是专业系统。**

### 2. 推断层

如果经纪人现在靠"问同事"解决专业问题，说明两个事：

**A. 他有专业信息需求**——否则不会问同事
**B. 现有供给不够好**——否则不会靠"感觉"

如果风声能比"问同事"更快、比"百度"更准、比"内训"更即时，经纪人有什么理由不用？

**推断：经纪人会用，前提是比他现在的方案好一个量级。**

### 3. 反证层

**什么情况下经纪人不会用？**

**情况A："被AI帮"伤害专业自尊**
- 经纪人心理："我干这行10年了，还需要AI教我？"
- 反证：经纪人已经在用贝壳VR看房系统推的话术，没觉得伤自尊
- 核心区别：贝壳推的是"帮你说"（销售工具），不是"教你"（知识灌输）
- 结论：定位必须不是"教你"，是"帮你说更快"——这不伤自尊

**情况B：推送时机不对，打扰客户对话**
- 经纪人心理："我跟客户聊着呢，你突然冒出来很尴尬"
- 反证：这是真实风险。如果客户说了"押金"，眼镜当场弹出"房东无权扣押金"，客户会觉得经纪人不专业
- 解决：推送只给经纪人看/听，不给客户看。AR卡片只有经纪人视角，语音只走耳机
- 结论：推送必须是私密的，这是产品设计的硬约束

**情况C：推送内容不准，反而误导**
- 经纪人心理："你推的东西跟客户问的不是一回事，我还得纠正"
- 反证：这是真实风险。关键词匹配天然有歧义——"定金"可能是买房的定金，也可能是租房的押金
- 解决：推送前加一步场景确认——先判断是rental还是trade，再推对应内容
- 结论：准确率不是100%才能用，但必须≥90%，否则经纪人会关掉

**情况D：经纪人不信任内容**
- 经纪人心理："这个法条是最新的吗？你别给我过期的"
- 反证：这是真实风险。法条更新频繁，241条词条没人持续维护就会过期
- 解决：每条内容标注更新日期+法条版本号，过期自动标红
- 结论：内容时效性是信任的基础，没有时效标注就没有信任

### 4. 边界层

**经纪人会用，但有边界：**

| 条件 | 不用→用的临界点 |
|------|---------------|
| 速度 | 比问同事快10倍（2秒 vs 拿出手机搜5分钟） |
| 准确率 | ≥90%场景匹配正确 |
| 隐私 | 只推给经纪人，不推给客户 |
| 时效 | 内容标注更新日期，过期标红 |
| 成本 | 基础内容免费，深层内容积分解锁 |

**越过这些临界点，经纪人会用。没越过，不会用。**

### 5. 结论

**经纪人会用，但必须同时满足4个条件：**
1. 比现有方案快一个量级（2秒 vs 5分钟）
2. 推送私密，不暴露给客户
3. 准确率≥90%
4. 内容有时效标注，可信任

**最关键的验证：找3-5个经纪人做1对1测试——不是问"你会不会用"，是让他面对模拟客户，观察他在有/没有专业支持的两种情况下的行为差异。行为差异比口头反馈真实100倍。**

## 问题二：内容可复制性怎么办？

### 1. 事实层

**我们的内容由什么构成？每一项是否可复制？**

| 内容类型 | 来源 | 是否公开 | 复制成本 |
|---------|------|---------|---------|
| 法律条文 | 民法典/住建部法规 | 完全公开 | 0 |
| 法院案例 | 裁判文书网 | 完全公开 | 爬取成本极低 |
| 行业标准 | 国标/行标/地方标准 | 完全公开 | 0 |
| 数据指标 | 统计局/行业报告 | 部分公开 | 低 |
| 专业解读 | 小眼镜编写 | **不公开** | 中——需要行业经验 |
| 场景话术 | 小眼镜编写 | **不公开** | 中——需要真实场景经验 |
| 行动指令(brokerNote) | 小眼镜编写 | **不公开** | 高——需要知道经纪人该做什么 |
| 对话流路径 | 小眼镜设计 | **不公开** | 高——需要理解经纪人工作流 |
| 触发词映射 | 小眼镜设计 | **不公开** | 中——需要理解客户表达方式 |

关键发现：**公开内容（法条/案例）占数据量70%，但不构成壁垒；不公开内容（解读/话术/行动指令/对话流）占数据量30%，但构成核心价值。**

### 2. 推断层

竞对如果复制我们，他能复制什么？

**他能复制的：**
- 法条原文→直接爬取
- 案例原文→直接爬取
- 词条框架→看我们的JSON结构就能模仿

**他复制不了的：**
- "房东无权扣押金"这个结论——法条写的是第714条，但把第714条翻译成经纪人能对着客户说的一句话，这需要行业经验
- brokerNote里的"这样做"——法条只告诉你权利义务，不告诉经纪人下一步该做什么
- 对话流路径——客户从"我想退租"到"房东说扣3万"到"怎么算合理"，这条路径是经验，不是数据
- 触发词映射——"押金"这个词在rental场景和trade场景意思是不同的，映射规则是经验

**推断：风声的壁垒不在原始数据，在"翻译层"——把法律语言翻译成经纪人语言，把权利义务翻译成行动指令。**

### 3. 反证层

**有没有可能大公司用AI自动生成"翻译层"？**

完全可能。流程是：
1. 爬取民法典+住建法规
2. 用GPT/Claude对每条法条生成"经纪人版解读"
3. 用AI生成brokerNote行动指令
4. 用AI生成对话流路径

**这个流程现在的成本是多少？**
- GPT-4o API：$5/百万token
- 800条词条×平均500字 = 40万字 ≈ 20万token
- 生成成本：约$1
- 加上审核校对人力：约2-3天

**结论：AI可以低成本生成"翻译层"，但质量不可控。**

### 4. 边界层

**那什么才是真正复制不了的？**

| 护城河类型 | 是否可复制 | 原因 |
|-----------|-----------|------|
| 原始数据（法条/案例） | ✅ 完全可复制 | 公开信息 |
| 翻译层（解读/话术/指令） | ⚠️ AI可生成，但质量差 | 需要行业经验+人工审核 |
| 3秒测试通过的内容 | ❌ 难复制 | 需要逐条验证+持续维护 |
| 真实用户验证过的内容 | ❌ 难复制 | 需要用户反馈循环 |
| 经纪人使用依赖 | ❌ 不可复制 | 需要时间+习惯养成 |
| "帮经纪人变专业"的品牌心智 | ❌ 不可复制 | 需要率先占位 |

**真正的壁垒不是内容本身，是"内容×验证×用户依赖×品牌心智"的组合。** 每一层单独都能被复制，但四层叠在一起，复制成本指数级上升。

### 5. 结论

**内容可复制性是真实风险，但不是致命的。应对策略：**

1. **翻译层是短期壁垒**——把法条翻译成经纪人能用的内容，AI能做但质量差，人工做质量高但慢。关键是用3秒测试卡住质量
2. **用户验证是中期壁垒**——内容被真实经纪人验证过=可信度高，AI生成的内容没人验证=可信度低
3. **品牌心智是长期壁垒**——"帮经纪人变专业"如果被风声率先占位，后来者只能是"另一个做知识库的"
4. **网络效应是终极壁垒**——经纪人越多，反馈越多，内容越精准，越不可替代

**一句话：法条谁都有，但"3秒测试通过+经纪人验证过+用习惯了"的内容，没人能抄。**

## 问题三：1人生产的天花板在哪？

### 1. 事实层

**当前产能：**
- 小眼镜1人，日均5-10条词条
- 已产出241条entries + 68条工具卡 + 17条案例 + 4张场景卡片
- 耗时：约2个月（含其他任务）

**持续维护成本：**
- 法条更新：民法典修订/新司法解释出台 → 涉及词条需更新
- 案例更新：新判例出现 → 案例库需补充
- 验证更新：3秒测试需定期重跑
- 估算：已有内容的维护约占产能的20%

**净产能：日均5-10条 × 80% = 4-8条/天**

### 2. 推断层

**不同目标量级的达成时间：**

| 目标 | 新增量 | 所需工作日 | 自然月（含周末） |
|------|--------|-----------|---------------|
| rental+trade闭环 | +200条 | 33-50天 | 1.5-2个月 |
| 6场景域全覆盖 | +560条 | 93-140天 | 4-6个月 |
| 含工具卡+案例+场景 | +800条(多类型) | 133-200天 | 6-9个月 |

**1人把6个场景域做到全覆盖，需要6-9个月。如果加上维护成本，可能需要9-12个月。**

这是乐观估计——假设没有其他任务干扰、小眼镜不生病、不处理其他工单。

### 3. 反证层

**有没有办法突破1人天花板？**

**方案A：AI辅助生产**
- 流程：爬取法条+案例 → AI生成初稿（consumerQ/simpleAnswer/brokerNote） → 小眼镜审核
- 产能提升：从5-10条/天 → 30-50条/天（初稿）→ 审核后20-30条/天
- 风险：AI生成的brokerNote可能不准确，需要逐条审核，审核本身就是瓶颈
- 边界：AI能做"翻译"，做不了"判断"——"这个案例值不值得收"、"这条brokerNote经纪人会不会照做"需要人的判断

**方案B：经纪人共创**
- 流程：积分驱动经纪人贡献内容（纠错/验证/补充/提问）
- 产能提升：取决于参与度，乐观估计日均5-10条UGC
- 风险：经纪人贡献的内容质量参差不齐，审核成本高
- 边界：需要先有用户基础，没有用户就没有共创——这是鸡生蛋问题

**方案C：招人**
- 流程：招1-2个内容运营，按模板生产
- 产能提升：2-3人团队 → 日均15-30条
- 风险：需要培训，产出质量需要把控
- 边界：资金成本，每月1-2万/人

**方案D：只做P0域**
- 流程：只做rental+trade，其他5个域暂不做
- 产能需求：+200条，1.5-2个月可达
- 边界：6个场景域只剩2个，"基础设施"的覆盖面不够

### 4. 边界层

**1人产能的天花板在哪？**

| 阶段 | 产能够用吗 | 天花板 |
|------|-----------|--------|
| Phase 1（rental+trade闭环）| ✅ 够 | 1人+AI辅助可完成 |
| Phase 2（4场景域）| ⚠️ 紧张 | 需要AI辅助+经纪人共创 |
| Phase 3（6场景域全覆盖）| ❌ 不够 | 必须扩人或缩范围 |

**1人的天花板大约在400-500条内容（含所有类型）。超过这个量级，要么加人，要么加AI，要么缩范围。**

### 5. 结论

**1人天花板在400-500条，约3-4个月产出。突破天花板的优先级：**

1. **AI辅助生产**——零成本增量，立即可用。法条→AI初稿→审核入库，效率提升3-5倍
2. **经纪人共创**——需先有用户基础，Phase 2启动
3. **只做P0域**——rental+trade闭环够用，不急着做6个域
4. **招人**——到Phase 3再说，现在用AI替代

**核心策略：用AI把1人产能从5-10条/天提到20-30条/天，400-500条天花板→800-1000条天花板。人不加，效率加。**

## 三个问题的交叉验证

| | 经纪人会用吗 | 内容可复制吗 | 1人天花板 |
|---|------------|------------|---------|
| 经纪人会用吗 | — | 用了就有依赖，依赖就是壁垒 | 有用户才有共创，共创突破天花板 |
| 内容可复制吗 | 用户验证过的内容更难复制 | — | AI生成可复制内容，但验证不可复制 |
| 1人天花板 | 天花板决定能不能满足用户 | 内容越多越难复制 | — |

**三个问题交汇在一个点：用户验证。**

- 经纪人会不会用 → 需要验证
- 内容不可复制 → 因为验证过
- 1人天花板 → 验证需要人，AI不能替

**所以所有任务的最高优先级不是生产内容，是找到经纪人验证。**
