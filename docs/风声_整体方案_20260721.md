# 风声 · 居住服务者专业基础设施 · 整体方案

> 生哥定调 · 2026-07-21
> 小眼镜梳理

---

## 一、定位

**居住服务者的专业基础设施**

不是工具，不是可选项，是底层依赖。就像水电煤是生活的基础设施，风声是居住服务者专业能力的基础设施。

定位链条：**居住服务者 → 专业 → 基础设施 → 记忆库 → 知识库 → 工具包**

- 记忆库：不用想就有（主动触发）
- 知识库：知道什么（分类/检索/验证）
- 工具包：用什么（方法论+佐证+案例的可执行方案）

目标用户：所有为居住提供专业服务的人。当前P0=房产经纪人，架构支撑物业管家/验房师/装修顾问/租赁管家。

---

## 二、核心壁垒

**能触发 + 给提示 + 给方法，三层焊死，行业无人做到。**

| 层级 | 能力 | 行业现状 |
|------|------|---------|
| 感知层 | 能触发——知道你现在需要什么 | 语音助手/眼镜能感知，但没行业专业记忆 |
| 判断层 | 给提示——告诉你该注意什么 | 法律AI能给提示，但不能场景触发 |
| 行动层 | 给方法——告诉你怎么做 | 工具卡是可执行方案，不是知识片段 |

本质：**客户的动作 → 触发 → 服务者该做什么**
- 贝壳在线上做到了（VR看房→推话术），风声要在真实对话中做到
- 贝壳覆盖交易场景，风声覆盖全生命周期
- 贝壳推销售支持，风声推专业支持

## 四、主动触发机制

### 4.1 触发优先级

| 触发方式 | 用户价值 | 优先级 | 原因 |
|---------|---------|--------|------|
| 语音触发 | ★★★★★ | **P0** | 80%时间在说话，零操作成本 |
| 对话流触发 | ★★★★★ | **P0** | 理解处境后推送，帮经纪人"接话" |
| 流程节点触发 | ★★★★ | P1 | 确定性强，不需要AI猜 |
| 位置/视觉触发 | ★★ | P2-P3 | 依赖硬件，锦上添花 |

### 4.2 触发闭环

```
客户说话 → 眼镜/麦克风采集 → 语音转文字
→ 风声trigger匹配 → 识别场景+处境
→ 推送oneLineAnswer（一句话结论）
→ 经纪人追问 → 展开keyPoints/legalBasis/cases
```

### 4.3 推送原则

- **轻推送**：只推oneLineAnswer，不推全文
- **可追问**：语音追问"展开"才展开详情
- **不打扰**：同一场景5分钟内不重复
- **可关闭**：经纪人可静默模式

## 五、知识库标准

### 5.1 入库三关

1. **找得到吗？** ≥2入口可达
2. **看得懂吗？** 3秒测试通过
3. **用得上吗？** brokerNote有明确动作指令

### 5.2 3秒测试

- 3秒看懂标题（客户能问出来的问题）
- 3秒理解结论（oneLineAnswer≤30字）
- 3秒知道下一步（brokerNote有明确动作指令）

### 5.3 双分类体系

- **数据域**（开发用）：policy/decor/trade/rental/homekeep等11个
- **用户场景域**（服务者用）：签约前/签约中/签约后/居住中/退租出售/职业成长

服务者按场景找，不按数据分类找。

### 5.4 五入口可达

场景导航 / 自然语言搜索 / 法条检索 / 案例检索 / 游戏化入口

### 5.5 验证核心

不是"产出了多少条"，是"经纪人能不能找到、看懂、用上"。

## 六、技术对接路径

### 6.1 知识底座如何被眼镜触发

**内容不写入眼镜，通过API被调用。**

| 路径 | 方式 | 状态 |
|------|------|------|
| 云端API | 眼镜感知→调风声API→返回专业内容 | P0，8月实现 |
| 智能体商店 | 封装为"房产经纪专业智能体"入驻GPASS/Rokid | P1，8-9月 |
| 端侧轻量模型 | 高频触发词压缩到眼镜端 | P3，2027年 |

### 6.2 标准接口

```
POST /memory/trigger
请求：signals(语音/位置/对话信号) + context(流程阶段/历史查询)
响应：triggered + sceneId + push(headline/oneLineAnswer/drilldown) + followUp
```

眼镜厂商只需要调这个API，不用管内部分类/索引/联动。

### 6.3 已开放平台

- 蚂蚁GPASS + 百宝箱：联合Rokid/雷鸟，已开放智能体开发
- Rokid YodaOS：支持用户记忆/知识图谱，与风声trigger_words直接对接
- Meta Ray-Ban SDK：已开放POV摄像头+麦克风API
- 字节豆包眼镜：二代主打"看见即服务"，待开放开发者平台

## 八、核心判断汇总

1. **居住服务者的专业基础设施**——缺了就不行的底层能力
2. **记忆库→知识库→工具包**三层递进，记忆库是根基
3. **能触发+给提示+给方法，三层焊死**，行业无人做到
4. **客户的动作触发服务者行为**——贝壳在线上做到了，风声要在真实对话中做到
5. **量不是壁垒，触发机制才是**——800条被动词条不如200条可触发
6. **先跑通闭环再填量**——rental+trade跑通 > 6个域半成品
7. **内容不写入眼镜，通过API被调用**——风声管记忆，眼镜管感知和展示
8. **搭数据是苦力活，搭触发机制才是护城河**
