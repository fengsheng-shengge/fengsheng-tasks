# 风声知识底座 · 词条体系5000+全景拆解规划

> 产出时间：2026-07-21  
> 当前基线：241条entries + 68条工具卡 = 309条  
> 目标规模：5000+条高质量词条，覆盖居住服务全生命周期6大场景域  
> 产品定位：居住服务者的专业基础设施（记忆库→知识库→工具包）

---

## 一、总体框架

### 1.1 设计原则

| 原则 | 说明 |
|------|------|
| **场景驱动** | 每条词条必须锚定具体业务场景，拒绝无场景的纯概念词条 |
| **法律为骨** | 涉及权责边界的词条必须有法源支撑（法典/法规/司法解释/判例） |
| **三秒可用** | 找得到+看得懂+用得上——服务者3秒内能定位并理解核心结论 |
| **深度不降** | 新词条保持现有词条深度（posSpeech/negSpeech/合规红线/消费者话术齐全） |
| **渐进覆盖** | M1→M2→M3 三个里程碑递进，先高频场景后低频场景 |

### 1.2 词条类型体系

| 类型 | 标记 | 说明 | 占比预估 |
|------|------|------|----------|
| 法律 | LAW | 法典条款/法规/司法解释/地方规定的核心解读 | 25% |
| 政策 | POL | 限购/限贷/税费/补贴/保障房等政策解读 | 15% |
| 流程 | PROC | 办事步骤/操作清单/时间节点/材料清单 | 18% |
| 标准 | STD | 国标/行标/地标/验收标准/质量标准 | 12% |
| 术语 | TERM | 行业术语/概念定义/专业名词解释 | 10% |
| 风控 | RISK | 风险识别/红旗信号/避坑指引/合规红线 | 12% |
| 案例 | CASE | 判例/纠纷案例/实操案例/典型场景 | 8% |

### 1.3 词条字段扩展说明

现有词条字段（241条entries）：`id/domain/name/alias/def/source/srcType/attrs/scene/posSpeech/negSpeech/consumerQ/simpleAnswer/consumerBenefit/severity`

**新增字段（5000+体系升级）**：

| 字段 | 说明 | 必填 |
|------|------|------|
| `entryType` | 词条类型：LAW/POL/PROC/STD/TERM/RISK/CASE | 是 |
| `sceneDomain` | 场景域：签约前/签约中/签约后/居住中/退租出售/职业成长 | 是 |
| `subScene` | 子场景：需求确认/房源匹配/合同条款解读/过户流程... | 是 |
| `priority` | 生产优先级：P0(核心)/P1(重要)/P2(补全)/P3(地方) | 是 |
| `dataSource` | 数据来源标注：法典名/标准号/政策文号/判例号 | 是 |
| `toolCardType` | 工具卡类型（可空）：决策卡/流程卡/评估卡/校准卡/风险卡/边界卡 | 否 |
| `relatedEntries` | 关联词条ID列表 | 否 |
| `localScope` | 地域范围：全国/一线城市/特定城市 | 否 |
