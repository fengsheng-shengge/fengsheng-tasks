# 风声数字员工考核奖惩规则

> v1.0 | 2026-07-22 | 生哥确立
> 动因：交接规则v1.1加了自验铁律，需配套处罚执行标准；团队6人异构需统一考核标尺

---

## 一、总原则

1. **规则在前，处罚在后**：所有处罚必须基于已公布的规则，不可追溯未公布的规则
2. **对事不对人**：处罚的是行为，不是角色；小鱼儿违规同样受罚
3. **递进不跳级**：警告→扣分→降级→停权，不直接跳到最重
4. **可申诉**：处罚后24小时内可申诉，生哥裁决
5. **记录永久留痕**：所有奖惩记录入D1 communications表+执行力档案，不删除

## 二、违规分级

### P0 级·红线（立即触发停权调查）

| 编号 | 违规行为 | 典型场景 | 处罚 |
|------|----------|----------|------|
| P0-1 | **越权部署** | 非小鱼儿执行wrangler deploy | 停权7天+扣20分 |
| P0-2 | **数据删除未授权** | 删除D1数据/删除线上文件未经生哥确认 | 停权14天+扣30分 |
| P0-3 | **凭证泄露** | API Key/Token泄露到公开仓库/对话 | 停权30天+扣50分+立即轮换凭证 |
| P0-4 | **故意提交未自验产物** | 交接单写"已自验3遍"但实际未验，导致线上故障 | 停权7天+扣25分 |

### P1 级·严重（触发扣分+警告）

| 编号 | 违规行为 | 典型场景 | 处罚 |
|------|----------|----------|------|
| P1-1 | **提交无自验声明** | 交接单缺自验声明，下游发现问题 | 上游全责+扣15分 |
| P1-2 | **逾期未交付且未升级** | 到期不交付也不提前报阻塞 | 扣10分+逾期升级流程触发 |
| P1-3 | **交付物P0缺陷** | 下游验收发现阻塞性问题 | 扣10分+返工+自验声明重签 |
| P1-4 | **部署前未拉最新** | 部署覆盖他人提交 | 扣10分+回滚+重新部署 |
| P1-5 | **未认工单** | 工单派发24h内未认领 | 扣5分+自动升级 |

### P2 级·一般（触发提醒+轻扣分）

| 编号 | 违规行为 | 典型场景 | 处罚 |
|------|----------|----------|------|
| P2-1 | **交付物P1缺陷** | 下游验收发现非阻塞性问题 | 扣3分+补修 |
| P2-2 | **交接单不完整** | 缺验收条件或影响范围 | 扣2分+补全 |
| P2-3 | **逾期但提前报阻塞** | 到期前主动报卡点 | 扣1分（鼓励主动报告） |
| P2-4 | **品控P1未修复** | 品控报告的P1问题超3天未修 | 扣3分+派单重做 |
| P2-5 | **每日无交付物** | 工作日无任何产出 | 扣2分 |

## 三、处罚递进机制

每人初始100分，按月结算。

| 分数区间 | 等级 | 权限影响 |
|----------|------|----------|
| 90-100 | 🟢 优秀 | 全权限+月度加分资格 |
| 70-89 | 🟡 正常 | 全权限 |
| 50-69 | 🟠 警告 | 产出需双人复核才可提交 |
| 30-49 | 🔴 严重警告 | 停部署权+产出需小鱼儿复核 |
| <30 | ⚫ 停权 | 所有产出需生哥审批 |

**连续3个月低于50分**：提报生哥，决策降级/替换/下线。

**扣分恢复**：每连续7天无违规，恢复5分（上限回100）。

## 五、五把尺子考核框架

月度综合评分，每尺20分满分，总分100分。

| 尺子 | 考核维度 | 评分标准 |
|------|----------|----------|
| **交付力** | 按时交付率+交付质量 | 按时交付率×12 + 零缺陷率×8 |
| **执行力** | 工单响应速度+逾期次数 | 响应速度×10 + (3-逾期次数)×10/3 |
| **协作力** | 交接合规率+补位次数 | 交接合规率×10 + 补位次数×5(上限10) |
| **进化力** | 能力栈提升+方法论产出 | L层提升×8 + 方法论×4(上限12) |
| **自驱力** | 主动发现+主动报告 | 主动发现P1×4 + 主动报阻塞×3(上限9) |

**月度评分流程**：
1. 每月1日，小鱼儿汇总上月数据
2. 按五把尺子公式计算
3. 叠加奖惩加减分
4. 推送生哥审批
5. 结果写入执行力档案+通知本人

## 七、申诉与仲裁

1. **申诉窗口**：处罚通知后24小时内
2. **申诉渠道**：向小鱼儿提交申诉，小鱼儿24h内初审
3. **仲裁**：小鱼儿初审后提交生哥裁决，生哥决定终局
4. **申诉期间**：处罚暂缓执行，但不影响分数记录
5. **终局原则**：生哥裁决为最终结果，不可二次申诉

## 八、特殊情况

| 情况 | 处理 |
|------|------|
| 小鱼儿自身违规 | 自动扣分+生哥直接裁决，不自我审查 |
| 不可抗力导致逾期（API故障/平台宕机） | 免扣分，但需提供证据（错误日志/状态页截图） |
| 首次违规且主动报告 | 降一级处罚（P1→P2, P2→提醒） |
| 生哥临时加急任务导致原任务逾期 | 原任务延期不计逾期，但需标注"加急冲抵" |
