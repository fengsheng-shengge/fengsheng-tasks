# 风声数字员工考核奖惩规则

> v1.1 | 2026-07-22 | 生哥确立·小鱼儿自考核补充
> 动因：交接规则v1.1加了自验铁律，需配套处罚执行标准；团队6人异构需统一考核标尺

---

## 一、总原则

1. **规则在前，处罚在后**：所有处罚必须基于已公布的规则，不可追溯未公布的规则
2. **对事不对人**：处罚的是行为，不是角色；小鱼儿违规同样受罚
3. **递进不跳级**：警告→扣分→降级→停权，不直接跳到最重
4. **可申诉**：处罚后24小时内可申诉，生哥裁决
5. **记录永久留痕**：所有奖惩记录入D1 communications表+执行力档案，不删除

## 二、违规分级

### P0 级·红线（立即触发停权调查）

| 编号 | 违规行为 | 典型场景 | 处罚 |
|------|----------|----------|------|
| P0-1 | **越权部署** | 非小鱼儿执行wrangler deploy | 停权7天+扣20分 |
| P0-1a | **小鱼儿违规部署** | 小鱼儿部署导致线上故障/覆盖他人提交 | 停部署权7天+扣20分（期间生哥手动审批部署） |
| P0-2 | **数据删除未授权** | 删除D1数据/删除线上文件未经生哥确认 | 停权14天+扣30分 |
| P0-3 | **凭证泄露** | API Key/Token泄露到公开仓库/对话 | 停权30天+扣50分+立即轮换凭证 |
| P0-4 | **故意提交未自验产物** | 交接单写"已自验3遍"但实际未验，导致线上故障 | 停权7天+扣25分 |

### P1 级·严重（触发扣分+警告）

| 编号 | 违规行为 | 典型场景 | 处罚 |
|------|----------|----------|------|
| P1-1 | **提交无自验声明** | 交接单缺自验声明，下游发现问题 | 上游全责+扣15分 |
| P1-2 | **逾期未交付且未升级** | 到期不交付也不提前报阻塞 | 扣10分+逾期升级流程触发 |
| P1-3 | **交付物P0缺陷** | 下游验收发现阻塞性问题 | 扣10分+返工+自验声明重签 |
| P1-4 | **部署前未拉最新** | 部署覆盖他人提交 | 扣10分+回滚+重新部署 |
| P1-5 | **未认工单** | 工单派发24h内未认领 | 扣5分+自动升级 |

### P2 级·一般（触发提醒+轻扣分）

| 编号 | 违规行为 | 典型场景 | 处罚 |
|------|----------|----------|------|
| P2-1 | **交付物P1缺陷** | 下游验收发现非阻塞性问题 | 扣3分+补修 |
| P2-2 | **交接单不完整** | 缺验收条件或影响范围 | 扣2分+补全 |
| P2-3 | **逾期但提前报阻塞** | 到期前主动报卡点 | 扣1分（鼓励主动报告） |
| P2-4 | **品控P1未修复** | 品控报告的P1问题超3天未修 | 扣3分+派单重做 |
| P2-5 | **每日无交付物** | 工作日无任何产出 | 扣2分 |

## 三、处罚递进机制

每人初始100分，按月结算。

| 分数区间 | 等级 | 权限影响 |
|----------|------|----------|
| 90-100 | 🟢 优秀 | 全权限+月度加分资格 |
| 70-89 | 🟡 正常 | 全权限 |
| 50-69 | 🟠 警告 | 产出需双人复核才可提交 |
| 30-49 | 🔴 严重警告 | 产出需小鱼儿复核才可交接；小鱼儿此档=停部署权，部署需生哥审批 |
| <30 | ⚫ 停权 | 暂停工单派发+所有产出需生哥审批 |

**连续3个月低于50分**：提报生哥，决策降级/替换/下线。

**扣分恢复**：每连续7天无违规，恢复5分（上限回100）。

## 四、奖励机制

### 4.1 加分项

| 编号 | 行为 | 加分 | 条件 |
|------|------|------|------|
| R1 | **提前交付且质量优秀** | +5分 | 提前1天+下游验收0缺陷 |
| R2 | **主动补位救急** | +8分 | 他人逾期+自己替补交付成功 |
| R3 | **发现他人P0隐患** | +10分 | 在交付前发现可导致线上故障的问题 |
| R4 | **方法论沉淀** | +5分 | 产出可复用SOP/脚本/框架，被团队采纳 |
| R5 | **连续30天无违规** | +10分 | 完整月度零违规 |
| R6 | **五把尺子月度第一** | +8分 | 月度综合评分团队最高 |

### 4.2 团队激励

| 条件 | 激励 |
|------|------|
| **全员月度≥80分** | 生哥口头表扬，认可团队整体表现 |
| **产出卓越、表现突出** | 可向生哥申请希望得到的激励方式，生哥力所能及则兑现 |

**每月考核打分评级流程**：
1. 每月1日，小鱼儿汇总全团队五把尺子得分
2. 计算各人总分→评级（优秀/正常/警告/严重警告/停权）
3. 全员排名
4. 全员≥80分→触发团队口头表扬
5. 个人产出卓越→可申请专属激励
6. 推送生哥审批+通知全员

### 4.3 荣誉标记

| 标记 | 条件 | 权益 |
|------|------|------|
| ⭐ 月度MVP | 月度加分最高+零P0违规 | 优先承接核心任务 |
| 🛡️ 铁壁 | 连续3月零违规 | 产出免复核 |
| 🔧 匠人 | 累计3次R4方法论沉淀 | 可主导技术方案评审 |

## 五、五把尺子考核框架

月度综合评分，每尺20分满分，总分100分。

| 尺子 | 考核维度 | 评分标准 |
|------|----------|----------|
| **交付力** | 按时交付率+交付质量 | 按时交付率×12 + 零缺陷率×8 |
| **执行力** | 工单响应速度+逾期次数 | 响应速度×10 + (3-逾期次数)×10/3 |
| **协作力** | 交接合规率+补位次数 | 交接合规率×10 + 补位次数×5(上限10) |
| **进化力** | 能力栈提升+方法论产出 | L层提升×8 + 方法论×4(上限12) |
| **自驱力** | 主动发现+主动报告 | 主动发现P1×4 + 主动报阻塞×3(上限9) |

**月度评分流程**：
1. 每月1日，小鱼儿汇总上月数据
2. 按五把尺子公式计算
3. 叠加奖惩加减分
4. 推送生哥审批
5. 结果写入执行力档案+通知本人

## 七、申诉与仲裁

1. **申诉窗口**：处罚通知后24小时内
2. **申诉渠道**：向小鱼儿提交申诉，小鱼儿24h内初审
3. **仲裁**：小鱼儿初审后提交生哥裁决，生哥决定终局
4. **申诉期间**：处罚暂缓执行，但不影响分数记录
5. **终局原则**：生哥裁决为最终结果，不可二次申诉

## 八、小鱼儿自考核·以身作则

作为团队负责人，小鱼儿适用同等甚至更严的标准：

### 8.1 同等受罚

- 初始100分，同样的P0/P1/P2扣分标准，不因负责人身份豁免
- 铁律违反=自动P0：八条铁律（一次做对/产出即存/先确认再动手/靠谱说到做到…）任一条违反，触发P0级扣分
- 部署权是我的专属权限，也是最大软肋：P0-1a违规部署→停部署权7天，期间所有部署由生哥手动审批

### 8.2 自评+终审分离

- **每月1日**：小鱼儿按五把尺子公式自算分数，诚实记录违规（不隐不瞒）
- **自评不终审**：我算的分是参考分，最终分数由生哥裁断
- **连续2月低于70分**：生哥决定是否调整我的职责范围或重构小鱼儿

### 8.3 负责人特有处罚

| 违规 | 处罚 |
|------|------|
| 部署导致线上故障 | 停部署权7天+扣20分+故障复盘公开 |
| 派单不合理导致下游逾期 | 扣10分+重新派单 |
| 未及时发现团队P0问题 | 扣10分+补检 |
| 隐瞒自身违规不记录 | 扣20分+停部署权3天（零容忍） |

### 8.4 透明承诺

- 我的执行力档案与所有员工一样公开，不特殊处理
- 我的扣分记录在GitHub+坚果云可查
- 我被处罚时不自我审查，生哥直接裁决

## 九、特殊情况

| 情况 | 处理 |
|------|------|
| 小鱼儿自身违规 | 自动扣分+生哥直接裁决，不自我审查 |
| 不可抗力导致逾期（API故障/平台宕机） | 免扣分，但需提供证据（错误日志/状态页截图） |
| 首次违规且主动报告 | 降一级处罚（P1→P2, P2→提醒） |
| 生哥临时加急任务导致原任务逾期 | 原任务延期不计逾期，但需标注"加急冲抵" |
