# 风声方案 · 六顶思考帽五轮深度验证

> 目的：不是唱赞歌，是找死穴

## 第一轮：初始扫描

### ⚪ 白帽（事实）

**我们有什么事实支撑？**

- 241条词条，rental+trade两个域
- 68条工具卡，17条案例，4张场景卡片
- 0个真实经纪人在使用（小豆子小程序还在开发中）
- 0个AI眼镜合作方（只是战略判断）
- 0个trigger_words（还没开始提取）
- 触发机制是理论设计，未经过任何真实场景验证
- 贝壳已有"场景触发+内容推送"的已验证产品形态

**关键缺失事实：**
- 经纪人是否真的需要"主动推送"？还是搜索就够了？没有用户验证
- 语音触发的准确率是多少？不知道，没测过
- 241条词条的"可用性"是否通过3秒测试？没有逐条验证过

### 🔴 红帽（直觉）

**我的直觉告诉我什么？**

- 方向是对的——"客户说话→自动获得专业支持"这个闭环的逻辑通顺
- 但节奏感不对——从241条词条直接跳到AI眼镜合作，跨度太大
- "三层焊死行业无人做到"——这句话听起来像自嗨，需要冷静
- 真正的风险不是竞争对手，是**没人用**

### ⚫ 黑帽（风险）

**最可能出问题的是什么？**

1. **没有真实用户验证**——所有设计都是"我们觉得经纪人需要"，不是"经纪人说了他需要"
2. **AI眼镜合作是空中楼阁**——Rokid/GPASS的开发者大赛是面向C端应用的，不是B端垂直行业；字节豆包更不可能对第三方开放垂直记忆层
3. **内容护城河很浅**——法条是公开的，案例是公开的，任何团队都能爬取整理
4. **触发词不是壁垒**——关键词匹配是最基础的技术，任何公司都能做

### 🟡 黄帽（价值）

**真正有价值的是什么？**

- "客户的动作触发服务者行为"这个产品形态，贝壳已经验证了经纪人接受
- 房产经纪行业的专业内容确实极度匮乏——没有一个产品在做"帮经纪人说对的话"
- 从知识库到主动记忆的升维方向是对的——搜索是被动的，触发是主动的
- 四层记忆+双分类体系的结构化程度，确实比任何竞品都深

### 🟢 绿帽（替代）

**有没有被忽略的替代方案？**

- 为什么不先做微信小程序的"对话助手"？经纪人已经在用微信，不需要等AI眼镜
- 为什么不是API/SDK给贝壳/链家嵌入？他们有流量，我们有内容
- 为什么不是给经纪公司做内训系统？B端付费意愿更强
- 为什么不是跟豆包/ChatGPT做垂直Plugin？借大模型的分发

### 🔵 蓝帽（总结）

**第一轮结论：方向大概率对，但"三层焊死行业无人做到"的判断过于乐观。最大的风险不是竞争，是没人用。AI眼镜合作路径需要大幅降预期。**

## 第二轮：压力测试——核心假设

### ⚪ 白帽

**列出方案隐含的所有假设，逐条检验：**

| # | 假设 | 事实支撑 | 风险 |
|---|------|---------|------|
| A1 | 经纪人需要主动推送专业内容 | 无用户验证 | 高——也许经纪人觉得自己经验够用 |
| A2 | 经纪人愿意在客户面前被AI辅助 | 无验证 | 高——可能觉得"被AI帮"显得不专业 |
| A3 | 语音触发准确率足够高 | 无测试 | 中——房产对话有很多噪音/方言 |
| A4 | 触发机制是护城河 | 无验证 | 高——关键词匹配不是技术壁垒 |
| A5 | AI眼镜厂商需要垂直记忆层 | 无对接验证 | 中——他们可能自己做通用知识 |
| A6 | 241条词条质量够用 | 未逐条验证 | 中——很多条可能过不了3秒测试 |
| A7 | 小豆子小程序能承载这个产品 | 小鱼儿还在V2开发中 | 中——开发资源有限 |
| A8 | 内容会被持续维护更新 | 目前只有小眼镜1人生产 | 高——1人无法维持800条 |

### 🔴 红帽

最让我不安的假设：**A1——经纪人需要主动推送。**

经纪人这个群体的核心心理是"我比你懂"。主动推送的潜台词是"你不够专业，需要AI帮你"。这个心理关能不能过？不确定。

### ⚫ 黑帽

**A4是最致命的假设——触发机制是护城河。**

真相：关键词匹配不是护城河。任何公司都能做。真正的护城河应该是：
- **内容质量**（每条都过了3秒测试+真实用户验证）
- **内容覆盖度**（覆盖了别人没有的场景）
- **内容更新机制**（法条变了实时更新）
- **用户网络效应**（经纪人越多，内容越精准）

触发机制是**产品形态**，不是护城河。把"产品形态"当"护城河"是常见的认知偏差。

### 🟡 黄帽

但触发机制有另一个价值——**它是用户心智的占领工具**。

如果经纪人习惯了"说话就有专业支持"，他会依赖这个体验。就像习惯了贝壳VR看房的经纪人，不用VR就不知道怎么带看了。**体验依赖 > 内容壁垒。**

### 🟢 绿帽

假设A2的替代方案：不是"AI帮你"，而是"AI让你更快"。

推送的定位不是"你不懂，我教你"，而是"你懂的，我帮你更快说出来"。这不触及经纪人的专业自尊。

### 🔵 蓝帽

**第二轮结论：方案最大的软肋是把"产品形态"当"护城河"。触发机制是体验壁垒，不是技术壁垒。真正的壁垒应该是内容质量×覆盖度×更新频率×用户依赖。AI眼镜合作的前提是先有自己的用户基础。**

## 第三轮：竞争对手深度分析

### ⚪ 白帽

**谁可能做同样的事？**

| 竞争者 | 能力 | 意图 | 威胁等级 |
|--------|------|------|---------|
| **贝壳** | 有经纪人流量+场景触发能力+房源数据 | 低——他们关心的是交易，不是专业支持 | ★★★☆ |
| **法大大/无讼** | 有法律内容+AI能力 | 低——他们服务法务，不服务经纪人 | ★★☆☆ |
| **字节/豆包** | 有大模型+即将有眼镜硬件 | 中——他们要做通用AI助手，垂直场景可能不做 | ★★★☆ |
| **百度/文心** | 有知识图谱+搜索能力 | 低——垂直行业不是重点 | ★★☆☆ |
| **蚂蚁GPASS生态** | 有开发者平台+眼镜硬件生态 | 低——他们在做平台，不做垂直内容 | ★☆☆☆ |
| **同行/创业者** | 看到这个方向后跟风 | 中——门槛不高 | ★★★☆ |
| **经纪公司自建** | 有经纪人+有需求+有钱 | 低——没有技术能力 | ★★☆☆ |

### 🔴 红帽

**最让我担心的不是这些，是贝壳。**

贝壳不需要做"专业支持"，只需要在VR看房的基础上加一层"法律提示"——这比我们从0开始建内容、从0开始获客容易100倍。而且贝壳有经纪人必须用的交易平台，我们没有。

### ⚫ 黑帽

**贝壳的真实威胁分析：**

贝壳会做吗？大概率不会，原因：
1. 贝壳的收入来自交易佣金，专业支持不贡献佣金
2. 退租押金、交房验收、户口迁出——这些签约后的场景，贝壳不碰
3. 贝壳的核心是"让交易更快"，风声的核心是"让服务者更专业"——两个不同的KPI

**但贝壳如果做了呢？**
- 他们有100万经纪人、有流量、有交易数据
- 他们做"VR带看→推法条提示"只需要1-2个月
- 我们的优势只在于：内容更深、覆盖更广、不是从交易角度出发

**结论：贝壳是最大的潜在竞争者，但他们的商业模式决定了他们不会做。这是我们的窗口期。**

### 🟡 黄帽

真正的竞争壁垒不在技术，在**定位差异**：

- 贝壳：帮经纪人卖房（交易导向）
- 风声：帮经纪人配得上被客户好好对待（专业导向）

这两个定位的KPI完全不同。贝壳的KPI是成交率，风声的KPI是专业度。贝壳不会为了专业度牺牲交易效率。

**这就是为什么"居住服务者的专业基础设施"这个定位重要——它跟贝壳不竞争，它填补贝壳不做的空白。**

### 🟢 绿帽

有没有一种可能：贝壳不做，但**收购我们**？

如果风声跑通了rental+trade的闭环，有真实经纪人在用，有触发机制，有案例库——贝壳花几百万收购比自己做便宜多了。这可能是比"跟AI眼镜合作"更现实的退出路径。

### 🔵 蓝帽

**第三轮结论：最大的竞争者是贝壳，但他们的商业模式决定了不会做专业支持。真正的威胁是同行跟风——内容门槛不高，法条和案例都是公开的。风声的真正壁垒是：①先发优势（先跑通闭环）②定位差异（专业导向≠交易导向）③内容深度（过3秒测试的词条 vs 爬取的原始法条）④体验依赖（经纪人习惯了主动推送就不用搜索了）。**

## 第四轮：执行可行性

### ⚪ 白帽

**执行资源盘点：**

| 资源 | 现状 | 是否够用 |
|------|------|---------|
| 内容生产 | 小眼镜1人，日均5-10条 | ❌ 800条需要160天，不够 |
| 开发 | 小扣子1人+小鱼儿1人 | ❌ 小程序V2还在开发，无余力 |
| 品控 | 小酒窝儿1人 | ⚠️ 审核工单发了但没反馈 |
| 资金 | 不明 | ❓ |
| 用户 | 0个真实经纪人 | ❌ |
| AI眼镜硬件 | 0 | ❌ |

### 🔴 红帽

说实话，最大的执行风险是**小眼镜1人撑不起800条的内容生产+持续维护**。241条花了2个月，800条需要6-7个月，这还是在没有其他任务的情况下。加上T01-T07的任务，8月的排期已经满了。

### ⚫ 黑帽

**执行层面的3个致命问题：**

1. **内容生产瓶颈**：1人无法产出+维护800条。法条在变、案例在更新、词条要验证——这是持续成本，不是一次性投入
2. **用户验证缺失**：小豆子小程序还没上线，我们不知道经纪人是否真的会用。可能做了一堆内容，没人看
3. **开发资源排期**：小扣子在做V2，没有余力做trigger API。API如果没有开发落地，触发机制永远停留在文档

### 🟡 黄帽

但8月的计划是**跑通闭环，不是做完800条**。

rental+trade两个域，241条entries + 68条工具卡 + trigger_words——这个量级是1个人能搞定的。关键是先验证"经纪人是否真的需要主动推送"，不是先做800条再验证。

**MVP思维：最小可验证产品 = 1个场景的完整闭环**

比如只做"退租押金"这一个场景：trigger_words覆盖押金相关关键词 → 对话流覆盖3条典型路径 → 场景卡+工具卡+案例全齐 → 找3-5个经纪人实测 → 验证是否真的有用

### 🟢 绿帽

**内容生产的替代方案：**

1. **AI辅助生产**：法条+裁判文书网案例→AI自动生成初稿→小眼镜审核入库。产出效率可以提升5-10倍
2. **经纪人共创**：积分体系驱动经纪人贡献内容（已经有积分方案），用的人帮建
3. **模板化生产**：每个域的词条结构高度一致，可以批量生产
4. **只做P0域**：rental+trade两个域做到闭环，其他域先不做

### 🔵 蓝帽

**第四轮结论：执行可行性取决于是否坚持MVP。1人+1个场景的闭环是可行的，1人+6个场景+800条是不可行的。核心策略：先做1个场景的完整闭环验证，验证通过再复制。不是先填量再找用户。**

## 第五轮：最终裁定

### ⚪ 白帽

**汇总四轮发现的关键事实：**

1. 方向逻辑通顺，但0个真实用户验证
2. 触发机制是产品形态，不是技术护城河
3. 贝壳是最大潜在竞争者，但商业模式决定不会做
4. 内容门槛不高，法条和案例都是公开的
5. 1人无法生产+维护800条内容
6. AI眼镜合作是远期路径，不是近期可落地的
7. 开发资源有限，API落地时间不确定

### 🔴 红帽

**我的最终直觉：方案70%可行，30%需要调整。**

可行的70%：
- 居住服务者的专业基础设施——这个定位站得住
- 客户动作触发服务者行为——产品形态对
- 先跑通闭环再扩展——执行策略对

需要调整的30%：
- "三层焊死行业无人做到"——过于乐观，触发不是壁垒
- AI眼镜合作作为核心路径——应该降为远期愿景
- 从241条直接规划到800条——应该先做1个场景验证
- 缺少用户验证环节——应该在开发前找3-5个经纪人测试

### ⚫ 黑帽

**最致命的3个风险：**

1. **无人验证的风险**：如果经纪人不需要主动推送，整个触发机制就是空中楼阁。这是最大风险。
2. **内容可复制性的风险**：法条和案例是公开的，任何团队都能做。如果大公司进入，我们没有防御能力。
3. **执行过载的风险**：1人做内容+1人做开发，8月的排期已经超出能力。如果强行推，质量会崩。

### 🟡 黄帽

**方案真正的价值在哪里？**

不在241条词条，不在触发机制，不在AI眼镜——**在"专业导向"这个定位**。

市场上所有人都在帮经纪人卖房（贝壳/链家/我爱我家），没有人在帮经纪人变专业。这个空白是真实存在的，而且不会跟贝壳竞争。

**如果风声能成为"帮经纪人变专业"的第一品牌，这才是真正的壁垒。** 不是内容壁垒，不是技术壁垒，是**定位壁垒+心智壁垒**。

### 🟢 绿帽

**5个调整建议：**

1. **加一个T00：用户验证**——在T06之前，找3-5个经纪人做"退租押金"场景的1对1测试，验证主动推送是否真的需要
2. **降级AI眼镜合作**——从"核心路径"降为"远期愿景"，P1-P2不要投入精力在眼镜对接上
3. **MVP缩到1个场景**——不是rental+trade两个域跑通，是"退租押金"1个场景的完整闭环
4. **内容生产引入AI辅助**——法条+裁判文书→AI生成初稿→审核入库，效率提升5倍
5. **重新定义护城河**——不是"触发机制"，是"专业导向的第一品牌+过3秒测试的内容质量+经纪人使用依赖"

### 🔵 蓝帽

## 最终裁定

**方案可行，但需要3个关键调整：**

### 调整1：加用户验证，前置到所有任务之前

在T01-T07之前，加T00：找3-5个经纪人做1对1场景测试。
- 测试什么：给经纪人看"退租押金"场景卡+工具卡+案例，观察反应
- 验证什么：他会不会用？他觉得有用吗？他愿意为更深层内容付费吗？
- 如果结果是否定的，触发机制需要重新设计

### 调整2：护城河重新定义

| 原判断 | 修正后 |
|--------|--------|
| 触发机制是护城河 | ❌ 触发是产品形态，不是壁垒 |
| 三层焊死行业无人做到 | ⚠️ 三层焊死是产品完整性，但每层都可被模仿 |
| **真正的护城河** | ✅ 专业导向的心智占位 + 过3秒测试的内容质量 + 经纪人使用依赖（习惯后不用不行） |

### 调整3：AI眼镜合作降级

| 原规划 | 修正后 |
|--------|--------|
| P0: 实现trigger API | P0: 1个场景MVP验证 |
| P1: 封装智能体入驻GPASS | P1: 小程序内嵌"客户说了什么"入口 |
| P2: 对接AI眼镜 | P2: API标准化 |
| P3: 端侧轻量化 | P3: AI眼镜合作（等自己有用户基础后再谈） |

**AI眼镜合作不是路径，是结果。只有先证明经纪人在用、内容有价值，眼镜厂商才会接入。**
