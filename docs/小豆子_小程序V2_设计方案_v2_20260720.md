## 一、产品定位（继承设计共识）

**经纪人专业的智能工具夹**——不是话术工具，不是学习平台，是在消费者面前证明经纪人专业、值得信任和托付的助力产品。

### 核心发心
让消费者觉得服务者靠谱、专业、值得托付和信任。

### 与话术工具的本质区别
| 话术工具（不是） | 智能工具夹（是） |
|---|---|
| 给话术让经纪人背 | 给依据让经纪人讲 |
| 客户感受到"套路" | 客户感受到"专业" |
| 一次性消费 | 越用越强 |
| 标准化输出 | 个性化+场景化输出 |

### 7.2 导师对话铁律（区别于话术工具）

- ❌ 不直接给话术让经纪人背
- ✅ 给依据让经纪人讲
- ✅ 每条建议必须附带：原理 + 依据（政策/案例/数据）
- ✅ 先理解客户真正在顾虑什么，再给策略

### 7.3 对话示例

```
👤 客户说"再看看"，怎么回？

🤖 客户说"再看看"通常不是真的要再看看，而是在顾虑：
   1. 价格是否合理（怕买贵）
   2. 房源是否靠谱（怕踩坑）
   3. 决策是否着急（怕后悔）

   建议应对：
   1. 给标准：用六维测评展示房源品质
   2. 给数据：展示 90 天成交均价对比
   3. 给时间：明确决策窗口期+依据

   📋 依据：
   · 《民法典》第X条（房源信息披露）
   · 该小区 90 天成交均价 XX 万
   · 行业惯例：决策窗口期通常 3-7 天

   💬 递给客户时可以这样说：
   "我理解您想再考虑，这个房子我帮您做了六维测评，
    成交数据也整理好了，您看完再决定不晚。"
```

## 十三、交互流程图（关键路径）

### 13.1 新用户首次使用流程

```

  → 首页（Hero 区显示 Lv1 见习经纪 + 50 积分）

  → 提示「解锁完整案例需 5 分」
  → 答题赚积分 → 解锁
```

### 13.2 贡献内容流程

```
用户在词条详情页点击「纠错」
  → 进入纠错表单
    → 选择错误类型
    → 填写正确内容
    → 上传证据（法条原文/案例链接）
  → 提交
  → 进入审核队列（小眼镜+小酒窝儿）
  → 审核通过
    → 积分到账（+20）
    → 词条标注贡献者署名
    → 通知用户「你的贡献已上线」
  → 审核不通过
    → 反馈原因
    → 不扣分
```

```
见客户前：
  → 首页「交易卡点雷达」前置排查

  → 导师页模拟对话准备

见客户时：
  → 客户提问
  → 掏手机查政策/数据给客户看（唯一正当场景）
  → 词条详情页→分享客户版卡片

见客户后：
  → 回到词条详情页
  → 点击「实战验证」记录客户反应
  → 获得 +10 积分
```

## 十四、设计决策回溯表

| 设计决策 | 回溯到设计共识铁律 |
|---|---|
| 4 张场景卡片而非 4 张功能卡片 | 定位翻转：场景洞察+转化工具双驱动 |
| 词条详情页专业理解+法律依据免费 | 积分底线：基础内容永远免费 |
| 导师对话必须附带依据 | 言之有物：每个输出必须有依据 |
| 游戏化每 3-5 关穿插知识 | 趣味性是手段，专业内容是目的 |
| 积分获取只奖励专业成长行为 | 用户贡献飞轮：时间不可压缩 |
| 场景卡片含客户原话 | 理解先于应对：先理解客户真正在顾虑什么 |
| 贡献署名展示 | 用户贡献飞轮：被收录+署名→其他人看到→也贡献 |
| 见客户时只查政策/数据 | 在客户面前掏手机=只有查政策/数据给客户看是正当场景 |
