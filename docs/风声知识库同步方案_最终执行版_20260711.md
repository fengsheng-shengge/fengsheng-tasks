## 二、自动同步链路（CodeAct定时脚本）

### 2.1 每日08:00自动执行

**触发方式**：Calendar定时 → CodeAct脚本（非crontab，沙箱不支持cron）

**脚本功能**：
1. 拉取GitHub Closed Issues → 生成快照到Obsidian Vault
2. 拉取GitHub Wiki → 同步到Obsidian Vault
3. entries.json → 推GitHub + 推坚果云（两步合一，不再分开做）
4. 写同步日志

### 2.2 每周一09:00完整性校验

**触发方式**：Calendar定时 → CodeAct脚本

**校验内容**：
- GitHub最新commit SHA vs 本地entries.json一致性
- 坚果云文件存在性+大小校验
- Obsidian Vault目录完整性
- 同步日志异常检查

### 2.3 entries.json更新后即时推送

**触发方式**：词条生产完成后立即调用

**流程**：
1. entries.json写入workspace
2. CodeAct脚本：Git Data API推GitHub（blob→tree→commit→ref）
3. CodeAct脚本：WebDAV PUT推坚果云
4. 两步在一个脚本里完成，不会漏

**为什么不用Obsidian Git插件**：
- Agent workspace在云端沙箱，无法安装Obsidian
- Git插件适合有人工Obsidian桌面端的场景
- CodeAct脚本更可靠，不依赖Obsidian运行环境

## 四、扣子知识库导出（小眼镜执行，一次性）

### 4.1 为什么只做一次

- entries.json是唯一数据源（7.5铁律）
- 扣子知识库只是展示层，不含增量知识
- 扣子无公开API，自动化成本高且不稳定
- 导出一次归档即可，后续增量走entries.json→坚果云→Obsidian

### 4.2 导出SOP

| 序号 | 导出内容 | 方式 | 存放位置 |
|------|---------|------|---------|
| 1 | 扣子Bot知识库源文件 | Bot设置→知识库→逐个下载 | `内容素材/扣子知识库/` |
| 2 | 扣子Bot QA对 | Bot设置→知识库→导出QA | `内容素材/扣子知识库/` |
| 3 | 小鱼儿System Prompt | 复制粘贴到md文件 | `方法论知识库/小鱼儿System_Prompt.md` |
| 4 | 决策日志 | 如有则导出 | `方法论知识库/决策日志/` |

### 4.3 导出后

- 坚果云已有备份：`/风声协作/风声知识库/`下全部文件
- Obsidian通过Remotely Save自动拉取
- 无需再做第二次

## 七、风险与应对

| 风险 | 概率 | 应对 |
|------|------|------|
| 坚果云月1GB流量限制 | 低 | 全是md/json，单次几十KB，月增量<50MB |
| Git Data API写权限变更 | 低 | 已验证blob→tree→commit→ref链路可用 |
| CodeAct脚本执行失败 | 中 | 心跳巡检+每周校验双重保障 |
| 扣子知识库导出格式不一致 | 低 | 导出后统一转md，人工检查一次 |
| Obsidian Vault与workspace路径不对齐 | 中 | 坚果云作为中间层，Remotely Save从云端拉取 |
