# 经验回流协议

> 来源：AI团队共享记忆方法论（2026-07-12提炼）
> 核心原则：验证过的经验必须回流到可自动调取的位置，不允许只存在对话记忆中

---

## 为什么要回流

AI团队最大的浪费不是token，是验证过的经验没有被下一次复用。
- 工作记忆（对话上下文）会丢失
- 长期记忆（结构化文档）才是资产
- 不回流 = 每次重新踩坑

## 回流流程

```
执行任务 → 产生经验 → 验证通过 → 压缩摘要 → 写入TOOLS.md锚点+experience/详情
```

### 什么时候回流

1. **话术三关通过后**：词条生产中的关键决策点
2. **脚本部署成功后**：环境配置/绕过方案/坑点
3. **方案被生哥确认后**：决策逻辑+约束条件
4. **踩坑并找到解法后**：问题+原因+方案+适用条件

### 回流格式

TOOLS.md锚点行：
```
- **短标题**：一句话结论。详见 基础设定/experience/xxx.md
```

experience/详情文件：
```markdown
# 短标题

## 问题
遇到了什么

## 原因
为什么会这样

## 解决方案
怎么解决的

## 适用条件
什么时候会再遇到

## 验证日期
YYYY-MM-DD
```
