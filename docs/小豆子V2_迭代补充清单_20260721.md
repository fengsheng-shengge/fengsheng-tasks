# 小豆子V2小程序 · 迭代补充清单

> 基于7.21产品定位升级+六顶思考帽验证结论
> 小眼镜 · 2026-07-21

## 背景

7.21确定了三个关键结论，直接影响小程序设计：

1. **产品定位升级**：智能工具夹 → **居住服务者的专业基础设施**（记忆库→知识库→工具包）
2. **用户验证是最高优先级**：0个用户验证=0个产品验证，必须在小程序里加验证入口
3. **触发机制是核心方向**：感知+判断+行动三层焊死，小程序是触发验证的最佳载体

## 迭代清单

### P0-01：首页加"客户说了什么"入口

**问题**：V2首页4张场景卡片是经纪人主动选场景，缺一个"输入客户原话→系统匹配场景"的入口。这是触发机制的MVP，也是用户验证的关键入口。

**方案**：在首页搜索框下方加一个"客户说了什么？"输入条

```
┌──────────────────────────┐
│  🔍 搜索词条/别名          │
├──────────────────────────┤
│  💬 客户说了什么？         │
│  "房东不肯退我押金"  [→]  │
├──────────────────────────┤
│  每日一题                  │
│  ...                      │
```

**交互流程**：
1. 经纪人输入/粘贴客户原话
2. 后端用68条search_aliases做简易trigger匹配
3. 匹配成功 → 推送场景卡+工具卡+案例
4. 匹配失败 → 引导到词典搜索+导师对话

**后端实现**：
- 第一版用search_aliases关键词匹配，不需要trigger_words
- 匹配逻辑：分词→hit alias→返回对应toolId的场景卡
- 7.31前T04搜索链路验证完成后可升级为语义匹配

**验证指标**：
- 使用率：有多少经纪人用了这个入口
- 匹配成功率：输入后匹配到内容的比例
- 点击转化率：匹配成功后点击展开详情的比例

### P0-02：行为埋点4核心指标

**问题**：V2方案没有定义行为埋点，没有埋点=没有验证=不知道经纪人会不会用。

**4个核心埋点**：

| 指标 | 埋点事件 | 定义 | 判断标准 |
|------|---------|------|---------|
| 点击率 | `content_click` | 看到卡片/词条→点击进入详情 | >30%为合格 |
| 展开率 | `content_expand` | 进入详情→展开案例/备忘/依据 | >50%为合格 |
| 收藏率 | `content_collect` | 展开内容→点击收藏 | >15%为合格 |
| 追问率 | `content_followup` | 收藏后→点击关联词条/导师对话 | >10%为合格 |

**额外埋点（P0-01验证用）**：

| 指标 | 埋点事件 | 定义 |
|------|---------|------|
| 触发入口使用率 | `trigger_input_use` | 首页"客户说了什么"输入条点击 |
| 触发匹配成功率 | `trigger_match_success` | 输入后匹配到内容 |
| 触发转化率 | `trigger_to_detail` | 匹配成功后点击进入详情 |

**埋点规范**：
```json
{
  "event": "content_click",
  "timestamp": "2026-07-22T10:30:00+08:00",
  "userId": "xxx",
  "contentId": "RNT-008",
  "contentType": "tool_card",
  "source": "trigger_input | search | scene_card | dictionary",
  "sceneId": "SCENE-01",
  "metadata": {}
}
```

### P1-04：词条详情页时效标注

**问题**：经纪人信任内容的前提是知道法条有没有过期。V2词条详情页没有"最后更新日期+法条版本号"。

**方案**：在法律依据区块下方加时效标注条

```
⚖️ 法律依据
《民法典》第710条、第712条

📅 时效验证
最后更新：2026-07-18 | 法条版本：现行有效
[法条原文 →]
```

**数据字段**：
```json
{
  "lastVerified": "2026-07-18",
  "legalStatus": "现行有效 | 已修订 | 部分失效",
  "legalVersion": "民法典（2021.1.1施行）"
}
```

**审核规则**：
- policy类词条：每季度核查，超90天未验证标黄"待验证"
- 非policy类：每半年核查，超180天未验证标灰"需确认"

### P1-05：6大场景域导航

**问题**：V2首页4张卡片只覆盖rental+trade的4个固定场景，经纪人可能不在当前4个场景里，但他在某个场景域里。

**方案**：首页场景洞察区下方加场景域导航条

```
── 场景域 ──
[签约前] [签约中] [签约后] [居住中] [退租出售] [职业成长]
```

**点击交互**：
- 进入场景域列表页，展示该域下所有场景卡片
- 每个场景卡片：场景名+痛点一句话+关联工具卡数+关联案例数
- 点击场景卡片→场景详情页（复用V2已有详情页结构）

**6大场景域×内容映射**：

| 场景域 | 英文标识 | 现有词条数 | 工具卡覆盖 |
|--------|---------|-----------|-----------|
| 签约前 | pre-contract | trade(37)+rental(31) | 20+ |
| 签约中 | in-contract | trade+policy | 15+ |
| 签约后 | post-contract | trade+homekeep | 10+ |
| 居住中 | living | homekeep(32)+decor(37) | 15+ |
| 退租出售 | exit | rental(31)+trade | 12+ |
| 职业成长 | career | talent(21)+quality | 8+ |

### P2-06：导师页对话流驱动

**问题**：V2导师页是通用AI对话，没有接入对话流索引。

**方案**：等T07 dialog_flow_map.json产出后，导师页根据经纪人的输入自动匹配对话流路径

**交互**：
1. 经纪人输入客户原话
2. 匹配到对话流节点
3. 导师沿路径引导：客户说X→他可能在顾虑Y→你可以用Z方法→依据是W
4. 每一步附带场景卡+工具卡+案例

**前置依赖**：T07 dialog_flow_map.json（截止8.8）

### P2-07：共创审核自动化

**问题**：V2贡献流程是人工审核，1人天花板论证发现必须用经纪人共创突破产能，人工审核会成为瓶颈。

**方案**：自动化初筛+人工终审

```
用户提交纠错/验证
  → 自动初筛（规则引擎）
    · 格式校验：必填字段是否完整
    · 来源校验：引用法条是否存在
    · 重复校验：是否已有相同纠错
    · 置信度评分：AI判断内容可信度
  → 置信度≥80%：直接入库，标记"待复核"
  → 置信度<80%：进入人工审核队列
  → 小眼镜/小酒窝儿终审
  → 审核结果通知用户
```

**前置依赖**：用户量达到日均10+条贡献时启用
