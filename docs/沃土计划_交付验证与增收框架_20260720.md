# 沃土计划 · 交付验证与增收框架

> 日期：2026-07-20
> 目标：2026年实现¥100万营收（333人×¥3000/人）
> 破局思路：验证交付质量→获得业务认可→扩大供给规模→收入闭环

---

## 一、问题诊断：闭环为什么没跑通

```
业务端                    高校端                   学生端
交付质量没验证 →          短训对学校无政绩价值 →    无制度约束 →
不认可 → 不付费 →         不投入 → 不设学分/制度 →   参与随意 →
各城市意愿度低            配合度低→无法批量          输出质量不稳定
     ↕                          ↕                        ↕
     └──────────── 三端利益未对齐 ────────────────────────┘
```

**根因**：交付质量没有业务认可的定义→没有验证→没有口碑→业务不信→不付费→闭环断

## 三、Phase 1：沈阳深度验证

### 3.1 验证设计

沈阳13个学员到店，6-7个门店，平均每店2人。
**业务端付费条件：学员1个月仍在岗即付费** → 验证周期30天就够！

**验证周期：30天主验证+30天辅助验证**

| 维度 | 指标 | 30天主验证 | 数据来源 | 对比基准 |
|---|---|---|---|---|
| **留存（核心！）** | 到店30天在岗率 | ✅ 直接决定付费 | 门店人事 | 校招30天留存率 |
| **上手速度** | 首次独立带看天数 | ✅ | 带教老师 | 校招平均天数 |
| **业务评价** | 店长评分（1-5分） | ✅ | 直接问 | 校招平均分 |

30天辅助验证（加分项，不阻塞扩城）：

| 维度 | 指标 | 60天出 | 数据来源 |
|---|---|---|---|
| **开单** | 首单周期 | ✅ | 门店业务数据 |
| **工具使用** | AI工具使用频次 | ✅ | 风声工具夹数据 |

**验证成功标准**：30天留存率>校招基准（这是业务付费的直接依据）

### 3.2 验证执行

| 时间 | 动作 | 负责人 |
|---|---|---|
| 7月底 | 沈阳门店对接+跟踪机制启动+店长问卷设计 | 项目负责人/小鱼儿 |
| **8月底** | **30天数据出表→留存率+店长评价→直接判定付费** | 小鱼儿 |
| **8月底** | **拿30天验证结果扩城：去2-3城谈合作** | 生哥 |
| 9月 | 辅助验证：60天数据（首单+工具使用） | 小鱼儿 |
| 9月 | 完整案例定稿（30天核心+60天加分项） | 小鱼儿 |

**关键节奏**：8月底30天数据一出就两件事并行——①沈阳门店按协议付费 ②拿数据去扩城。不等60天。

## 五、高校端破局：短训→学分制订单班

### 5.1 问题

短训→组织难→学校配合度低→无法批量→供给量上不去

根因：**短训对学校没有政绩价值**

### 5.2 对齐学校利益

| 学校要什么 | 短训能给的 | 订单班能给的 |
|---|---|---|
| 就业率/就业质量 | 弱（2周培训不算就业） | 强（正式实习+就业出口） |
| 产教融合成果 | 弱（无制度绑定） | 强（学分制+协议合作） |
| 院系政绩 | 弱（零散参与） | 强（专业/学院级合作） |
| 学生满意度 | 中（有培训但无约束） | 强（有学分+有实习+有补贴） |
| 企业资源引入 | 弱 | 强（贝壳品牌+门店+导师） |

### 5.3 订单班合作模型

```
短训模式（现在）                    订单班模式（目标）
─────────────                    ─────────────
2周在校培训                      学期制（1学期或1学年）
无学分                           纳入学分体系（2-4学分）
自愿参加                         专业必修/选修（制度约束）
零散组织                          专业/学院级批量组织
学校不投入                        学校出场地+排课+师资对接
无实习保障                        协议保障实习岗位+补贴
```

### 5.4 订单班谈判要点

**对学校说**：
- "我们不是来办个短训班，是来建**产教融合订单班**"
- "学生修完拿学分+贝壳认证+实习岗位+补贴，就业率你直接涨"
- "贝壳出课程+出导师+出实习出口，学校出学分+出排课+出组织"
- "这是教育部推的产教融合方向，对你的政绩是加分项"

**合作级别递进**：

| 级别 | 合作深度 | 产出规模 | 难度 |
|---|---|---|---|
| L1 单次短训 | 2周/自愿 | 10-20人 | 低（已跑通） |
| L2 学分选修 | 1学期/选修学分 | 30-50人 | 中 |
| L3 专业订单班 | 1学年/必修学分 | 50-100人 | 高 |
| L4 学院级合作 | 多专业/共建课程 | 100+人 | 最高 |

**今年目标**：从L1推到L2，争取1-2所学校到L3

## 十四、收入倒推与节奏（7.20更新）

```
¥100万 = 333人 × ¥3000/人

倒推：
  333人 ÷ 50人/校 = 7校
  7校 ÷ 2校/城 = 4城（含沈阳已有）
  
  Phase1(7-8月)：沈阳13人验证→门店付费→案例出笼
  Phase2(9月)：扩2-3城→秋招签约开班→2-3校×50人=100-150人
  Phase3(10-12月)：秋招培训到店→4城7校×50人=350人→超目标
  
  收入节奏：
  8月：沈阳13人×¥3000=¥3.9万（验证期收入）
  10月：Phase2到店100-150人→¥30-45万
  12月：Phase3到店350人→累计¥105万✅
```

关键假设与风险：
| 假设 | 风险 | 应对 |
|---|---|---|
| 沈阳30天在岗率≥85% | 学员流失率高 | 30天前项目负责人密切跟踪 |
| 店长评价均值≥3.5 | 业务不认可差异价值 | 人才报告升级+作品证据+对比数据 |
| 秋招前能签2-3校 | 高校配合度低 | 学分制订单班方案+学校利益对齐 |
| 每校出50人 | 跨专业组织难 | 学分约束+实习稳定性考核 |
