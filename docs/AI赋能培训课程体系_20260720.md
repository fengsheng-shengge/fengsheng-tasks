## 写在前面：这套体系从哪来

不是从PPT里编出来的，是从"做"里长出来的——

**生哥的AI学习路径（真实经历）**：
```
2025年初  开始关注AI → 听傅盛/王煜全/夏鹏 → 拓展认知
2025年中  下场做 → 培养小鱼儿 → 踩坑迭代 → 真正理解AI能力边界
2025年底  带6人异构AI团队 → 管调度管考核管进化 → 知道AI怎么组织化使用
2026年    亲自教学生1天AI实操 → 发现1天根本不够 → 决心把AI实操拉成持续训练
2026.7    遇到TRAE知识库(40万字) → 发现方法论可以系统化 → 这套课出来了
```

**核心体悟**：
1. **只听课不实操=0**。看10遍不如自己做1遍（费曼学习法）
2. **AI不是工具，是同事**。工具你用完就放，同事你跟它磨合越用越好
3. **场景是唯一的入口**。没有人想学"AI"，所有人都想解决"我工作里的XX问题"
4. **做到才能讲到**。没亲自用AI产出过东西的人，讲不清楚

这套体系把这些体悟结构化，让每个人都能走通这条路。

## 一、课程设计哲学

### 1.1 道法术器四层

| 层次 | 回答什么 | 对应内容 | 学时占比 | 交付物 |
|---|---|---|---|---|
| **道** | 为什么用AI？AI能改什么？ | 认知升级+思维转变 | 15% | 认知笔记（用自己的话说） |
| **法** | 怎么学？怎么用才有效？ | 方法论+学习框架 | 20% | 方法论卡片（可复用） |
| **术** | 具体怎么做？什么场景怎么操作？ | 6大场景实操指南 | 45% | 作品证据（AI产出物） |
| **器** | 用什么工具？ | TRAE Work/豆包/风声工具夹 | 20% | 工具使用频次数据 |

**学习路径**：道→法→术→器，但不是串行，是**螺旋上升**：
- 第一圈：知道为什么(道) + 学1个方法(法) + 做1个场景(术) + 装1个工具(器)
- 第二圈：深一层理解(道) + 优化方法(法) + 做3个场景(术) + 熟练2个工具(器)
- 第N圈：自己悟出新道→教别人(费曼)→内化

### 1.2 五条铁律

1. **场景驱动**：不讲"AI能做什么"，讲"你遇到XX问题时，AI怎么帮你"
2. **产出即证据**：每节课必须有可交付的AI产出物，不是笔记是作品
3. **费曼检验**：学完用自己的话讲出来，讲不清楚=没学会
4. **行业适配**：方法通用，场景行业化。同一套法术器，经纪人/设计师/顾问各自场景不同
5. **持续进化**：不是培训完就结束，是到工作中继续用→越用越强

## 三、模块0：认知破冰（道）

> **目标**：从"AI是工具"→"AI是让你补短板的同事"
> **生哥体悟**：经纪人经验不足是短板，AI能补。但前提是你知道怎么跟AI"说话"

### 3.1 第1节：AI能改什么（45min）

**破除三个误解**：
| 误解 | 真相 | 生哥原话佐证 |
|---|---|---|
| AI会取代我 | AI取代的是重复劳动，取代不了专业判断+人际关系 | "风声不是AI替代，是智能工具夹" |
| 我不需要AI | 你不用的同行在用，差距只会越来越大 | "通过下场做来拓展认知边界" |
| AI很难学 | 难的不是技术，是"怎么描述你要什么"(Prompt) | "学会了套路≠内化，做到才能讲到" |

**认知实验**（课堂实操，15min）：
- 全员用同一个AI工具（豆包/TRAE），同一个任务："写一份XX报告"
- 不给Prompt模板，自由写 → 对比结果质量差距
- **结论**：同样的AI，差距在Prompt（你怎么跟AI说话）

### 3.2 第2节：怎么跟AI说话（45min）

**Prompt三要素**（从TRAE方法论提炼+生哥实践验证）：

```
好的Prompt = 角色设定 + 任务边界 + 输出要求

❌ 坏示例："帮我写个报告"
✅ 好示例：
"你是一个有10年经验的XX行业顾问。
请根据以下数据，为客户写一份XX分析报告。
要求：
1. 包含XX/XX/XX三个板块
2. 每个板块有数据支撑+结论
3. 语言专业但客户能看懂
4. 最后给出行动建议
数据：[附上数据]"
```

**课堂实操**（30min）：
- 每人选1个工作场景 → 写3版Prompt → 对比产出质量 → 找到自己的Prompt感觉
- **费曼检验**：用自己的话说"好Prompt和坏Prompt差在哪"

## 四、模块1：数据分析（术·器）

> **目标**：能用AI从数据中挖出洞察，不再怕数据
> **行业适配示例**：经纪人→商圈成交分析 | 顾问→客户需求分析 | 设计师→用户行为分析

### 4.1 第1节：AI辅助数据分析入门（60min）

**场景导入**（5min）："你手里有一堆数据，老板/客户问'这说明什么'，你怎么30秒给出有洞察的回答？"

**操作步骤**（实操30min）：
1. 准备数据：提供样本数据集（或学员自带）
2. 写Prompt：
```
你是一位数据分析师。请分析以下数据，找出3个关键洞察和1个行动建议。

数据：
[粘贴数据]

输出格式：
- 洞察1：[发现]（数据支撑：XX）
- 洞察2：[发现]（数据支撑：XX）
- 洞献3：[发现]（数据支撑：XX）
- 建议行动：[具体可执行的建议]

注意：不要编造数据中没有的信息。
```
3. AI生成 → 人工校验 → 迭代优化

**验收标准**：
- [ ] 有3个数据支撑的洞察
- [ ] 有1个可执行建议
- [ ] 无事实错误（AI编造的数据=红线）

### 4.2 第2节：行业场景数据分析（60min）

**分行业实操**：

| 行业 | 场景 | 输入数据 | 产出物 |
|---|---|---|---|
| 房产经纪 | 商圈成交数据分析 | 成交量/均价/周期 | 商圈分析报告 |
| 咨询顾问 | 客户满意度分析 | 调研问卷数据 | 改进建议报告 |
| 教育培训 | 学员成绩分析 | 考核分数/出勤/就业 | 培训效果报告 |
| 金融保险 | 客户画像分析 | 年龄/收入/购买记录 | 精准营销方案 |
| 零售连锁 | 门店业绩对比 | 销售额/客流/转化率 | 优化建议 |

### 4.3 第3节：数据可视化（60min）

- 用AI生成图表描述 → 用工具(Excel/TRAE/Python)实现
- 产出：含3种图表的数据分析报告

## 五、模块2：文档写作（术·器）

> **目标**：用AI写专业文档，质量>自己手写，效率×5
> **生哥体悟**："内容必须言之有物，空话不是专业，有支撑才是"

### 5.1 第1节：专业报告写作（60min）

**核心方法**：AI写初稿→人工加专业判断→校验事实→定稿

**Prompt模板**（通用版，可行业定制）：
```
你是一位[行业]资深顾问。请为客户写一份[文档类型]。

背景：[客户情况/业务背景]
要求：
1. 结构：[指定章节结构]
2. 每个结论必须有依据（数据/案例/政策）
3. 语言风格：专业但客户能看懂
4. 避免空话套话，每个段落都有信息量
5. 最后给出明确的行动建议

自检清单：
- [ ] 每个结论有依据
- [ ] 无事实错误
- [ ] 客户能看懂
- [ ] 有行动建议
```

### 5.2 第2节：方案/计划书写作（60min）

**行业场景**：

| 行业 | 文档类型 | 场景 |
|---|---|---|
| 房产经纪 | 房源分析报告 | 为购房客户做房源对比 |
| 咨询顾问 | 咨询方案 | 为企业客户写改进方案 |
| 教育培训 | 课程方案 | 设计培训方案 |
| 通用 | 工作计划/项目方案 | 写项目执行计划 |

### 5.3 第3节：沟通文案写作（60min）

- 客户沟通邮件/微信话术
- **铁律**：话术必须有"为什么这样说"的原理，不教套路（风声产品理念铁律2）

## 六、模块3：汇报演示（术·器）

> **目标**：用AI做专业汇报，从"手写PPT2小时"→"AI生成10分钟+人工精修20分钟"
> **生哥实践**：科目二实操=AI生成PPT+录讲盘视频，学员评分79.33

### 6.1 第1节：AI生成PPT（60min）

**操作流程**：
1. 明确汇报目标+受众+核心信息
2. Prompt→AI生成PPT大纲
3. 大纲确认→AI生成每页内容
4. 用PPT工具(WPS/TRAE)生成幻灯片
5. 人工精修：加品牌色/图表/案例

**Prompt模板**：
```
你是一位[行业]汇报专家。请生成一份汇报PPT大纲。

汇报场景：[向谁汇报什么]
核心信息：[3-5个要点]
受众：[XX领导/客户/同事]
时长：[XX分钟]

每页要求：
- 标题简洁有力
- 内容要点化，不超过6行
- 关键数据加粗
- 每页有1个核心结论

输出格式：每页=标题+要点+结论
```

### 6.2 第2节：短视频脚本+录制（60min）

**核心公式**（从风声自媒体模块提炼）：
```
好短视频 = Hook(3秒抓人) + 核心价值(30秒干货) + CTA(5秒引导)
```

**Prompt模板**：
```
你是一位[行业]短视频策划。请写一个30-60秒短视频脚本。

目标：[获客/教育/品牌]
受众：[目标客户画像]
核心信息：[这条视频讲什么]

输出格式：
- Hook(前3秒)：[话术+画面]
- 正文(30-50秒)：[3-5个要点+话术+画面建议]
- CTA(最后5秒)：[行动引导]

注意：话术口语化，不是念稿。画面建议要可执行。
```

### 6.3 第3节：讲盘/路演实操（60min）

- AI生成PPT → 录制讲盘/路演视频 → AI评分+人工评分
- **验收**：视频时长≥3min，逻辑清晰，有数据支撑

## 七、模块4：信息检索与调研（术·器）

> **目标**：用AI做深度调研，比百度搜得深、比手动快10倍

### 7.1 第1节：AI辅助深度调研（45min）

**场景**：

| 行业 | 调研任务 | 产出 |
|---|---|---|
| 房产经纪 | 某楼盘/商圈深度调研 | 调研报告(配套+价格+竞品+趋势) |
| 咨询顾问 | 行业/竞品调研 | 竞品分析报告 |
| 教育培训 | 政策/就业市场调研 | 政策分析报告 |
| 通用 | 任何需要快速了解的领域 | 结构化调研笔记 |

**方法**：AI不是搜索引擎替代品，是"研究助手"——
1. 先让AI给框架（"调研XX应该看哪些维度"）
2. 逐维度用AI+搜索填充
3. AI汇总+人工判断
4. 产出结构化报告

### 7.2 第2节：持续学习与知识管理（45min）

- 用AI做"每日学习"：每天1个知识点→费曼笔记→积累成知识库
- Obsidian/坚果云做知识管理
- **生哥方法论**：学完用自己的话说出来，说不清=没学会

## 八、模块5：工作流自动化（术·器·进阶）

> **目标**：让AI做重复性工作，人做创造性工作
> **适用**：已有模块1-4基础的人

### 8.1 第1节：识别可自动化工作流（60min）

**方法**：
1. 列出每天/每周重复做的事
2. 标注：纯信息搬运型(可自动化) vs 需要判断型(人做)
3. 对可自动化的：设计AI辅助流程

**行业示例**：

| 行业 | 可自动化 | 人做 |
|---|---|---|
| 房产经纪 | 房源信息标准化/每日早安发/客户跟进提醒 | 讲盘/谈判/关系维护 |
| 咨询顾问 | 会议纪要/数据整理/报告格式化 | 客户沟通/方案设计 |
| 教育培训 | 作业批改(客观题)/成绩统计/通知发送 | 教学设计/学员辅导 |

### 8.2 第2节：搭建自动化工作流（60min）

- 用TRAE Work/豆包搭建1个实际工作流
- 测试→优化→验收
- 产出：可运行的自动化工作流1套

## 九、模块6：持续进化（法·道·螺旋）

> **目标**：从"学会用AI"到"越用越强"到"能教别人"
> **生哥底层使命**：实操→SOP→方法论→可复制→帮到更多人

### 9.1 第1节：个人AI方法论沉淀（45min）

**费曼学习法落地**（生哥亲述）：
```
学习循环：
  学 → 做 → 讲 → 写
  │    │    │    │
  看懂 会用 能说清 能教人
  
  每一层是下一层的前提：
  看懂≠会用（听过≠做到）
  会用≠能说清（会做≠会教）
  能说清≠能教人（自己懂≠别人能学会）
```

**产出**：个人AI使用方法论卡片（3-5张）
- 卡片1：我最擅长的AI使用场景+为什么
- 卡片2：我的Prompt心法（从踩坑中提炼）
- 卡片3：我踩过的坑+避坑指南
- 卡片4：我的AI工具组合（哪些工具做哪些事）
- 卡片5：我的下一步进化方向

### 9.2 第2节：教学相长——教别人是最好的学习（45min）

- 每人选1个自己最擅长的场景 → 做5分钟教学演示
- 听众反馈 → 迭代教学内容
- **这是费曼法的最终检验**：你能让别人也学会=你真内化了

## 十、课程运营设计

### 10.1 训练节奏

**模式A：密集集训（2天）**
```
Day1 上午：模块0(认知破冰) + 模块1(数据分析入门)
Day1 下午：模块2(文档写作) + 模块3(PPT生成)
Day2 上午：模块4(信息检索) + 模块3(短视频)
Day2 下午：模块5(自动化入门) + 模块6(方法论沉淀)
```

**模式B：持续训练（4周·每周2次·每次2节）**
```
W1：模块0 + 模块1
W2：模块2 + 模块3(PPT)
W3：模块3(视频) + 模块4
W4：模块5 + 模块6
+ 每周课后作业(用AI产出1份作品)
```

**模式C：自学+辅导（Obsidian知识库+每周1次答疑）**
```
自学者按知识库指南自学 → 遇卡点问辅导 → 每周交1份作品
```

**推荐**：模式B——不压缩到2天，给消化时间，每周产出作品→积累证据

## 十一、行业适配指南

核心方法论（道+法）跨行业通用，场景（术）和工具（器）按行业定制。

### 11.1 适配方法

| 步骤 | 做什么 | 产出 |
|---|---|---|
| 1. 场景采集 | 找3-5个该行业从业者，问"你每天最重复/最耗时/最需要专业判断的事" | 场景清单 |
| 2. 场景映射 | 把通用模块的技能映射到行业场景 | 场景-Prompt对照表 |
| 3. Prompt定制 | 通用Prompt模板+行业术语+行业数据格式 | 行业Prompt库 |
| 4. 验收标准 | 行业专家定义"什么算合格产出" | 验收清单 |
| 5. 试跑迭代 | 3人试跑→反馈→优化 | 优化版课程 |

### 11.2 已适配行业

| 行业 | 状态 | 关键场景差异 |
|---|---|---|
| **房产经纪** | ✅ 已设计 | 商圈分析/讲盘PPT/房源报告/客户匹配 |
| **咨询顾问** | 🔄 Prompt待定制 | 行业调研/方案写作/客户分析 |
| **教育培训** | 🔄 Prompt待定制 | 课程设计/学员分析/教学复盘 |
| **通用办公** | 🔄 可直接用通用版 | 报告/PPT/数据分析/自动化 |

### 11.3 面向"所有服务者"的通用版

房产经纪之外，所有服务行业的共性：
- **都要跟客户沟通**→模块2文档写作+模块3汇报演示
- **都要做专业判断**→模块1数据分析+模块4信息检索
- **都有重复工作**→模块5自动化
- **都要持续学习**→模块6方法论

差异只在：行业术语不同、数据源不同、产出格式不同。
**Prompt模板用参数化设计**：`[行业]` `[场景]` `[产出格式]` 三个变量一替换即可。
