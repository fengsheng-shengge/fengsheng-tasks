# 风声团队思维模型验收速查卡

> 生哥验收专用：下次他们交活儿，拿这个卡一下。

---

## 🔴 小眼镜（知识生产）

**硬指令：话术生产三关自审**

| 关口 | 必查项 | 验收标准 |
|------|--------|----------|
| 第一关·SCQA | 每条posSpeech/negSpeech | 有S(情境)→C(冲突)→Q(问题)→A(答案)四段结构，不能零散罗列金句 |
| 第二关·图尔敏 | 每条涉法源内容 | 有主张→依据→保证→支持→反驳→限定，缺反驳或限定的不通过 |
| 第三关·六帽 | 提交前自审 | 🤍法源在吗→🖤有红线吗→💛有价值吗→💚能更好吗→❤️直觉如何→💙走完流程了吗 |

**怎么卡**：抽查任意1条词条，让小眼镜说出SCQA拆解和六帽自审结论。说不出来=没执行。

## 🔵 小酒窝儿（品控审核）

**硬指令：六帽品控检查单逐项输出**

| 帽色 | 审核维度 | 验收标准 |
|------|----------|----------|
| 🤍白帽·事实 | 法源是否真实、条文号是否准确、发布日期是否标注 | 必须有双源，标注出处和日期 |
| 🖤黑帽·风险 | 有无合规红线、有无误导性表述 | ⚠️合规红线必须有，不能漏 |
| 💛黄帽·价值 | 对服务者有没有实际帮助、能不能直接用 | posSpeech必须能开口就说 |
| 💚绿帽·改进 | 还能怎么更好、有没有缺场景 | 至少提1条改进建议 |
| ❤️红帽·直觉 | 整体感觉对不对、有没有违和感 | 给出直觉判断，不能只说"没问题" |
| 💙蓝帽·流程 | 上面5帽都走了吗、流程走完了吗 | 必须有蓝帽总结，不能跳过 |

**怎么卡**：看小酒窝儿的审核结论，必须是6帽逐项输出，不能只给一句"审核通过"。缺帽=没执行。

## 🟢 小扣子（开发部署）

**硬指令：XY判断 + OODA循环写commit**

### XY自检（动手前）
| 项目 | 内容 |
|------|------|
| X（真正要的）| 这个修复/功能到底要解决什么问题？ |
| Y（我准备做的）| 我的方案真的能解决X吗？ |

**验收标准**：每个commit message必须包含XY判断，格式：`fix: [X=xxx] [Y=xxx]`

### OODA循环（执行中）
| 阶段 | 动作 |
|------|------|
| Observe（观察）| 当前状态是什么、报了什么错 |
| Orient（判断）| 根因是什么、有哪些方案 |
| Decide（决策）| 选哪个方案、为什么 |
| Act（行动）| 执行并验证 |

**怎么卡**：看commit message有没有XY判断。没有=没执行。复杂修复还要看OODA分析。

## 🟡 小豆子（需求PM）

**硬指令：黄金圈写需求 + 推理阶梯跨角色沟通**

### 黄金圈（写需求时）
| 层次 | 内容 | 验收标准 |
|------|------|----------|
| Why（为什么做）| 用户痛点/业务目标 | 必须说清"为谁解决什么问题" |
| How（怎么做）| 方案路径/功能设计 | 必须说清核心逻辑 |
| What（做什么）| 具体需求条目 | 必须可量化可验收 |

**验收标准**：需求文档必须从Why开始，不能一上来就列功能清单。

### 推理阶梯（跨角色讨论时）
| 阶段 | 内容 |
|------|------|
| 选择关注什么 | 我关注的是什么现象 |
| 怎么赋予意义 | 我怎么理解这个现象 |
| 假设是什么 | 我的假设是什么 |
| 所以结论是 | 所以我的结论是什么 |

**怎么卡**：看需求文档有没有Why段。和小豆子讨论时，问他"你的假设是什么"，答不上来=没执行。

---

## 🟣 小鱼儿（运营协调）

**硬指令：三层分层反馈**

| 层次 | 内容 | 验收标准 |
|------|------|----------|
| 事实层 | 发生了什么、数据是什么 | 只说客观事实，不加判断 |
| 推论层 | 根据事实能推出什么 | 必须标注"这是推论"，不能混同事实 |
| 判断层 | 我的建议是什么 | 必须和推论分开，明确说"我的判断" |

**怎么卡**：看小鱼儿的反馈/巡检报告，能不能分清"事实""推论""判断"三层。混在一起=没执行。
