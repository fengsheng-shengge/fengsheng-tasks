## 1. 技术选型

### 1.1 框架选择：uni-app

| 方案 | 优点 | 缺点 |
|------|------|------|
| 原生微信小程序 | 性能最优、审核通过率最高 | iOS/Android两套代码，无法复用 |
| Taro | React生态丰富 | 体积大，审核问题多 |
| **uni-app** ✅ | 一套代码覆盖微信+支付宝+IOS/Android+Web，审核稳定 | 需适应uni语法 |
| mpvue | 接近Vue | 已停止维护，生态弱 |

**决策**：uni-app
**理由**：现有技术栈是Vue，uni-app上手快；一套代码覆盖所有平台符合MVP快速验证目标；审核通过率稳定。

### 1.2 后端架构：Workers + D1（与主站共用）

```
┌─────────────────────────────────────────────┐
│              微信小程序 (uni-app)             │
│    风声助手（解码器/品质测评/Agent培养师）      │
└──────────────────┬────────────────────────┘
                    │ HTTPS
                    ▼
┌─────────────────────────────────────────────┐
│        Cloudflare Workers API 网关           │
│   /api/decode   /api/assess   /api/agent    │
│   /api/subscribe /api/stats   /api/event    │
│              + Auth Middleware               │
│         (JWT Token 验证/速率限制)            │
└──────────┬────────────────────────┬──────────┘
           │                        │
           ▼                        ▼
┌─────────────────┐    ┌────────────────────────┐
│  Cloudflare D1  │    │   微信支付 API          │
│   (主数据库)     │    │  (WX Pay Partner)      │
└─────────────────┘    └────────────────────────┘
```

**关键原则**：后端100%复用主站Workers API，前端只做uni-app适配。

## 2. API层设计

### 2.1 已有API（复用主站）

| 端点 | 方法 | 功能 | 小程序复用 |
|------|------|------|-----------|
| `/api/event` | POST | 事件追踪 | ✅ 直接用 |
| `/api/stats` | GET | 统计数据 | ✅ 直接用 |
| `/api/subscribe` | POST | 订阅管理 | ✅ 直接用 |
| `/api/decode` | POST | 客户解码 | ⚠️ 需确认v2接口 |

### 2.2 新增API（小程序专用）

#### 2.2.1 微信授权登录
```
POST /api/auth/wx-login
Body: { code: string }  // 微信小程序wx.login()返回的code
Response: {
  token: string,       // JWT，7天有效
  openid: string,
  userId: string
}
```

#### 2.2.2 获取用户信息
```
GET /api/user/profile
Header: Authorization: Bearer <token>
Response: {
  openid: string,
  nickname: string,
  avatar: string,
  subscribe: {
    type: 'none' | 'monthly' | 'yearly',
    expireAt: string | null
  }
}
```

#### 2.2.3 解码器v2接口
```
POST /api/decode/v2
Header: Authorization: Bearer <token>
Body: {
  input: string,      // 用户输入
  type: 'text' | 'voice'
}
Response: {
  category: string,   // 'buying' | 'selling' | 'investing' | 'living'
  profile: object,    // 客户画像
  insights: object[],  // 洞察列表
  suggestions: object[] // 建议话术
}
```

#### 2.2.4 品质测评接口
```
POST /api/assess
Header: Authorization: Bearer <token>
Body: {
  answers: object[]   // 各维度答题结果
}
Response: {
  scores: {
    safety: number,    // 安全 0-100
    health: number,    // 健康 0-100
    convenience: number, // 便利 0-100
    economy: number,   // 经济 0-100
    comfort: number,   // 舒适 0-100
    beauty: number,    // 美观 0-100
    freedom: number    // 自在 0-100
  },
  report: string,       // 报告链接
  level: string         // 'A' | 'B' | 'C'
}
```

#### 2.2.5 微信支付
```
POST /api/pay/create-order
Header: Authorization: Bearer <token>
Body: {
  productId: string,   // 'monthly' | 'yearly'
  amount: number       // 单位：分
}
Response: {
  orderId: string,
  prepayId: string,    // 统一下单prepay_id
  sign: string         // 调起支付签名
}

POST /api/pay/callback
// 微信支付回调，通知支付结果
```

### 2.3 三端共用后端策略

```
主站 (H5) ──┐
            ├──► 同一套 Workers API ──► D1 数据库
小程序 ─────┘
```

**实现方式**：
- Workers API 不区分来源，通过 `origin` header 或 `token.type` 区分调用方
- 小程序使用微信授权登录（`wx.login()`），主站使用账号密码/JWT登录
- 共用同一数据库表，`source` 字段记录来源

## 4. 微信认证策略

### 4.1 账号体系

| 阶段 | 认证方式 | 限制 |
|------|---------|------|
| MVP期（当前） | 个人主体 | 无法使用支付、无法加急审核 |
| v1.1 | 升级个体工商户 | 开通微信支付 |
| v2 | 企业主体 | 全功能 |

**当前方案**：MVP期先做功能验证，支付功能在v1.1升级后补齐。

### 4.2 微信登录流程

```
用户点击"微信登录"
    │
    ▼
wx.login() → 获取 code
    │
    ▼
POST /api/auth/wx-login { code }
    │
    ├──► 服务端向微信API换 openid + session_key
    ├──► 生成 JWT token
    └──► 返回 { token, openid }
    │
    ▼
前端存储 token，后续请求自动附加
```
