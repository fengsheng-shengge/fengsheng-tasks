# 小程序用户数据库设计

**作者**：小扣子
**日期**：2026-06-27
**版本**：v1
**用途**：P4微信小程序 + P5数据基建共用

---

## 1. 设计原则

1. **最小字段**：只存必须的用户数据，遵循GDPR最小采集原则
2. **来源区分**：微信用户 vs 主站用户通过 `source` 字段区分
3. **软删除**：不物理删除用户数据，用 `deleted_at` 标记
4. **不存微信敏感信息**：不存 session_key（微信敏感数据不落库）

---

## 2. 表设计

### 2.1 `users` — 用户主表

```sql
CREATE TABLE IF NOT EXISTS users (
  id          TEXT PRIMARY KEY,           -- 主键（wx_openid 或 email hash）
  source      TEXT NOT NULL DEFAULT 'mini_program',  -- 'mini_program' | 'web' | 'h5'
  openid      TEXT,                       -- 微信 openid（仅小程序用户）
  unionid     TEXT,                       -- 微信 unionid（跨主体时）
  nickname    TEXT,                        -- 用户昵称（微信授权获取）
  avatar      TEXT,                        -- 头像URL
  created_at  INTEGER NOT NULL DEFAULT (unixepoch()),
  updated_at  INTEGER NOT NULL DEFAULT (unixepoch()),
  deleted_at  INTEGER                      -- 软删除时间戳（NULL=正常）
);

CREATE INDEX idx_users_source ON users(source);
CREATE INDEX idx_users_openid ON users(openid) WHERE openid IS NOT NULL;
CREATE INDEX idx_users_created ON users(created_at);
```

### 2.2 `subscriptions` — 订阅表

```sql
CREATE TABLE IF NOT EXISTS subscriptions (
  id            TEXT PRIMARY KEY,
  user_id       TEXT NOT NULL REFERENCES users(id),
  product       TEXT NOT NULL,            -- 'decode' | 'assess' | 'agent'
  plan          TEXT NOT NULL,            -- 'monthly' | 'yearly'
  amount        INTEGER NOT NULL,         -- 金额（分）
  currency      TEXT NOT NULL DEFAULT 'CNY',
  status        TEXT NOT NULL,            -- 'active' | 'cancelled' | 'expired' | 'refunded'
  started_at    INTEGER NOT NULL,         -- 订阅开始时间戳
  expires_at    INTEGER NOT NULL,         -- 订阅到期时间戳
  auto_renew    INTEGER NOT NULL DEFAULT 1,  -- 1=自动续费
  wx_prepay_id  TEXT,                     -- 微信支付预付ID
  wx_order_id   TEXT,                     -- 微信支付订单号
  created_at    INTEGER NOT NULL DEFAULT (unixepoch()),
  updated_at    INTEGER NOT NULL DEFAULT (unixepoch())
);

CREATE INDEX idx_subs_user ON subscriptions(user_id);
CREATE INDEX idx_subs_status ON subscriptions(status);
CREATE INDEX idx_subs_expires ON subscriptions(expires_at);
```

### 2.3 `decode_results` — 解码结果存档

```sql
CREATE TABLE IF NOT EXISTS decode_results (
  id          TEXT PRIMARY KEY,
  user_id     TEXT NOT NULL REFERENCES users(id),
  input       TEXT NOT NULL,              -- 用户原始输入
  category    TEXT NOT NULL,              -- 解码分类
  profile     TEXT NOT NULL,              -- JSON格式客户画像
  insights    TEXT,                       -- JSON格式洞察列表
  suggestions TEXT,                        -- JSON格式建议话术
  source      TEXT NOT NULL DEFAULT 'mini_program',  -- 来源
  created_at  INTEGER NOT NULL DEFAULT (unixepoch())
);

CREATE INDEX idx_decode_user ON decode_results(user_id);
CREATE INDEX idx_decode_category ON decode_results(category);
CREATE INDEX idx_decode_created ON decode_results(created_at);
```

### 2.4 `assess_results` — 品质测评结果存档

```sql
CREATE TABLE IF NOT EXISTS assess_results (
  id            TEXT PRIMARY KEY,
  user_id       TEXT NOT NULL REFERENCES users(id),
  answers       TEXT NOT NULL,            -- JSON格式答题记录
  scores        TEXT NOT NULL,            -- JSON格式评分结果
  level         TEXT NOT NULL,            -- 'A' | 'B' | 'C'
  report_url    TEXT,                     -- 报告链接（PDF/H5）
  source        TEXT NOT NULL DEFAULT 'mini_program',
  created_at    INTEGER NOT NULL DEFAULT (unixepoch())
);

CREATE INDEX idx_assess_user ON assess_results(user_id);
CREATE INDEX idx_assess_level ON assess_results(level);
CREATE INDEX idx_assess_created ON assess_results(created_at);
```

## 4. 索引策略

| 表 | 索引 | 用途 |
|----|------|------|
| users | `idx_users_source` | 按来源统计用户量 |
| users | `idx_users_openid` | 微信登录快速查询 |
| users | `idx_users_created` | 用户增长趋势 |
| subscriptions | `idx_subs_user` | 查询用户订阅状态 |
| subscriptions | `idx_subs_expires` | 定时任务：过期提醒 |
| decode_results | `idx_decode_user` | 查询用户解码历史 |
| subscriptions | `idx_subs_status` | 统计各状态订阅数 |

---

## 5. 数据安全

| 措施 | 说明 |
|------|------|
| 软删除 | `deleted_at` 标记，不物理删除用户记录 |
| 索引条件 | `WHERE openid IS NOT NULL` 避免 null 走索引 |
| 无明文密码 | 微信 openid 为用户标识，无需密码 |
| JWT 7天 | Token 7天过期，定期刷新 |

---

## 6. 与主站D1表共用策略

**方案**：不新建D1，直接复用现有数据库

- 现有表：`events`、`feedback` — 共用 ✅
- 新增表：`users`、`subscriptions`、`decode_results`、`assess_results` — 小程序专用
- 共用方式：同一 D1 Database，通过 `source` 字段区分来源

**迁移说明**：
```sql
-- 如果将来要区分小程序DB和主站DB：
-- 1. 新建 D1 Database for Mini Program
-- 2. 将 users/subscriptions/decode_results/assess_results 迁移过去
-- 3. 旧数据 source='web'/'h5' 留在主站DB
```
