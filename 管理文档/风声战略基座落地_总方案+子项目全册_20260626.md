# 第一部分：战略基座总纲

## 一、使命·愿景·价值观

| 维度 | 内容 |
|------|------|
| 使命 | 让服务者配得上被好好对待 |
| 愿景 | 让服务者赢得消费者的物质和精神双认可 |
| 核心用户 | 服务者（消费者是服务者服务的对象，产品从服务者视角出发） |
| 北极星 | "住得好"= 物质得到保障 + 情感得到满足 + 精神得到尊重 |

### 七维度（"住得好"拆解）

| 层次 | 维度 | 含义 |
|------|------|------|
| 物质层 | 安全 | 住房结构、消防、环境安全 |
| 物质层 | 健康 | 空气、水质、采光、噪音 |
| 物质层 | 便利 | 通勤、配套、教育、医疗 |
| 物质层 | 经济 | 性价比、持有成本、增值空间 |
| 情感层 | 舒适 | 空间感、温度、收纳合理性 |
| 情感层 | 美观 | 装修风格、视觉协调、审美表达 |
| 精神层 | 自在 | 选择自由、不被将就、住得像自己 |

### 双线人生路径

| 路线 | 阶段 | 说明 |
|------|------|------|
| 购房线 | 首套→婚房→学区→改善→养老→传承 | 6阶段 |
| 租住线 | 初入职场→合租→整租→长租公寓 | 4阶段 |

> 两条线平行独立，不做强制关联。MVP阶段先聚焦购房线3阶段（首套/婚房/学区）。

### 三条产品线

| 线 | 定位 | 说明 |
|----|------|------|
| 认知线 | 说什么 | 知识底座=认知拉齐工具，信息呈现不做AI对话/判断 |

## P1 知识底座升级 + GEO化

### 基本信息

| 项 | 内容 |
|----|------|
| 项目经理 | 小豆子 |
| 技术执行 | 小扣子 |
| 产品运营 | 小眼镜（知识萃取+用户赋能） |
| 品控 | 小酒窝儿 |
| GitHub Issue | [#111](https://github.com/fengsheng-shengge/fengsheng-tasks/issues/111) |

### 战略定位

知识底座=风声认知线的地基。70%购房者先问AI，风声词条必须成为AI可采信的权威源（GEO）。这是所有产品的前置基础设施。

### 目标

从8域139条升级为GEO-ready的结构化知识图谱，成为居住领域AI采信的权威源。

### 里程碑

| # | 里程碑 | 交付物 | 截止 | 验收标准 |
|---|--------|--------|------|----------|
| 1 | 词条字段扩展方案 | 9字段→20+字段评估报告，标注必选/可选/GEO关键 | 7.3 | 每个字段有GEO价值说明+对标行业162字段 |
| 2 | 购房线3阶段样板 | 首套/婚房/学区 × 物质/情感/精神 = 9组词条全量升级 | 7.10 | 每组词条覆盖20+字段+双视角(服务者+消费者) |
| 3 | 结构化数据层 | 每条词条输出FAQ+对比表+权威引用的JSON-LD标记 | 7.17 | Google结构化数据测试工具通过+AI引擎可抓取 |
| 4 | AI提及率指标建立 | stats API新增AI提及率追踪维度+采集方案 | 7.10 | 指标可量化+采集自动化 |
| 5 | GEO化上线 | 知识底座页面可被AI引擎结构化抓取 | 7.24 | 3个以上AI引擎可返回风声词条内容 |

### 核心原则

- 知识底座=认知拉齐工具（信息呈现，不做AI对话/判断），AI交互放在作业线工具里
- 消费者版+服务者版双视角
- GEO优先：70%购房者先问AI，词条必须AI可采信
- 增量迭代：不覆盖现有8域139条，先做购房线3阶段样板验证
- 底座先扎实，产品跟着长

### 小眼镜·产品运营任务

1. 从专业资料中萃取结构化知识，补充词条内容
2. 每条词条达到GEO可采信标准：权威引用+FAQ+对比表
3. 配合小豆子做购房线3阶段×3层=9组词条的内容验证
4. 交付物：7.3首批知识萃取交付

### 风险与应对

| 风险 | 应对 |
|------|------|
| 字段扩展后内容填充量大 | 先做9组样板验证，不全量铺开 |
| GEO效果难以短期验证 | 先做结构化标记，AI提及率作为长期指标 |
| 知识萃取质量参差 | 小酒窝儿品控门禁+三方验证 |
