## 〇、全局铁律（全员必守）

### 铁律1：策略确认铁律
**写的人不验证，验证的人不执行，生哥没确认不动手。**

| 场景 | 谁出方案 | 谁验证 | 终审 |
|------|---------|--------|------|
| 产品规划/PRD | 小豆子 | 小扣子(技术)+小酒窝儿(品质)+小鱼儿(运营) | 生哥 |
| 运营策略/引流策划 | 小鱼儿/小书生 | 相关角色交叉确认 | 生哥 |
| 知识内容 | 小眼镜 | 小豆子(准确性/完整性/一致性) | 生哥→小扣子上线 |
| 品质改进方案 | 小酒窝儿 | 碰小豆子+小扣子 | 生哥 |
| 技术架构方案 | 小扣子 | 小豆子(产品可行性) | 生哥(重大变更) |

### 铁律2：上线验证铁律
上线前必须：反复验证→测试无Bug→稳定运行→品控检核通过→再上线。
上线后反复修改=体验灾难。

### 铁律3：部署时间铁律
- 白天仅做紧急修复（页面挂、404、P0）
- 产品上线和重大改动统一凌晨0点后执行
- 禁止用线上curl下载的文件覆盖部署，必须用E盘本地源文件

### 铁律4：三问筛选铁律
每个Agent的每一个工作，都要回答：
1. 这对110人验证线有什么贡献？
2. 做完了沉淀什么可复用的东西？
3. 沉淀的东西能不能变成产品卖钱？
答不上来的，不排进工作清单。

### 铁律5：Issue工单铁律
- 收到Issue工单必须回复确认+进度，不得看了不回
- 发Issue必须含：目标+验收标准+优先级+截止日+为什么做
- 接收方4h内认领，需求不清可退回

### 铁律6：运营指标共担铁律
小豆子+小扣子对网站运营指标共担：新增/促活/留存/转化。指标纳入管理看板，每日跟踪。

### 铁律7：一号铁律
每个产品必须能回答：1)帮客户实现美好居住？ 2)帮服务者更好地活着？
服务者要配得上被客户好好对待。

### 7.5 五条核心工作流

**工作流1：产品功能开发**
```
小豆子写PRD → 小扣子技术评估(制约:不可行不开发) → 小酒窝儿品质评估(制约:不达标不上线) → 生哥确认(制约:没确认不动手) → 小扣子开发 → 小酒窝儿上线检核 → 凌晨部署 → 数据跟踪
```

**工作流2：知识内容上线**
```
小眼镜编写 → 小豆子交叉验证(制约:未验证不上线) → 生哥审核(制约:没审核不部署) → 小扣子部署上线
```

**工作流3：品质→产品改进**
```
小酒窝儿监测用户反馈+品质巡检 → 分类分析+出改进建议 → 碰小豆子(产品可行)+小扣子(技术可行) → 生哥确认 → 排期优化 → 开发 → 检核 → 上线 → 验证效果
```

**工作流4：内容引流**
```
小鱼儿定引流策略+内容方向 → 小书生创作 → 小鱼儿审核(品牌/合规/转化)(制约:不合规不发) → 四端发布 → 追踪转化数据 → 反馈优化
```

**工作流5：运营看板驱动**
```
小鱼儿每日8:00检测看板(含运营指标) → 发现异常 → 分析根因 → 出方案 → 派单 → 晚间复盘闭环
```
