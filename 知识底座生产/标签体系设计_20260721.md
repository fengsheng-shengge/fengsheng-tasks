# #186 词条标签体系设计文档

> 版本：v1.0 | 2026-07-21
> 交付日期：2026-07-27
> 关联：#185 小豆子产品重设（场景驱动+道法术器四层交付）
> 目标：5800词条按「客户类型×业务阶段×道法术器层次」三维打标签，使工具能精准调用

---

## 一、设计原则

1. **词条不直接展示给用户**——由工具按场景自动调用
2. **只给当前场景相关的词条**——不是500条全堆
3. **标签是调用接口不是展示分类**——机器读的，不是人看的
4. **一个词条可打多个标签**——跨场景复用是常态
5. **标签粒度够用就行**——不是越细越好，够精准调取即可

---

## 二、三维标签体系

### 维度1：客户类型（clientType）

| 标签值 | 含义 | 映射逻辑 |
|--------|------|---------|
| `buyer` | 买房客户 | consumerQ对应买房场景的问题 |
| `renter` | 租房客户 | consumerQ对应租房场景的问题 |
| `seller` | 出售业主 | ownerQ对应卖房场景的问题 / ownerBenefit有值且关联出售 |
| `landlord` | 出租业主 | ownerQ对应出租场景的问题 / ownerBenefit有值且关联出租 |

**映射规则：**
- 所有词条默认包含 `buyer`（风声P0聚焦经纪人服务买房客户）
- 有 `ownerQ` 或 `ownerBenefit` 的词条额外打 `seller` 或 `landlord`
- LIV域词条同时打 `buyer` + `renter`（居住问题买房租房都涉及）
- OWN域词条按 subScene 细分：OWN-SEL→seller, OWN-REN→landlord, OWN-OPS→seller+landlord
- EXT域词条：EXT-SELL→seller, EXT-RENT→landlord
- CAR/CRO域词条：buyer+renter+seller+landlord（职业成长和通用能力服务全角色）

### 维度2：业务阶段（stage）

| 标签值 | 含义 | 对标贝壳5阶段 | 映射域 |
|--------|------|-------------|--------|
| `pre` | 售前（获客/匹配/决策前） | 前期获客+看房阶段 | PRE, EXT-SELL/RENT的前期, CAR的部分 |
| `mid` | 售中（签约/交易执行） | 谈判与签约+交易执行 | SNG, PST的部分, EXT-SELL/RENT的执行期 |
| `post` | 售后（交付/居住/长期维护） | 售后维护+居住 | PST-DELV/PST-RGHT, LIV, CAR的部分 |

**映射规则（按subScene细化）：**

| subScene前缀 | stage标签 |
|-------------|----------|
| PRE-* | `pre` |
| SNG-CONT, SNG-DEPO, SNG-DUAL, SNG-FUND | `mid` |
| SNG-NET, SNG-PAY | `mid` |
| PST-DELV, PST-RGHT | `post` |
| PST-TITL, PST-TAX, PST-LOAN, PST-INSR | `mid`（过户/缴税/贷款/保险属于交易执行中） |
| LIV-* | `post` |
| EXT-SELL | `pre`（出售决策期） + `mid`（执行期） |
| EXT-RENT | `pre`（退租决策期） |
| EXT-EXCH | `pre` + `mid` |
| EXT-MOVE | `post` |
| OWN-SEL, OWN-REN | `pre`（业主获客期） |
| OWN-OPS, OWN-PRP | `post`（运营维护期） |
| OWN-POL, OWN-DSP | `mid`（纠纷处理期） |
| CAR-EXAM, CAR-SKILL | `pre`（能力储备期） |
| CAR-COMP, CAR-GROW | `pre` + `post`（竞争力/成长贯穿全周期） |
| CRO-GEN | `pre` + `mid` + `post`（通用能力贯穿） |

### 维度3：道法术器层次（layer）

| 标签值 | 含义 | 映射entryType | 典型内容 |
|--------|------|-------------|---------|
| `dao` | 道——底层原理、法律原理、行业本质 | LAW, POL | 法条解读、政策原理、行业底层逻辑 |
| `fa` | 法——方法论、流程规范、标准框架 | PROC, STD | 操作流程、服务标准、评估框架 |
| `shu` | 术——实操技巧、话术、应对策略 | TERM, RISK | 沟通技巧、风险应对、谈判策略 |
| `qi` | 器——工具、模板、数据、案例 | CASE, 有toolType的 | 实用工具、计算模板、判例佐证 |

**映射规则：**
- LAW → `dao`
- POL → `dao`（政策也是原理层）
- PROC → `fa`
- STD → `fa`（标准也是方法论层）
- TERM → `shu`
- RISK → `shu`（风险识别是术的层面）
- CASE → `qi`（判例是工具/佐证）
- 有 `toolType` 字段的词条额外打 `qi`（工具型词条）

## 三、标签数据结构

每个词条新增 `tags` 字段：

```json
{
  "id": "PRE-NEED-001",
  "name": "首套房首付要多少",
  "tags": {
    "clientType": ["buyer"],
    "stage": ["pre"],
    "layer": ["fa"]
  }
}
```

### 标签组合示例

| 词条ID | name | clientType | stage | layer |
|--------|------|-----------|-------|-------|
| PRE-NEED-001 | 首套房首付要多少 | buyer | pre | fa |
| SNG-CONT-001 | 定金和订金有什么区别 | buyer, seller | mid | dao |
| LIV-PROP-001 | 物业费不交会怎样 | buyer, renter | post | shu |
| OWN-REN-001 | 租客拖欠租金怎么办 | landlord | post | shu |
| CAR-SKILL-001 | 客户说再考虑怎么办 | buyer, renter, seller, landlord | pre | shu |
| PST-TITL-001 | 过户需要哪些材料 | buyer, seller | mid | fa |

### 规则3：基于特征字段的增强标签

- 有 `ownerQ` 字段且非空 → clientType 补充对应业主类型
- 有 `toolType` 字段且非空 → layer 补充 `qi`
- severity=hard 且 legalRef 非空 → layer 补充 `dao`（深层法律原理）
- `consumerQ` 含"租" → clientType 补充 `renter`

## 五、MVP优先打标策略

对齐#185产品重设：**MVP = 买房客户 × 3个P0场景**

### MVP范围（优先打标）
1. **客户解码**（pre阶段 buyer）→ PRE-NEED, PRE-QUAL 全部词条
2. **带看服务**（pre阶段 buyer）→ PRE-MATCH, PRE-PRICE, PRE-SCH 全部词条
3. **谈判斡旋**（mid阶段 buyer+seller）→ SNG-CONT, SNG-DEPO, SNG-DUAL 全部词条

### 打标优先级
| 优先级 | 范围 | 词条数（估计） | 完成时间 |
|--------|------|-------------|---------|
| P0-MVP | 买房客户×3场景 | ~350条 | 7.24 |
| P1-扩展 | 全部buyer+seller | ~2500条 | 7.26 |
| P2-补全 | renter+landlord+通用 | ~3300条 | 7.27 |

## 六、对齐#185的调用逻辑

小豆子V2产品重设后，词条调用路径：

```
用户输入（语音/文字）
  → 场景识别（感知层）
    → 匹配 clientType + stage
      → 按layer排序：dao→fa→shu→qi
        → 返回 top-N 词条
          → 工具卡渲染展示
```

**示例调用：**
- 经纪人说"客户问首付" → clientType=buyer, stage=pre → 返回 PRE-NEED 中 layer=fa+shu 的词条
- 经纪人说"业主问定价" → clientType=seller, stage=pre → 返回 OWN-SEL 中 layer=dao+fa 的词条
- 经纪人说"过户出问题" → clientType=buyer+seller, stage=mid → 返回 PST-TITL 中 layer=fa+shu 的词条

## 八、验收标准

1. 5800条词条 100% 有 tags 字段
2. tags 字段包含 clientType + stage + layer 三个子字段
3. 每个字段至少1个标签值

5. 跨域词条（CRO/CAR）clientType 包含全部4种客户类型
