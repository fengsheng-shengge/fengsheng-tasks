# 客户解码 · 测学练考评体系 · 测/学侧框架 v1.0

> **定位**：课程侧完整框架，供产品侧对齐功能边界和上线节奏
> **底层逻辑**：AI融入业务场景和交互过程，不单独教AI
> **三阶递进**：切入期→习惯期→伙伴期
> **双线覆盖**：交易线（A1经纪人）+ 租赁线（B4客户经理）
> **版本**：v1.0 | 2026.06.24 | 小眼镜·课程侧

---

## 一、整体架构

```
用户进入 fengsheng.tech
    → 身份选择（经纪人 / 客户经理）
    → 入学诊断（测）—— 5题3分钟 → 档位+薄弱维度+学习路径
    → 微课学习（学）—— 3节×30min → 每节有实操+产品衔接
    → 解码器实操（练）—— 真实客户场景 → 解码报告 → 话术输出
    → 阶段测评（考）—— 情景题+实操评分 → 档位更新
    → 成长档案（评）—— 学习曲线+档位变化+真实应用记录
```

**测/学侧先行，练/考/评侧等产品功能边界确认后对齐。**

## 二、测：入学诊断

### 2.1 诊断逻辑

| 项目 | 说明 |
|------|------|
| 目标 | 测"你读懂客户的能力在哪个档"，不是测AI能力 |
| 时长 | 3分钟（5题快筛） |
| 输入 | 身份选择 + 5道情景判断题 |
| 输出 | 客户洞察档位（L0/L1/L2）+ 薄弱维度 + 推荐学习路径 |
| 底层 | 人才字典A1/A2/B1/B4"客户洞察"相关技能点 |

### 2.2 四大诊断维度

| 维度 | 测什么 | 交易线重点 | 租赁线重点 |
|------|--------|-----------|-----------|
| **D1 客户识别** | 快速判断客户类型和真实需求 | 购房动机（刚需/改善/投资）、家庭阶段 | 租客画像（通勤/学区/性价比）、租期预判 |
| **D2 需求挖掘** | 从模糊表达中挖出真实诉求 | 预算弹性、决策权判断、隐性需求 | 合租/整租偏好、特殊需求、续租意愿 |
| **D3 信任建立** | 首次接触建立专业信任 | 风险主动提醒、不推销先诊断、价格透明 | 费用说明透明、维修响应承诺、退租流程提前说 |
| **D4 场景应对** | 典型业务场景正确判断 | 犹豫/比价/截胡/家属意见不一 | 押金争议/拖欠/涨租/提前退租 |

### 2.3 档位定义

| 档位 | 特征 | 学完应达到 |
|------|------|-----------|
| **L0 本能型** | 凭感觉做事，没有方法论 | → L1 |
| **L1 标准型** | 能按流程做，但不灵活 | → L2 |
| **L2 专业型** | 能独立判断，会应变 | → 巩固+进练 |
| **L3 专家型** | 能教别人，能创新 | MVP暂不覆盖 |

### 2.4 MVP题库（5题快筛）

**通用题2题**（双线共用）：
- Q1 [D1客户识别]：客户说"找个不错的房子"，最佳第一反应？
  - ✓ 先诊断再推荐（3分）| △ 只问硬条件/二选一（2分）| ✗ 上来就推（1分）
- Q2 [D3信任建立]：客户嫌价格高，怎么回应最建信任？
  - ✓ 承认感受+给数据+给方案（3分）| △ 有服务意识没依据（2分）| ✗ 否定感受/交定价权（1分）

**交易线3题**（A1经纪人）：
- Q3 [D2需求挖掘]：夫妻看房关注点不同，怎么判断？
  - ✓ 识别双人决策不同关注点（3分）| △ 有拆解但优先级可能错（2分）| ✗ 只看表面/粗暴试探（1分）
- Q4 [D4场景应对]：发现房屋有抵押未解清，怎么做？
  - ✓ 主动提醒风险+安全路径（3分）| △ 方向对但信息不完整（2分）| ✗ **零容忍**：隐瞒风险直接判L0
- Q5 [D1+D2综合]：代际购房看了一个月没定，真实情况？
  - ✓ 三重确认叠加（支付者≠决策者≠使用者）（3分）| △ 部分正确（2分）| ✗ 简单归因（1分）

**租赁线3题**（B4客户经理）：
- Q3' [D1客户识别]：低预算+地铁附近，第一反应？
  - ✓ 预算约束下先精准定位筛选条件（3分）| △ 细化了交通但忽略其他维度（2分）| ✗ 直接匹配/否定预期（1分）
- Q4' [D4场景应对]：退租押金争议，怎么处理？
  - ✓ 合同+证据+中立（3分）| △ 有调解意识缺依据（2分）| ✗ 推卸/选边站（1分）
- Q5' [D2需求挖掘]：优质租客续租要求降价，怎么判断？
  - ✓ 用数据帮房东算账（降vs空置哪个亏更多）（3分）| △ 没查数据直接判断（2分）| ✗ 传话筒/忽略空置成本（1分）

### 2.5 评分与输出

**快筛评分**：

| 总分 | 档位 | 推送策略 |
|------|------|---------|
| 13-15 | L2 专业型 | 直接进「练」——你有方法论，实战巩固 |
| 9-12 | L1 标准型 | 推微课2+3——会做但不灵活，补方法论+工具 |
| 5-8 | L0 本能型 | 从微课1开始——需要建立方法论底子 |

**薄弱维度输出**：每维度单独计分，低于2分标记"需强化"，学习路径优先推送对应模块。

**零容忍项**：隐瞒风险（Q4选A/D）→ 直接判L0，无论总分多少。

### 2.6 技术落地需求（产品侧对齐）

| # | 需求 | 课程侧要求 | 产品侧待定 |
|---|------|-----------|-----------|
| T1 | 题库存储 | 10题（5通用+5双线分化），支持后续扩到40题 | 存在哪？后台/独立DB？ |
| T2 | 出题逻辑 | MVP固定5题；v2自适应（根据前3题正确率调整后2题难度） | 固定 or 自适应？ |
| T3 | 评分计算 | 前端即可（简单加权）；v2后端算（自适应逻辑复杂） | 前端 or 后端？ |
| T4 | 档位展示 | 页面展示档位+维度雷达图；或Bot对话推送 | 页面 or Bot？ |
| T5 | 与解码器关系 | 诊断后推荐进入解码器试用，形成测→学→练闭环 | 独立 or 联动？ |
| T6 | 身份切换 | 经纪人/客户经理决定后3题走哪条线 | 一套题库双线分化 |

## 三、学：3节微课

### 3.1 课程设计底层

```
课程主线 = 业务流程（读客户 → 解码客户 → 用解码结果服务客户）
AI = 交互过程中的自然能力（嵌入每个环节的提效工具）
解码器 = 业务流程中的一个环节（不是"学工具"，是"做业务时顺手用"）
每节课 = 想明白时刻(2min) + 方法论(10min) + 实操(15min) + 课后(3min) = 30min
```

**课程风格**：借鉴得到/罗振宇/刘润——知识转译（复杂东西大白话讲透）+ 底层逻辑（案例→模型→公式）+ 金句驱动认知升级

### 3.2 第1课：客户到底在说什么？

| 项目 | 内容 |
|------|------|
| **学习目标** | 能从客户的3句话里判断出他真正要什么 |
| **想明白时刻** | "经纪人老张带看20次才成交，小李3次就签了。区别不是小李嘴甜，是小李会'读人'。" |
| **核心模型** | **客户解码四层模型** |
| **验收标准** | L1能说出四层分别是什么；L2能对一段真实客户对话做四层拆解 |

**客户解码四层模型**：

| 层级 | 含义 | 示例 |
|------|------|------|
| 第1层·表面需求 | 他说出来的 | "我要个两居室" |
| 第2层·真实需求 | 他真正在意的 | "我要的是孩子有独立房间" |
| 第3层·隐性需求 | 他没说但很重要 | "老婆刚怀二胎，不想让老人来带时太挤" |
| 第4层·决策逻辑 | 他怎么下决心 | "价格、学区、老婆意见，三关都过才签" |

**金句**：**"客户说的不是需求，是他以为的需求。你的工作是找到他不知道自己知道的那个需求。"**

**实操环节**（15min）：
- 场景A：用解码器输入一段客户对话 → AI拆出四层需求 → 学员对比自己的判断
- 场景B：自己写一段客户对话 → 用四层模型拆 → 课后作业模板输出
- **AI嵌入**：解码器的需求拆解 = 四层模型的放大镜，不是"学用工具"，是"读客户时有个更快帮手"

**课后作业**：下次接待客户，记录他说的话，用四层模型拆一遍。交作业=上传你的拆解结果。

### 3.3 第2课：3秒读懂客户——解码报告实战

| 项目 | 内容 |
|------|------|
| **学习目标** | 能用解码器3秒出客户解码报告，基于报告制定服务策略 |
| **想明白时刻** | "同一个客户，A说'预算有限偏刚需'，B说'他是帮孩子买的改善型，决策权在孩子'。谁签了？B。因为B读对了。" |
| **核心模型** | **客户分类解码表 + 解码报告三要素** |
| **验收标准** | L1能读懂解码报告；L2能基于报告制定差异化服务策略 |

**客户分类解码表**（双线分化）：

| 维度 | 交易线（A1） | 租赁线（B4） |
|------|-------------|-------------|
| 决策类型 | 独立/家庭/代际决策 | 个人/合租/企业决策 |
| 时间敏感度 | 紧急（置换）/不急（观望） | 即住/月内/季内 |
| 价格弹性 | 刚需低弹性/改善有弹性/投资看ROI | 预算刚性/品质溢价/合租分摊 |
| 信息需求 | 学区/税费/贷款/升值空间 | 通勤/配套/合租规则/退租条件 |
| 风险关注 | 查封/抵押/学区政策变化 | 押金安全/房东靠谱度/合同陷阱 |

**解码报告三要素**：
1. **客户画像**：什么人 + 什么阶段 + 什么驱动
2. **需求优先级**：P0必须满足 / P1最好有 / P2加分项
3. **服务策略**：推荐什么 + 怎么说 + 先解决什么顾虑

**金句**：**"3秒不是AI快，是你平时30分钟做的事，现在有人帮你3秒出初稿。你的专业判断才是终稿。"**

**实操环节**（15min）：
- 场景A：用demo客户数据 → 解码器一键出报告 → 验证报告准不准
- 场景B：拿到报告 → 制定服务策略（推荐哪套+先说什么话+预判什么顾虑）
- 场景C（租赁线）：租客"预算2000地铁附近" → 报告告诉你真实需求
- **AI嵌入**：解码器出报告 = 客户分析过程被加速；你判断准不准 = 专业能力在升级

**课后作业**：用解码器做一个真实客户的解码报告，写下你的服务策略。

### 3.4 第3课：从解码到成交——让每个客户都感觉"你真懂我"

| 项目 | 内容 |
|------|------|
| **学习目标** | 能把解码报告转化成客户愿意买单的服务方案，实现读客户→服务客户闭环 |
| **想明白时刻** | "解码报告不是给自己看的，是帮客户'被读懂'。客户说'你真懂我'的那一刻，成交就是时间问题。" |
| **核心模型** | **四步闭环 + 信任飞轮** |
| **验收标准** | L1能说出四步闭环；L2能完成从解码报告到服务方案的完整输出 |

**从解码到成交四步闭环**：
```
解码报告（知道客户要什么）
    ↓
匹配方案（推荐+为什么推荐这套）
    ↓
预判顾虑（客户还没问你就先回答了）
    ↓
促成决策（不是催签，是帮客户下决心）
```

**"你真懂我"的三个对比**：

| 场景 | 普通经纪人 | 懂客户的经纪人 |
|------|-----------|--------------|
| "再看看" | "好的您再考虑" | "您是不是对XX还有顾虑？我帮您确认一下" |
| 嫌贵 | "这个价真不贵" | "理解，咱们看看同小区成交价对比，您心里就有数了" |
| 被截胡 | "那套确实不错" | "您要是喜欢那套，我帮您查查有没有抵押查封风险" |

**信任飞轮**：
```
读对客户 → 说对话 → 解决真问题 → 客户信任你
    ↑                                    ↓
    ← 更多转介绍 ← 更多成交 ← 客户觉得你靠谱
```

**金句**：**"成交不是说服，是帮客户做他本就想做的决定。解码报告告诉你他在犹豫什么，你帮他跨过去。"**

**实操环节**（15min）：
- 场景A：基于第2课的解码报告 → 写一段服务话术（理解客户后的自然表达，不是模板背诵）
- 场景B：模拟客户3个常见顾虑 → 用解码报告数据逐一回应
- 场景C（租赁线）：续租谈判场景 → 用数据帮房东算账 → 从"降不降"变成"降vs空置哪个亏"
- **AI嵌入**：解码报告→话术建议 = AI帮你把"读到的"变成"说出来的"，不是替你说，是帮你组织好

**课后作业**：下一次真实客户服务，用解码报告+四步闭环走一遍，记录结果。

### 3.5 三阶递进映射

| 课程 | 对应阶段 | 学员体感 |
|------|---------|---------|
| 第1课 | **切入期** | "原来客户说的和要的不一样，解码器帮我看到了" |
| 第2课 | **切入期→习惯期** | "用解码器3秒出报告，比我自己分析快10倍，每个新客我都用" |
| 第3课 | **习惯期→伙伴期** | "AI不只帮我出报告，还提醒我该跟进什么，像多了个搭档" |

## 四、测→学→产品闭环设计

### 4.1 闭环流程

```
入学诊断（测）
    → 输出档位+薄弱维度
    → 推荐学习路径（从哪节课开始）
    → 完成微课（学）
    → 进入解码器实操（练）—— 测中薄弱的维度，练中强化
    → 阶段测评（考）—— 重测诊断维度，看档位是否升级
    → 成长档案（评）—— 可视化学习曲线
```

### 4.2 与产品功能映射

| 课程侧模块 | 产品侧功能 | 映射关系 |
|-----------|-----------|---------|
| 入学诊断5题 | 测评入口 | 诊断后推荐微课+解码器试用 |
| 微课3节 | 学习模块（视频/图文） | 每节课嵌入解码器实操环节 |
| 课后作业 | 作业提交+AI评分 | 学员上传拆解结果/解码报告 |
| 四层模型 | 解码器需求拆解功能 | 方法论→产品功能的直接映射 |
| 解码报告 | 解码器输出 | 第2课学读报告，第3课学用报告 |
| 四步闭环 | 解码器→话术建议 | 解码结果→服务策略→话术输出 |
| 档位升级 | 成长档案 | 测/考更新档位，可视化进步 |

### 4.3 与已确认产品方案对接

基于#97生哥6.24确认的方案：

| 产品方案要素 | 测/学侧对接 |
|------------|-----------|
| Prompt模板集v2（5种客户类型×2场景） | 微课第2课的5种客户类型 = 5种Prompt模板的教学入口 |
| 多版本定价（¥29/59/199/月） | 测+3节微课 = 免费体验层；完整课程+实操 = 付费层 |
| POC门店验证 | 微课实操环节 = POC验证的学员端测试场景 |
| 微信小程序 | 测/学入口放小程序首页，3分钟完成诊断→推荐课程 |

## 五、MVP上线节奏（6.30 vs 7.15）

### 方案A：6.30最小可行版

| 模块 | 范围 | 说明 |
|------|------|------|
| 测 | 5题快筛（双线切换） | 题库固定，前端评分，页面展示档位 |
| 学 | 3节微课图文版 | 非视频，md/html格式，内嵌解码器入口 |
| 练 | 解码器试用（1个demo场景） | 交易线1个+租赁线1个 |
| 考 | 暂不上 | 用测的5题复用 |
| 评 | 档位展示 | 单次结果，无历史曲线 |

**工作量**：课程侧2天+产品侧3天，6.28可联调。

### 方案B：7.15完整版

| 模块 | 范围 | 说明 |
|------|------|------|
| 测 | 20题完整诊断（自适应出题） | 每维度5题，3级难度 |
| 学 | 3节微课视频版+完整教案 | 视频+PPT+课后作业模板 |
| 练 | 解码器全场景实操 | 5种客户类型×2场景 |
| 考 | 阶段测评+档位升级 | 情景题+实操评分 |
| 评 | 成长档案+学习曲线 | 历史档位变化+应用案例记录 |

**工作量**：课程侧1周+产品侧2周。

### 建议

**6.30上线方案A，7.15迭代到方案B。** 理由：
1. 方案A已验证核心闭环（测→学→练），用户3分钟可完成诊断+开始学习
2. 生哥已确认POC验证，方案A足够支撑门店测试
3. 方案B的20题+视频+完整考评，需产品功能全部就位

## 七、待产品侧对齐的5个关键问题

1. **解码器功能边界**：输入什么（客户对话文本？结构化信息？）→ 输出什么（四层拆解？解码报告？话术建议？）→ 决定微课实操环节怎么设计
2. **测评入口怎么放**：小程序首页独立入口 vs 解码器页面内嵌 vs Bot对话触发 → 决定用户体验流程
3. **交易vs租赁的解码差异**：共用一个解码器+切换视角 vs 两个独立解码器 → 决定双线课程分化粒度
4. **MVP上线节奏**：6.30方案A（5题+图文微课+1个demo场景）还是7.15方案B（完整版）→ 决定开发排期
5. **数据回流闭环**：学员用解码器的数据能不能回流到学习档案 → 决定评模块能不能做自动化
