## 想明白时刻

### 场景一：90%的人用AI的方式是错的

你有没有这种感觉：
- 问AI一个问题，它回答了，你发现不是你要的
- 再问一次，换个问法，它又给你另一堆东西
- 最后你花了20分钟调教，AI才给你一个勉强能用的东西

**问题不在AI，在你用的方式。**

你把AI当"问答机"——你问，它答，答完结束。但你的工作不是问答，是**干活**。问答机解决不了干活的问题。

**碳基思考：你想让AI怎么"替你干活"？**

> 不是"你问AI答"——你是发动机，推着它走。
> 是"AI按你的方式主动干"——AI是发动机，你是验收方。

## 方法论讲解

### 核心方法：被动应答 → 自主闭环

**被动应答模式：**
```
你（人）→ 问问题 → AI → 给答案 → 你（人）→ 再问 → AI → 再答 → ...
```
- 每次你都要想"怎么问"
- AI每次都从头开始，不记得你上一步
- 你是发动机，AI是零件

**自主闭环模式：**
```
你：一次性给完所有信息 + 明确验收标准
    ↓
AI：自动执行 → 产出结果
    ↓
你：只验证 → 通过 or 打回重跑
```
- 你告诉它要什么结果，它自己跑完
- 你只负责检查对不对
- AI从零件变成发动机

### 碳基做什么：想明白你要什么结果

**闭环任务4要素：**

| 要素 | 你要回答 |
|------|---------|
| **输入要素** | 这个任务需要什么信息？一次性能给完吗？ |
| **处理规则** | AI知道"怎么处理"吗？规则能写出来吗？ |
| **产出格式** | 你要的是文档？表格？清单？格式能提前说清楚吗？ |
| **验收标准** | 什么是"完成了"？什么情况要打回重跑？ |

**能闭环的任务特征：** 输入→处理→输出都能提前说清楚，不需要边做边问。

**不适合闭环的任务：** 需要外部信息、需要灵活判断、需要边做边调整。

### 硅基做什么：一次性说清楚，不让AI再问

**格式要求三层次：**

| 层次 | 你说 | AI给 | 风险 |
|------|------|------|------|
| **基础层** | "给我一个方案" | AI觉得什么方案都行 | 你要的可能没有 |
| **进阶层** | "给我一个方案，要有背景分析+3个建议" | 有框架但细节不一定对 | 大方向对但细节要改 |
| **完整层** | "给我一个方案..."+格式+禁止 | 你能直接用 | 需要多次调教 |

### 识别你的学习风格

**三种学习类型：**

| 类型 | 特征 | 你习惯的信息格式 |
|------|------|----------------|
| **文档型** | 喜欢看文字、读案例 | 长文、报告、说明文档 |
| **动手型** | 喜欢看示例、跟着做 | 清单、步骤、模板 |
| **对话型** | 喜欢聊、问一句答一句 | 问答、讨论、启发式 |

**自测3问：**

1. 学新技能第一反应？A.找文档看 → 文档型  B.找视频跟着做 → 动手型  C.找懂的人问 → 对话型
2. 收方案希望什么形式？A.Word文档 → 文档型  B.Excel清单 → 动手型  C.PPT边讲边演示 → 对话型
3. AI给方案第一反应？A."写得很详细，再看看" → 文档型  B."步骤很清楚，试试" → 动手型  C."等等，我有几个问题" → 对话型

## 实操演练

### 练习1：识别你的学习风格（5分钟）

回答上面3个问题，确定你是文档型/动手型/对话型/混合型。

把结果更新到AI设定文档：

```markdown
## 沟通偏好
- 我的学习类型：[文档型/动手型/对话型/混合型]
- 我习惯的信息格式：[描述你喜欢的格式]
- 我不喜欢的格式：[描述你会扔掉重来的格式]
```

### 练习2：设计你的第一个闭环任务（25分钟）

```markdown
【任务名称】：____________________

【任务目标】：____________________

【输入要素】：
- [要素1]：[具体内容或标注"待补"]
- [要素2]：[具体内容或标注"待补"]
- [要素3]：[具体内容或标注"待补"]

【处理规则】：
1. [规则1]
2. [规则2]
3. [规则3]

【产出格式】：
- 格式类型：[文档/表格/清单/PPT/其他]
- 必须包含：[列1、列2...]
- 不包含：[明确排除的内容]
- 禁止出现：[明确不要的内容]

【验收标准】：
- [√] 完成了：[什么情况算完成]
- [×] 打回：[什么情况下要重跑]

【执行记录】：
- 第一次执行：____年__月__日
- 执行结果：□通过  □打回（原因：__________）
- 第二次执行：____年__月__日
- 执行结果：□通过  □打回（原因：__________）
- 最终结果：□通过（闭环完成）  □未通过（卡点：__________）
```

**举例：装修方案任务**

```
【任务名称】：批量装修方案生成

【任务目标】：一次性生成3个装修方案，业主可以直接选

【输入要素】：
- 户型：[80平/两室一厅]
- 预算：[15万]
- 家庭结构：[中年夫妻，无孩]

【处理规则】：
1. 方案必须控制在15万内
2. 每个方案列出主材品牌和价格
3. 给出3个备选方案对比表

【产出格式】：
- 表格输出，列：项目/品牌/型号/单价/数量/合计
- 不确定的地方标注[需确认]
- 禁止出现：超过15万的内容

【验收标准】：
- [√] 3个方案都在预算内
- [√] 表格完整，有品牌有价格
- [×] 有未标注的猜测内容就打回
```

**验收：** 任务设计完整，能跑通（有输入→有处理→有输出→有验证）。

### 练习3：跑通并验证（20分钟）

用你设计的闭环任务，让AI执行：
1. 一次性给完所有信息+格式要求
2. 看AI产出是否符合验收标准
3. 通过 or 打回记录原因
4. 重复直到闭环完成

**验收：** 闭环任务完整跑通（最多3次调整），产出物格式匹配你的学习方式。

## 常见问题

### Q1：什么任务适合闭环？什么不适合？

**适合闭环：**
- 有明确输入的任务（客户信息、产品参数、日期数量）
- 有标准处理规则的任务（报价、方案生成、报告撰写）
- 有固定产出格式的任务（表格、清单、模板）

**不适合闭环：**
- 需要外部信息才能开始（外部审批、数据查询）
- 需要灵活判断（战略决策、创意发散）
- 第一次探索（你自己都不知道要什么）

### Q2：闭环任务跑一半要调整怎么办？

**正常流程：**
1. AI输出 → 你发现问题 → 打回标注 → AI重跑 → 再验证
2. 每次打回要记录："哪不对，为什么，应该是什么样"
3. 3次以内跑通 → 闭环成功

**超过3次：** 说明任务设计有问题，重新梳理输入要素和处理规则。

### Q3：AI跑完结果我一看就扔，怎么让它一次到位？

**加上这三句：**
- "我要的是[X]，不是[Y]"（明确不要什么）
- "重点是[X]，其他可以简略"（明确重点）
- "如果有不确定的，标注[待确认]，不要瞎编"（诚实标注）
