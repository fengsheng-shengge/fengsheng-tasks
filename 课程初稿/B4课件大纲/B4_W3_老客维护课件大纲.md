## 二、课程导入（10分钟）

### 2.1 想明白时刻：老客不是"维护"出来的，是"经营"出来的

**引导话术：**

> 你有没有遇到过这种情况：
>
> 客户租了房，交了钱，然后……就没然后了。你忙新客户，顾不上老客户。等客户要续租或转介绍的时候，才发现"哦，这个人我好像很久没联系了"。
>
> 更扎心的是——老客户退租的时候，你才发现空窗期要赔多少。一套房空置1个月，损失的就是1整月的租金。
>
> 碳基思考：**老客不是"维护"出来的，是"经营"出来的。** 维护是被动的，经营是主动的。你要在客户还没开口之前，就帮他想到下一步。
>
> 硅基帮你做的是：记住每个客户的关键节点，在你忙的时候替你"在场"——到期提醒、续租预警、转介绍时机，一个都不漏。

**互动提问：**
1. 你的老客户中，有多少人主动给你转介绍了？（0-3个为正常，说明经营不够）
2. 你的老客户中，有多少人到期后选择了别人？（流失率是多少？）
3. 你有没有因为"忘了跟进"而丢过续租或转介绍？

**数据认知：**

| 数据 | 含义 | 对客户经理的启示 |
|------|------|------------------|
| 获客成本 | 开发一个新客的成本是维护老客的5-7倍 | 老客经营ROI远高于新客开拓 |
| 转介绍率 | 满意客户主动转介绍的比例约15% | 经营好1个老客=低成本的获客渠道 |
| 续租率 | 行业平均续租率约50-60% | 续租率每提升10%，空置损失减少30% |
| 空置成本 | 每空置1个月=1整月租金损失 | 二出速度决定利润 |

**设计意图：** 让学员从"维护老客=偶尔问候"切换到"经营老客=主动创造价值"，建立老客是最便宜获客渠道的认知。

## 三、方法论一：老客维护五步法（40分钟）

### 3.1 教学目标

- 理解老客维护是一个**系统性经营闭环**，不是"偶尔问候"
- 掌握每步的关键动作和碳基硅基分工
- 能独立设计老客维护节奏和话术

### 3.2 核心概念

**老客维护五步法**：维护老客户的标准流程，让客户持续产生价值。核心逻辑 = 定期触达 → 需求预判 → 超值服务 → 关系深化 → 价值转化。

**一句话记忆**：**触→预→超→深→转**，五步经营不漏客。

**老客价值公式：** 老客价值 = 续租价值 + 转介绍价值 + 二出价值

| 价值类型 | 是什么 | 单客价值估算 |
|----------|--------|-------------|
| 续租价值 | 客户续租节省的空置+获客成本 | ≈ 1-2个月租金 |
| 转介绍价值 | 老客推荐来的新客 | ≈ 0.5-1个新客/年 |
| 二出价值 | 客户退租后快速二出减少的空置损失 | ≈ 节省15-30天空置期 |

### 3.3 案例引入

**场景还原：**

> 两个客户经理，同样管50个老客户：
>
> **客户经理A**：从不主动联系老客户。客户到期了才知道，匆忙找续租，3个客户流失了，2套房空置了1个月。转介绍？0个。
>
> **客户经理B**：用五步法经营老客户——每月1条有价值信息，每季度1次需求预判，帮客户解决过2次生活问题，请老客户吃过1次饭。结果：50个老客户续租率85%，转介绍8个新客，2个退租的都在到期前15天找到了新租客——零空置。
>
> 同样50个老客，A产出=0，B产出=续租+8个新客+零空置。差别在"经营"还是"忽略"。

### 3.4 每步详细教学设计

#### 第①步：定期触达

**教学目标**：学会用有价值的信息保持存在感，不做"骚扰型"联系

**核心要点：**

| 要点 | 说明 | 示例 |
|------|------|------|
| 有价值才触达 | 不发"在吗""最近怎么样"，发有用信息 | 区域租金趋势/新开便利超市/物业通知 |
| 固定节奏 | 每月1次，不过频不遗漏 | 每月15号发1条市场简报 |
| 差异化内容 | 不同客户发不同内容 | 预算敏感型→省钱攻略；品质型→新盘推荐 |
| 轻量触达 | 一条信息即可，不要长篇大论 | 100字以内，可读性强 |

**触达内容日历：**

| 月份 | 主题 | 内容方向 |
|------|------|----------|
| 1月 | 年度租金盘点 | "2026年XX区域租金变化，您的房子值多少？" |
| 2月 | 春季居住贴士 | "春季除湿/通风小技巧" |
| 3月 | 续租提醒 | "您的租期还剩X个月，提前规划更从容" |
| 4月 | 社区活动 | "XX社区周末市集/亲子活动预告" |
| 5月 | 换房窗口 | "毕业季将至，优质房源提前锁定" |
| 6月 | 半年市场分析 | "上半年租金走势+下半年预判" |

**碳基硅基分工：**

| 碳基（人做） | 硅基（AI帮） |
|-------------|-------------|
| 判断哪个客户适合什么内容 | 按客户标签自动分组，生成差异化触达内容 |
| 个性化调整语气和表达 | 生成100字以内的轻量信息模板 |
| 选择触达时机 | 自动排期提醒，每月15号提醒触达 |

**操作步骤：**
1. 打开六维品质测评 → 导出老客分层清单
2. AI按客户标签分组 → 生成差异化触达内容
3. 人工审核+微调 → 群发/定向发送
4. AI追踪：谁回复了/谁没回复 → 下次触达优化

**常见错误：**
- ❌ 每周都发 → 客户烦了，拉黑你
- ❌ 只发广告 → 客户觉得你只想卖房
- ❌ 群发同一内容 → 客户觉得你不走心
- ✅ 每月1次 + 有价值 + 差异化

#### 第②步：需求预判

**教学目标**：学会在客户开口之前就发现他的下一步需求

**核心要点：**

| 要点 | 说明 | 示例 |
|------|------|------|
| 每季度1次深度沟通 | 不是发信息，是打电话或面聊 | "最近住得怎么样？有什么变化吗？" |
| 关注3个信号 | 换工作/家庭变化/预算变化 | 客户换了工作→可能要换区域 |
| 提前6个月预判续租 | 续租不是到期才谈 | 到期前6个月就开始铺垫 |
| 记录+更新画像 | 每次沟通后更新客户画像 | 新需求/新偏好/新联系人 |

**需求预判清单：**

| 预判维度 | 观察信号 | 可能需求 | 应对策略 |
|----------|----------|----------|----------|
| 职业变化 | 换工作/升职/调岗 | 换区域/升级品质 | 主动推荐通勤更优/品质更高的房源 |
| 家庭变化 | 结婚/生子/父母同住 | 面积升级/换户型 | 提前准备大户型/家庭型房源 |
| 预算变化 | 加薪/降薪 | 品质升级/降档 | 调整推荐范围 |
| 满意度变化 | 投诉增多/回复变慢 | 流失风险 | 主动服务+解决痛点 |
| 合同节点 | 到期前6个月 | 续租/换租 | 提前沟通续租意向+备选方案 |

**碳基硅基分工：**

| 碳基（人做） | 硅基（AI帮） |
|-------------|-------------|
| 读懂客户话外之音 | 分析客户沟通记录变化，标记异常信号 |
| 感知客户情绪变化 | 自动标记"回复频率下降""投诉增多"等预警信号 |
| 做判断和决策 | 整理预判清单+推送提醒 |

**操作步骤：**
1. AI自动扫描：哪些客户有"信号"（回复变慢/投诉/合同临近）
2. 人工判断：哪些信号值得深度沟通
3. 打电话/面聊 → 确认需求变化
4. 更新客户画像 → 调整维护策略

**常见错误：**
- ❌ 到期了才问续租 → 客户已经找了别人
- ❌ 只问"续不续" → 不了解客户真实需求变化
- ✅ 提前6个月铺垫 + 关注3个信号 + 更新画像

#### 第③步：超值服务

**教学目标**：学会帮客户解决问题，让客户"欠你人情"

**核心要点：**

| 要点 | 说明 | 示例 |
|------|------|------|
| 不是分内事才做 | 超出合同范围的帮助 | 帮客户找搬家公司/推荐靠谱维修工 |
| 解决真问题 | 客户的痛点=你的服务点 | 客户投诉噪音→帮联系物业解决 |
| 小事积累信任 | 不需要大事，小事就够了 | 下雨提醒收衣服/节假日的祝福 |
| 不求回报 | 不要每次帮忙都提要求 | 先付出，信任自然来 |

**超值服务清单：**

| 服务类型 | 具体内容 | 频率 | 成本 |
|----------|----------|------|------|
| 生活提醒 | 天气/节日/社区活动 | 适度 | 0 |
| 问题解决 | 报修/投诉/邻里纠纷 | 按需 | 时间 |
| 资源对接 | 搬家/清洁/维修/家政 | 按需 | 0 |
| 小礼物 | 生日/入住周年/节日 | 每年1-2次 | 50-100元 |
| 紧急帮助 | 突发事件/临时需求 | 按需 | 时间 |

**碳基硅基分工：**

| 碳基（人做） | 硅基（AI帮） |
|-------------|-------------|
| 判断客户需要什么帮助 | 记录客户历史问题，自动推荐可提供的帮助 |
| 做超出预期的服务 | 设置生日/入住周年提醒 |
| 提供情感价值 | 生成个性化祝福语/服务通知 |

**操作步骤：**
1. AI标记客户关键日期（生日/入住纪念日）
2. 客户提问题时 → 记录+准备解决方案
3. 主动提供超值服务 → 不等客户开口
4. 每次服务记录存档 → 积累服务档案

**常见错误：**
- ❌ 只做分内事 → 客户觉得你只是"中介"
- ❌ 帮了忙就提要求 → 客户觉得你有目的
- ❌ 承诺做不到的事 → 透支信任
- ✅ 主动+不求回报+说到做到

#### 第④步：关系深化

**教学目标**：学会从"客户关系"升级到"朋友关系"

**核心要点：**

| 要点 | 说明 | 示例 |
|------|------|------|
| 每半年1次深度互动 | 不是发微信，是面对面 | 请吃饭/约咖啡/邀请参加活动 |
| 找到共同话题 | 超越房子，聊生活 | 孩子教育/运动/美食/旅行 |
| 建立多维度连接 | 不止一个联系方式 | 加了微信，还知道他孩子在哪上学 |
| 长期投资心态 | 不是为了成交才维护 | 真心关心，成交是水到渠成 |

**关系深化路径：**

```
交易关系（租了房）
    ↓ 超值服务
服务关系（帮了忙，欠你人情）
    ↓ 定期触达+需求预判
信任关系（有问题第一个想到你）
    ↓ 深度互动+共同话题
朋友关系（不止聊房子）
    ↓ 自然发生
转介绍（主动帮你推荐）
```

**碳基硅基分工：**

| 碳基（人做） | 硅基（AI帮） |
|-------------|-------------|
| 真诚沟通，建立情感连接 | 提醒深度互动节点（每半年1次） |
| 找共同话题 | 分析客户标签，推荐可能感兴趣的话题 |
| 把握关系深化的节奏 | 记录互动历史，建议下一步动作 |

**操作步骤：**
1. AI提醒：本季度需要深度互动的客户清单
2. 选择方式：请吃饭/约咖啡/邀请活动
3. 准备话题：AI推荐3个可能感兴趣的话题
4. 互动 → 记录新信息 → 更新画像

**常见错误：**
- ❌ 关系深化=频繁发消息 → 客户觉得你闲
- ❌ 聊着聊着又推房 → 客户觉得你有目的
- ❌ 只在线上聊 → 关系停留在表面
- ✅ 每半年1次面对面 + 超越房子的话题

#### 第⑤步：价值转化

**教学目标**：学会把经营的老客关系转化为续租/转介绍/二出

**核心要点：**

| 要点 | 说明 | 示例 |
|------|------|------|
| 转介绍三时机 | 满意时/帮忙后/节日时 | 客户刚入住很满意→问转介绍 |
| 续租三提前 | 提前6个月铺垫+3个月沟通+1个月签约 | 不等到期才谈 |
| 自然转化 | 不硬推，水到渠成 | 关系到位了，转介绍自然发生 |
| 感谢闭环 | 转介绍成功一定要感谢 | 小礼物+真诚感谢 |

**转介绍话术：**

| 时机 | 话术 |
|------|------|
| 入住满意时 | "住得还满意吧？以后有朋友找房子，您帮忙推荐推荐，我给您的朋友也安排最好的服务！" |
| 帮了忙之后 | "小事一桩！以后有什么需要随时找我。对了，您身边有朋友在找房子吗？我可以同样用心帮他们。" |
| 节日问候时 | "[客户称呼]新年快乐！新的一年有朋友要租房的，记得推荐给我哈，我一定好好服务！" |

**碳基硅基分工：**

| 碳基（人做） | 硅基（AI帮） |
|-------------|-------------|
| 判断转介绍的最佳时机 | 标记"满意度高""最近有互动"的客户 |
| 真诚表达感谢 | 生成个性化转介绍话术 |
| 处理转介绍的后续服务 | 新客自动关联"来自XX推荐"，优先服务 |

**操作步骤：**
1. AI标记：本周期内满意度最高的客户Top10
2. 人工判断：哪些客户适合转介绍/续租沟通
3. 用话术模板自然引导
4. 转介绍成功 → 感谢闭环 → 记录

**常见错误：**
- ❌ 每次聊天都问转介绍 → 客户烦
- ❌ 转介绍成功不感谢 → 下次不会推荐了
- ❌ 续租到期才谈 → 客户已经找好下家
- ✅ 时机对+自然引导+感谢闭环

## 四、方法论二：二出招租五步法（35分钟）

### 4.1 教学目标

- 理解二出是**时间战**，核心是减少空置期
- 掌握每步的时间节点、关键动作和AI辅助
- 能独立设计二出招租时间表

### 4.2 核心概念

**二出招租五步法**：客户退租后，快速完成二次出租的方法。核心逻辑 = 提前预警 → 精准匹配 → 快速带看 → 高效签约 → 交接过渡。

**一句话记忆**：**警→配→看→签→交**，五步二出零空置。

**时间就是金钱：**

| 空置天数 | 月租5000元损失 | 月租8000元损失 |
|----------|---------------|---------------|
| 7天 | 1,167元 | 1,867元 |
| 15天 | 2,500元 | 4,000元 |
| 30天 | 5,000元 | 8,000元 |
| 60天 | 10,000元 | 16,000元 |

### 4.3 案例引入

**场景还原：**

> 同一套房，两种处理方式：
>
> **方式A**：客户退租了才知道，匆忙拍照发房源，等了3周才找到新租客。空置21天，损失5000×21/30=3,500元。
>
> **方式B**：提前30天就知道客户不续租，提前20天开始匹配新客，到期前5天就签好了新租约，到期当天无缝交接。空置0天，损失0元。
>
> 同一套房，差别就在"提前"还是"事后"。

### 4.4 每步详细教学设计

#### 第①步：提前预警

**教学设计**：这是五步中最重要的——错过预警，后面步步被动

**时间节点**：到期前30天

**关键动作：**

| 动作 | 怎么做 | 产出 |
|------|--------|------|
| 续租意向确认 | 主动问客户"到期后有续租打算吗" | 明确续/不续/待定 |
| 不续原因分析 | 问"是什么原因不考虑续租" | 了解流失原因 |
| 房屋状况评估 | 去实地看/请租务管家评估 | 整备清单 |
| 新客预热启动 | 不续→立即启动二出流程 | 二出时间表 |

**续租意向确认话术：**
```
"[客户称呼]，您租期还剩大概1个月了，
 想问问您接下来有什么打算？
 如果续租的话我帮您走流程，
 如果想换房的话我也帮您留意着。"
```

**AI辅助：**
- 自动标记到期前30天的客户 → 预警清单
- 分析历史续租率 → 预判哪些客户可能不续
- 不续客户 → 自动启动二出流程模板

**常见错误：**
- ❌ 到期前7天才问 → 来不及找新客
- ❌ 只问续不续，不问原因 → 错过挽留机会
- ✅ 提前30天 + 问续租+问原因 + 同步启动二出

#### 第②步：精准匹配

**教学设计**：基于老客画像找到"最对"的新客

**时间节点**：到期前20天

**关键动作：**

| 动作 | 怎么做 | 产出 |
|------|--------|------|
| 老客画像分析 | 这套房适合什么人？预算/职业/偏好 | 新客画像 |
| 潜在客户筛选 | 从待租客户库中找匹配对象 | 候选名单 |
| 房源价值重塑 | 退租后有没有可优化的点？ | 优化方案 |
| 六维品质测评 | 用测评做房源品质评估 | 品质报告 |

**精准匹配操作：**

```
1. 六维品质测评 → 评估房源6个维度得分
2. 根据得分 → 确定目标客户画像
   - 交通分高 → 通勤型客户
   - 配套分高 → 生活便利型客户
   - 品质分高 → 品质型客户
3. 从待租客户库筛选匹配 → 候选名单
4. 人工判断 → 选Top5优先推进
```

**AI辅助：**
- 六维品质测评自动生成品质报告+目标客群画像
- 按画像从客户库匹配 → 候选名单
- 生成个性化推荐话术

**常见错误：**
- ❌ 什么客户都推 → 匹配度低，浪费时间
- ❌ 不做品质评估 → 不知道房子卖给谁
- ✅ 测评定画像 → 画像定客群 → 客群定推荐

#### 第③步：快速带看

**教学设计**：在有限时间内最大化带看效率

**时间节点**：到期前15天

**关键动作：**

| 动作 | 怎么做 | 产出 |
|------|--------|------|
| 集中带看安排 | 把候选客户集中在2-3个时段 | 高效带看 |
| 现场品质展示 | 提前整理好房屋状态 | 最佳呈现 |
| 紧迫感营造 | "这套很快有人要定了" | 加速决策 |
| 收集反馈 | 每次带看后记录客户反馈 | 优化依据 |

**集中带看排期模板：**

| 时间 | 客户 | 匹配度 | 重点展示 | 反馈 |
|------|------|--------|----------|------|
| 周六10:00 | 张先生 | 高 | 通勤+采光 | |
| 周六14:00 | 李女士 | 中 | 配套+价格 | |
| 周日10:00 | 王先生 | 高 | 品质+朝向 | |

**AI辅助：**
- 自动排期：按匹配度+客户时间偏好推荐带看时段
- 生成每类客户的带看重点和话术
- 带看后自动跟进提醒

**常见错误：**
- ❌ 房屋没整理就带看 → 第一印象差
- ❌ 散着约，效率低 → 一次1个，来回跑
- ✅ 集中安排 + 提前整备 + 按匹配度排优先级

#### 第④步：高效签约

**教学设计**：快速锁定客户，不拖泥带水

**时间节点**：到期前7天

**关键动作：**

| 动作 | 怎么做 | 产出 |
|------|--------|------|
| 筛选最优客户 | 从带看反馈中选意向最高的 | 签约对象 |
| 快速报价 | 给出有竞争力的价格 | 报价方案 |
| 消除最后顾虑 | 逐一解决签约前的疑虑 | 无障碍签约 |
| 限时签约 | 给出签约截止时间 | 加速决策 |

**签约推进话术：**
```
"[客户称呼]，这套房子看的人不少，
 我帮您争取了[优惠/条件]，
 您如果确定的话，咱们今天把合同签了？
 我这边帮您安排好一切。"
```

**AI辅助：**
- 分析带看反馈 → 推荐最优签约对象
- 生成报价方案（参考市场价+房东底线）
- 合同模板自动填充

**常见错误：**
- ❌ 同时跟多个客户谈 → 都不确定
- ❌ 没有签约截止时间 → 客户一直拖
- ✅ 选最优+限时签约+消除顾虑

#### 第⑤步：交接过渡

**教学设计**：保证老客退租→新客入住的无缝衔接

**时间节点**：到期前3天~到期当天

**关键动作：**

| 动作 | 怎么做 | 产出 |
|------|--------|------|
| 老客退租交接 | 验房+退押金+感谢 | 平稳退租 |
| 房屋快速整备 | 清洁+小修+布置（1-3天内） | 最佳状态 |
| 新客入住交接 | 验房+签入住确认+服务说明 | 顺利入住 |
| 关系承接 | 新客→进入老客维护五步法 | 持续经营 |

**交接检查清单：**

| 检查项 | 老客退租 | 新客入住 |
|--------|----------|----------|
| 水电燃气 | 读数记录 | 读数确认 |
| 家具家电 | 完好确认 | 功能演示 |
| 钥匙门禁 | 全部收回 | 全部交付 |
| 房屋清洁 | 退租标准 | 入住标准 |
| 物业交接 | 费用结清 | 费用说明 |

**AI辅助：**
- 自动生成交接清单
- 新客信息自动导入客户解码器 → 启动老客维护流程
- 设置新客入住7天回访提醒

**常见错误：**
- ❌ 新老客交接有gap → 空置期损失
- ❌ 退租验房不仔细 → 纠纷风险
- ❌ 新客入住后不跟进 → 错过经营起点
- ✅ 无缝交接 + 严格验房 + 新客即入维护流程

## 五、方法论三：以出定收两聚焦两把控（25分钟）

### 5.1 教学目标

- 理解"以出定收"是**反向决策**——先看能不能租出去，再决定要不要收进来
- 掌握两聚焦（区域+户型）和两把控（节奏+成本）的具体方法
- 能独立做收房决策分析

### 5.2 核心概念

**以出定收**：根据出房情况反向决定收房策略。核心逻辑 = 先问"能不能出"，再决定"要不要收"。

**两聚焦两把控：**

| 维度 | 是什么 | 为什么重要 |
|------|--------|-----------|
| **聚焦区域** | 只收你能出得动的区域 | 你不了解的区域=赌博 |
| **聚焦户型** | 只收市场流动性最好的户型 | 冷门户型=空置风险 |
| **把控节奏** | 控制收房速度，不贪多 | 收太多出不动=资金压力 |
| **把控成本** | 控制收房价格，留足利润空间 | 收太贵=怎么出都亏 |

**一句话记忆**：**区+型聚，奏+本控**，以出定收不踩坑。

### 5.3 案例引入

**场景还原：**

> 客户经理老赵上个月一口气收了5套房——3套在熟悉区域，2套在陌生区域。
>
> 熟悉区域的3套，2周内全部出租。陌生区域的2套，1个月了还没人来看。
>
> 为什么？老赵在陌生区域没有客户积累，不了解当地客群需求，推广也不知道从哪发力。
>
> 教训：**收房不是越多越好，是你能出得动的才收。**

### 5.4 详细展开

#### 两聚焦之一：聚焦区域

**区域聚焦三原则：**

| 原则 | 说明 | 判断标准 |
|------|------|----------|
| 熟悉度 | 只收你深耕过的区域 | 在该区域有≥10个老客户 |
| 客户池 | 该区域有稳定的客户来源 | 每月有≥3个该区域咨询 |
| 竞争度 | 你在该区域有竞争优势 | 转介绍/社区深耕有基础 |

**区域评分卡：**

| 评分维度 | 权重 | 打分(1-10) | 加权得分 |
|----------|------|-----------|----------|
| 熟悉度 | 30% | | |
| 客户池 | 40% | | |
| 竞争度 | 30% | | |
| **总分** | | | ≥7分才收 |

**碳基硅基分工：**

| 碳基（人做） | 硅基（AI帮） |
|-------------|-------------|
| 判断区域熟悉度 | 六维品质测评分析区域出房数据 |
| 评估竞争优势 | 统计该区域的客户咨询量+成交率 |

#### 两聚焦之二：聚焦户型

**户型聚焦三原则：**

| 原则 | 说明 | 判断标准 |
|------|------|----------|
| 流动性 | 收市场最需要的户型 | 该户型平均出房天数≤15天 |
| 匹配度 | 你的客户群体最需要的户型 | 客户需求Top3户型 |
| 利润率 | 收租售比最优的户型 | 年租金/收房价≥合理回报率 |

**户型出房数据参考：**

| 户型 | 流动性 | 适合客群 | 风险等级 |
|------|--------|----------|----------|
| 一居/开间 | 高 | 单身/年轻白领 | 低 |
| 两居 | 最高 | 情侣/小家庭 | 最低 |
| 三居 | 中 | 家庭/合租 | 中 |
| 四居+ | 低 | 大家庭/办公 | 高 |

**AI辅助：** 六维品质测评分析各户型出房速度+租金回报+空置率数据

#### 两把控之一：把控节奏

**节奏把控三原则：**

| 原则 | 说明 | 具体方法 |
|------|------|----------|
| 不贪多 | 每月收房量≤出房量 | 收出比控制在1:1.2以上 |
| 先出后收 | 先有客户再收房 | 确认有≥2个意向客户再收 |
| 预留缓冲 | 保持10-15%的安全库存 | 不过度收房，留周转空间 |

**收出节奏模板：**

| 指标 | 安全线 | 警戒线 | 危险线 |
|------|--------|--------|--------|
| 收出比 | ≥1:1.2 | 1:1.0 | <1:1.0 |
| 空置率 | ≤10% | 10-20% | >20% |
| 出房天数 | ≤15天 | 15-30天 | >30天 |

#### 两把控之二：把控成本

**成本把控三原则：**

| 原则 | 说明 | 具体方法 |
|------|------|----------|
| 留利润 | 收房价+整备成本<市场租金×合理倍数 | 确保有利润空间 |
| 算总账 | 不是只看收房价，要看全部成本 | 收房+整备+空置+管理 |
| 防超支 | 整备预算提前定，不超预算 | 先评估再收房，不在收房后加预算 |

**成本计算器：**

```
总成本 = 收房价 + 整备费用 + 预估空置损失 + 管理成本
保本租金 = 总成本 ÷ 12个月
目标租金 = 保本租金 × (1 + 目标利润率)

判断标准：
- 目标租金 ≤ 市场均价的110% → 可收
- 目标租金 > 市场均价的110% → 不收（风险高）
```

**AI辅助：** 六维品质测评自动计算成本+租金回报+风险评估

### 5.5 练习设计

**练习：收房决策分析（10分钟）**

```
场景：房东找你收一套房——XX小区3室1厅，120㎡，报价6800元/月收房价
- 你对XX小区有3个老客户，但不是深耕区域
- 该户型市场平均出房天数：25天
- 同小区3居市场租金：8500元/月
- 预估整备费用：8000元
- 预估空置期：20天

1. 区域评分（3维度打分）
2. 户型评分（3维度打分）
3. 节奏判断（收出比是否合理？）
4. 成本计算（保本租金？目标租金？是否可收？）
5. 最终决策：收/不收/有条件收？理由是什么？
```

## 六、方法论四：推广四步（20分钟）

### 6.1 教学目标

- 理解推广是房源去化的**加速器**，不是"发个帖子"这么简单
- 掌握每步的案例+操作方法
- 能独立设计房源推广方案

### 6.2 核心概念

**推广四步**：房源推广的标准四步法。核心逻辑 = 信息整理 → 多渠道分发 → 精准推送 → 效果追踪。

**一句话记忆**：**整→发→推→追**，四步推广快去化。

### 6.3 每步案例+操作

#### 第①步：信息整理

**案例：**
> 同一套房，两种推广：
>
> **客户经理A**：随手拍3张照片，写"XX小区两居，5800元"。发出去3天，0个咨询。
>
> **客户经理B**：拍8张精选照片（含户型图+窗外景观），写3个亮点+周边配套+适合人群。发出去2小时，3个咨询。

**操作要点：**

| 整理要素 | 内容 | 标准 |
|----------|------|------|
| 照片/视频 | ≥8张精选照片+1个视频 | 覆盖每个房间+窗外景观 |
| 文案 | 3个亮点+周边配套+适合人群 | 100字以内，有吸引力 |
| 价格策略 | 定价参考市场+留议价空间 | 市场价95-105% |
| 标签 | 3-5个精准标签 | 地铁近/精装修/拎包入住等 |

**AI辅助：** AI根据房源信息自动生成文案+推荐标签+建议定价

#### 第②步：多渠道分发

**案例：**
> 客户经理小李只在一个平台发房源，每天等咨询。客户经理小张在5个渠道同步分发，咨询量是小李的3倍。

**渠道分发策略：**

| 渠道 | 内容形式 | 分发时间 | 优先级 |
|------|----------|----------|--------|
| 内部系统 | 标准房源档案 | 收房当天 | ⭐⭐⭐ |
| 微信朋友圈 | 精选照片+亮点文案 | 晚8-9点 | ⭐⭐⭐ |
| 小红书 | 图文笔记/视频 | 工作日中午 | ⭐⭐ |
| 短视频平台 | 15-60秒房源视频 | 工作日晚7-9点 | ⭐⭐ |
| 老客推送 | 定制化推荐 | 工作日上午 | ⭐⭐⭐ |

**AI辅助：** AI一键生成多渠道适配内容（不同格式+不同文案风格）

#### 第③步：精准推送

**案例：**
> 客户经理小王把同一套房源推给了20个人，2个回复。客户经理小赵用六维品质测评筛选了5个最匹配的客户，3个回复，1个约带看。精准>海量。

**精准推送操作：**

```
1. 六维品质测评 → 房源画像
2. 房源画像 × 客户需求 = 匹配清单
3. 按匹配度排序 → 选Top5
4. 生成个性化推荐语 → 定向推送
```

**AI辅助：** 六维品质测评自动匹配+生成个性化推荐话术

#### 第④步：效果追踪

**案例：**
> 客户经理小刘发了10条房源推广，从来不看数据，不知道哪条有效。客户经理小陈每条推广都追踪效果，3个月后总结出"小红书图文+晚8点发"转化率最高，优化了推广策略。

**追踪指标：**

| 指标 | 怎么看 | 优化方向 |
|------|--------|----------|
| 浏览量 | 多少人看了 | 浏览低→优化封面/标题 |
| 咨询量 | 多少人问了 | 咨询低→优化文案/定价 |
| 带看量 | 多少人来看了 | 带看低→优化匹配/话术 |
| 成交量 | 多少人签了 | 成交低→优化带看/谈判 |
| 平均出房天数 | 从发到签多久 | 全流程优化 |

**AI辅助：** AI自动统计追踪数据+推荐优化方向

### 6.4 练习设计

**练习：设计1套房源的推广方案（8分钟）**

```
3. 精准推送（2分钟）：描述目标客户画像+推荐话术
4. 效果追踪（2分钟）：列出3个核心追踪指标
```

## 七、方法论五：整备五项（15分钟）

### 7.1 教学目标

- 理解整备是**提升房源价值**的关键环节，不是"打扫一下"
- 掌握5项检查清单+执行标准
- 能独立完成整备评估和执行方案

### 7.2 核心概念

**整备五项**：房屋整备的5个关键环节。核心逻辑 = 清洁 → 维修 → 美化 → 布置 → 呈现。

**一句话记忆**：**洁→修→美→布→呈**，五项整备房子靓。

### 7.3 检查清单+执行标准

#### ① 清洁

| 检查项 | 执行标准 | 不达标的代价 |
|--------|----------|-------------|
| 地面 | 无污渍/无灰尘/无异味 | 客户第一印象差，5分钟想走 |
| 厨卫 | 无油渍/无水垢/无霉斑 | 客户觉得"脏"，直接排除 |
| 窗户 | 玻璃明亮/窗台无灰 | 采光打折扣，价值感下降 |
| 空气 | 无异味/通风良好 | 客户进门就皱眉 |
| 细节 | 开关/门把手/角落无灰 | 影响整体精致感 |

**执行标准**：达到"自己愿意住"的标准

**预算**：专业清洁200-500元/次

#### ② 维修

| 检查项 | 执行标准 | 不处理的代价 |
|--------|----------|-------------|
| 水电 | 水龙头/灯/开关正常 | 客户觉得"破"，砍价理由 |
| 家电 | 所有家电功能正常 | 客户担心后续问题 |
| 门窗 | 开关顺畅/密封良好 | 影响安全和隔音 |
| 墙面 | 无裂缝/无脱落/无水渍 | 客户觉得"旧"，砍价理由 |
| 管道 | 无堵塞/无漏水 | 隐患=纠纷风险 |

**执行标准**：所有功能正常，无安全隐患

**预算**：小修500-2000元，大修需评估ROI

#### ③ 美化

| 检查项 | 执行标准 | 不做的代价 |
|--------|----------|-----------|
| 墙面刷新 | 局部补漆或全屋刷新 | 老旧感=降低档次 |
| 灯光优化 | 暖光灯/亮度适中 | 冷白光=没有家的感觉 |
| 绿植点缀 | 1-2盆绿植 | 空间没有生气 |
| 窗帘更换 | 简约干净 | 老旧窗帘=降档次 |
| 气味管理 | 香薰/清新剂 | 异味=直接劝退 |

**执行标准**：达到"样板间感觉"

**预算**：基础美化500-1500元

#### ④ 布置

| 检查项 | 执行标准 | 不做的代价 |
|--------|----------|-----------|
| 家具摆放 | 合理动线+空间感 | 拥挤感=觉得面积小 |
| 软装搭配 | 抱枕/地毯/挂画 | 空旷感=没有温度 |
| 收纳整理 | 物品归位+隐藏收纳 | 杂乱=空间感差 |
| 镜面擦拭 | 所有镜面明亮 | 暗淡=空间缩小 |

**预算**：软装美化500-3000元

#### ④ 布置

| 检查项 | 执行标准 | 不做的代价 |
|--------|----------|-----------|
| 家具摆放 | 动线合理/空间最大化 | 拥挤感=客户觉得小 |
| 视觉焦点 | 每个房间有1个焦点 | 没记忆点=看完就忘 |
| 功能分区 | 客厅/卧室/工作区清晰 | 功能模糊=不知道怎么用 |
| 留白 | 适当空出空间 | 塞满=压抑感 |

**执行标准**：客户走进每个房间，5秒内知道"这里可以做什么"

**预算**：如需增加家具，1000-5000元

#### ⑤ 呈现

| 检查项 | 执行标准 | 不做的代价 |
|--------|----------|-----------|
| 拍照 | ≥8张精选+1个视频 | 照片差=线上0咨询 |
| 文案 | 3个亮点+适合人群 | 没文案=客户不知道好在哪 |
| 看房体验 | 提前开窗通风+开灯+放音乐 | 冷冰冰=客户没有感觉 |
| 季节适配 | 夏季开空调/冬季开暖气 | 不舒适=客户不想久待 |

**执行标准**：照片达到"看了想来"的效果，带看达到"来了不想走"的效果

**预算**：0（时间投入为主）

### 7.4 整备ROI计算

```
整备投入：清洁(300) + 维修(1000) + 美化(1500) + 布置(0) = 2800元

不整备的风险：
- 空置期多15天 → 损失 5800×15/30 = 2900元
- 租金被压低200元/月 → 一年损失 2400元

结论：整备投入2800元，避免损失5300元，ROI=1:1.9
整备是投资，不是成本。
```

### 7.5 练习设计

**练习：整备评估+方案（5分钟）**

```
场景：一套即将二出的房源，已空置5天，当前状态：
- 地面有灰尘，厨房有油渍
- 一个水龙头漏水，2个灯泡不亮
- 墙面有1处脱落，窗帘老旧
- 家具摆放拥挤，没有绿植

任务：
1. 按五项检查清单打分（1-10分）
2. 列出必须做的项目（优先级排序）
3. 估算整备预算
4. 预计整备完成时间
```

## 九、产品衔接：六维品质测评实操引导（15分钟）

### 9.1 产品价值主张

> 六维品质测评 = 帮客户经理把"靠感觉判断"变成"靠数据决策"的工具

### 9.2 六维品质模型简介

**六维品质**：评估房源品质的6个维度

| 维度 | 评估什么 | 影响什么 |
|------|----------|----------|
| 交通 | 通勤便利性/地铁距离 | 通勤型客户的决策 |
| 配套 | 商业/教育/医疗/生活配套 | 生活便利型客户的决策 |
| 品质 | 装修/家电/维护状态 | 品质型客户的决策 |
| 采光 | 朝向/楼层/窗户 | 居住舒适度 |
| 隔音 | 墙体/门窗/临街情况 | 居住体验 |
| 安全 | 门禁/监控/小区管理 | 安全感 |

### 9.3 课堂演示（5分钟）

1. 打开 fengsheng.tech/quality
2. 输入模拟房源信息："XX小区，2室1厅，85㎡，15层，南向，精装修，地铁8分钟"
3. 展示AI生成的六维品质报告：
   - 6个维度评分（雷达图）
   - 目标客群推荐
   - 整改建议
   - 推广策略建议
4. 强调：这就是"以数据做决策"的实际落地——不用靠感觉，测一测就知道

### 9.4 学员实操（10分钟）

每位学员完成以下操作：

**Part 1：房源品质评估（5分钟）**
- 输入1套真实/模拟房源
- 查看六维品质报告
- 根据报告调整推广策略

**Part 2：客户分层（5分钟）**
- 导出10个老客户清单
- 用六维品质测评做客户→房源匹配
- 截图保存分层结果

**验证计数：** 本课学员到fengsheng.tech/quality完成注册 + 首次测评 → 计数+1
