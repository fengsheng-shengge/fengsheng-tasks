## 一、本课目标

**一句话：** AI得知道你怎么学最快——你是看文档型、动手型、还是对话型？教会AI按你的方式干活，不是AI按它的方式喂你。

**学完这课你能：**
1. 识别自己的学习风格——你是哪种类型，AI就该按哪种方式跟你互动
2. 设计一个自主闭环任务——输入→处理→输出→验证，AI不用等指令就能跑完
3. 让AI按你习惯的格式输出——你要摘要别给长文，你要表格别给段落

**交付物：** 跑通1个自主闭环任务（含任务说明书+执行记录+产出物）

**验收标准：**
- 闭环任务完整跑通（有输入→有处理→有输出→有验证）
- 产出物格式匹配你的学习方式
- AI在执行过程中无需你重复解释同一个问题

## 二、开场：你是在用AI，还是在被AI用？

### 2.1 90%的人用AI的方式是错的

你有没有这种感觉：
- 问AI一个问题，它回答了，你发现不是你要的
- 再问一次，换个问法，它又给你另一堆东西
- 最后你花了20分钟调教，AI才给你一个勉强能用的东西

**问题不在AI，在你用的方式。**

你把AI当"问答机"——你问，它答，答完结束。但你的工作不是问答，是**干活**。问答机解决不了干活的问题。

### 2.2 什么是"被动应答"？

```
你（人）→ 问问题 → AI → 给答案 → 你（人）→ 再问 → AI → 再答 → ...
```

这种模式的问题：
- 每次你都要想"怎么问"
- AI每次都从头开始，不记得你上一步说了什么
- 你是发动机，AI是零件——你在推着它走

### 2.3 什么是"自主闭环"？

```
AI → 自动执行 → 产出结果 → 你只验证 → 不对就打回重跑
```

AI变成发动机，你是**验收方**——你告诉它要什么结果，它自己跑完，你只负责检查对不对。

**关键转变：**
> 从"你问AI答" → "AI按你的方式主动干"

## 三、实操第一步：识别你的学习风格

### 3.1 三种学习类型

不是所有人吸收信息的方式都一样。在让AI按你的方式干活之前，**你得先知道自己是哪种类型**。

| 类型 | 特征 | 你习惯的信息格式 | AI应该给你什么 |
|------|------|----------------|----------------|
| **文档型** | 喜欢看文字、读案例 | 长文、报告、说明文档 | 段落式输出，有背景有结论有细节 |
| **动手型** | 喜欢看示例、跟着做 | 清单、步骤、模板 | 分步骤操作指南，每步有动作有产出 |
| **对话型** | 喜欢聊、问一句答一句 | 问答、讨论、启发式 | 短句回应，多问问题引导你思考 |

### 3.2 怎么判断自己是什么类型？

回答3个问题：

**问题1：当你学一个新技能时，你第一反应是什么？**
- A. 找文档看 → 文档型
- B. 找个视频跟着做 → 动手型
- C. 找个懂的人问 → 对话型

**问题2：你收到一个方案，你希望是什么形式？**
- A. Word文档，有背景有分析有建议 → 文档型
- B. Excel清单，第一步做什么第二步做什么 → 动手型
- C. PPT，边讲边演示 → 对话型

**问题3：AI给了你一个方案，你第一反应是什么？**
- A. "写得很详细，我再看看" → 文档型
- B. "步骤很清楚，我试试" → 动手型
- C. "等等，我有几个问题想探讨" → 对话型

**统计你的答案：**
- 3个A → 文档型
- 3个B → 动手型
- 3个C → 对话型
- 混合型 → 记录你的组合，AI输出时要兼顾两种格式

### 3.3 把你的学习类型告诉AI

找到你AI的设定文档（上一课你做的"AI身份设定文档"），在"沟通偏好"部分加一条：

```
## 沟通偏好
- 我的学习类型：[文档型/动手型/对话型]
- 我习惯的信息格式：[描述你喜欢的格式]
- 我不喜欢的格式：[描述你会扔掉重来的格式]
```

## 四、实操第二步：设计一个自主闭环任务

### 4.1 什么是自主闭环？

自主闭环 = AI不用你下第二道指令，就能从"开始"跑到"完成"。

```
输入 → AI自动处理 → 产出结果 → 你验证 → 通过 or 打回重跑
         ↑                              ↓
         ←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←
```

**举例：**

**非闭环任务（被动应答）：**
```
你：帮我写一个装修方案
AI：好的，请告诉我客户的户型、预算、需求...
你：80平，15万，业主是中年夫妻...
AI：[给出一版方案]
你：方案不错，但第3点太复杂了，能简化吗？
AI：[修改]
你：可以了
```

**闭环任务（自主）：**
```
你：[一次性给完所有信息+明确验收标准]
   "我要一个装修方案，业主情况：80平/15万/中年夫妻。
   要求：①控制在15万内 ②列出主材品牌和价格 ③给出3个备选方案。
   按表格输出，每列：项目/品牌/型号/单价/数量/合计。
   不确定的地方标注[需确认]。"

AI：[一次性输出完整方案，包含3个备选+表格+[需确认]标注]

你：第2个备选方案pass，其他可以
AI：[输出最终版+执行清单]

你：[验收通过]
```

### 4.2 闭环任务的4个要素

不是所有任务都能闭环。你得先判断这个任务能不能闭环。

**1. 输入要素明确**
- 这个任务需要什么信息？
- 这些信息是你一次性能给完的，还是需要外部输入的？

**2. 处理逻辑可描述**
- AI知道"怎么处理"吗？
- 规则、标准、判断逻辑能写出来吗？

**3. 产出格式可定义**
- 你要的是文档？表格？清单？
- 格式能提前说清楚吗？

**4. 验收标准可验证**
- 什么是"完成了"？
- 什么情况下要打回重跑？

**能闭环的任务特征：** 输入→处理→输出都能提前说清楚，不需要边做边问。

**不能闭环的任务：** 需要外部信息、需要灵活判断、需要边做边调整。

### 4.3 你的第一个闭环任务

**任务设计模板：**

```
【任务名称】：[给这个任务起个名字]

【任务目标】：[一句话说清楚要什么结果]

【输入要素】：
- [要素1]：[具体内容或标注"待补"]
- [要素2]：[具体内容或标注"待补"]

【处理规则】：
1. [规则1]
2. [规则2]
3. [规则3]

【产出格式】：
- 格式类型：[文档/表格/清单/PPT/其他]
- 必须包含：[列1、列2...]
- 不包含：[明确排除的内容]
- 格式示例：[如果有的话]

【验收标准】：
- [√] 完成了：[什么情况算完成]
- [×] 打回：[什么情况下要重跑]
- [注意]：[特别注意的点]

【执行记录区】：
[跑通后记录：执行时间、输入、输出、问题、调整]
```

## 五、实操第三步：让AI按你的方式输出

### 5.1 你不只是在要结果，你要的是"你能用的"结果

很多人犯的错：让AI跑完就完了，结果发现格式不对、白跑了。

**你给AI结果要求时，要同时给：**
1. **内容要求**（要什么）
2. **格式要求**（长什么样）
3. **禁止要求**（什么不要）

### 5.2 格式要求的三个层次

| 层次 | 你说 | AI给 | 风险 |
|------|------|------|------|
| **基础层** | "给我一个方案" | AI觉得什么方案都行 | 你要的可能没有 |
| **进阶层** | "给我一个方案，要有背景分析+3个建议" | 有框架但细节不一定对 | 大方向对但细节要改 |
| **完整层** | "给我一个方案..."+格式+禁止 | 你能直接用 | 需要多次调教 |

### 5.3 格式要求的万能模板

在给AI下任务时，加上这段：

```
【格式要求】
- 篇幅：[具体字数或"控制在X字以内"]
- 结构：①[部分1] → ②[部分2] → ③[部分3]
- 呈现：[表格/清单/段落/对比式...]
- 禁止出现：[明确不要的内容]
- 必须出现：[明确要有的内容]
```

## 七、常见问题

### Q1：什么任务适合闭环？什么不适合？

**适合闭环：**
- 有明确输入的任务（客户信息、产品参数、日期数量）
- 有标准处理规则的任务（报价、方案生成、报告撰写）
- 有固定产出格式的任务（表格、清单、模板）

**不适合闭环：**
- 需要外部信息才能开始（外部审批、数据查询）
- 需要灵活判断（战略决策、创意发散）
- 第一次探索（你自己都不知道要什么）

### Q2：闭环任务跑一半要调整怎么办？

**正常流程：**
1. AI输出 → 你发现问题 → 打回标注 → AI重跑 → 再验证
2. 每次打回要记录："哪不对，为什么，应该是什么样"
3. 3次以内跑通 → 闭环成功

**超过3次：** 说明任务设计有问题，重新梳理输入要素和处理规则。

### Q3：AI跑完结果我一看就扔，怎么让它一次到位？

**原因：输入没说清楚**

加上这三句：
- "我要的是[X]，不是[Y]"（明确不要什么）
- "重点是[X]，其他可以简略"（明确重点）
- "如果有不确定的，标注[待确认]，不要瞎编"（诚实标注）
