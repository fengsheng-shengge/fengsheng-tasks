#### 4.1.4 解码逻辑改进

**当前问题：解码结果是固定模板文本，跟客户输入的具体内容不联动。

**迭代版解码逻辑：**

```
解码引擎（关键词匹配 + 模板拼接：

输入 → 分词 → 命中关键词 → 拼接对应道/法/术
```

1. **关键词库（6 组关键词，每组对应一套"道+法+术"模板
2. **动态拼接**：根据客户输入的关键词命中越多，解码结果越贴近客户输入中提到的关键信息

例：客户说"这个户型我不太满意，价格也高了"
→ 关键词命中：价格焦虑 / 户型不满
→ 道：客户不是价格，是"值不值" + 户型是否满足需求
→ 法：先认同价格感受 → 拆解户型具体哪里不满意 → 给出具体建议
→ 术："我理解你的感受，价格和户型都是..."

#### 4.2.2 关键改进：评分从 `Math.random()` → 基于答案的**真实评分**

**每一题对应一个维度，每一个选项有固定分数（0-3 分）：
- 高选项 → 3 分
- 中选项 → 2 分
- 低选项 → 1 分
- 极低选项 → 0 分

**每一个维度 3 题，维度总分 = 3 × 3 = 9 分
**六维总分 = 6 × 9 = 54 分
**最终百分制换算：(实际得分 / 54) × 100

**评分分级：**
- 90+：居住品质优秀
- 75-89：品质良好
- 60-74：品质一般
- <60：需要改善

**理想居住画面**：**跟答题内容联动**（SVG 绘制：
- 采光高 → 画面中有大窗户、阳光
- 安静高 → 画面中有绿植/窗帘
- 温度高 → 画面中有空调/暖气
- 布局高 → 画面中房间布局合理
- 邻里高 → 画面中有社区
- 配套高 → 画面中有商店/学校
