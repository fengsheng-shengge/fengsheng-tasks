## 一、老客分层标准（A/B/C/D四级客户）

### 1.1 分层逻辑

```
老客价值 = 续租价值 + 转介绍价值 + 口碑价值 + 升级价值

分层目的：把有限精力投给最有价值的客户，不做"平均主义"
```

### 1.2 四级分层标准

#### A级客户——超级节点（约10%）

| 维度 | 标准 |
|------|------|
| **定义** | 持续续租 + 高频率转介绍 + 愿意为你背书 |
| **识别条件**（满足≥3条即为A级） | ① 已续租≥2次（租期≥3年）；② 过去12个月转介绍≥3个且成交≥1个；③ 主动在社交圈推荐过你；④ 接受过你的超值服务并表达感谢；⑤ 租金支付从未逾期 |
| **维护策略** | 重点经营关系（非交易关系）：每季度1次线下见面（吃饭/喝茶）、重要节日送定制礼品（搬家纪念日/生日）、优先推荐最优房源、邀请参加VIP活动 |
| **维护频率** | 每月≥1次有温度的触达（不是群发广告） |
| **预期产出** | 年度续租率 ≥90%，年度转介绍 ≥3个/人 |
| **精力投入** | 占老客总数10%，投入30%精力 |

#### B级客户——稳定产出（约30%）

| 维度 | 标准 |
|------|------|
| **定义** | 稳定续租 + 偶尔转介绍 + 关系良好 |
| **识别条件**（满足≥3条） | ① 已续租≥1次；② 过去12个月转介绍≥1个；③ 沟通顺畅、信任度高；④ 按时交租、无纠纷；⑤ 对服务基本满意 |
| **维护策略** | 保持专业服务 + 适当关系维护：每季度1次深度回访（电话/见面）、每月1条有价值信息（非广告）、入住/退租时超值服务、适时提醒转介绍 |
| **维护频率** | 每月≥1次有效触达 |
| **预期产出** | 年度续租率 ≥70%，年度转介绍 ≥1个/人 |
| **精力投入** | 占老客总数30%，投入40%精力 |

#### C级客户——常规维护（约40%）

| 维度 | 标准 |
|------|------|
| **定义** | 正常租住、关系一般、无明显不满但也不主动 |
| **识别条件** | ① 租期≤2年，未续租过；② 无转介绍记录；③ 无纠纷但沟通较少；④ 按时交租 |
| **维护策略** | 标准化维护，不投入额外精力：每季度1次标准回访（微信/短信）、到期前45天启动续租/退租沟通、按需服务、不主动打扰 |
| **维护频率** | 每季度≥1次标准触达 |
| **预期产出** | 年度续租率 ≥50%，年度转介绍 ≥0.3个/人 |
| **精力投入** | 占老客总数40%，投入20%精力 |

#### D级客户——低价值/风险（约20%）

| 维度 | 标准 |
|------|------|
| **定义** | 有纠纷/欠租/投诉记录，或明确不续租 |
| **识别条件**（满足≥1条） | ① 有过欠租/逾期记录；② 有过投诉/纠纷；③ 明确表示不续租；④ 长期联系不上/不回应；⑤ 对服务明确不满 |
| **维护策略** | 风险管控为主：欠租/纠纷类→按公司流程处理，优先保障房东权益；明确不续租→到期前60天启动退租流程，同步找新客；不回应类→保留基本服务通道，不主动投入 |
| **维护频率** | 仅按需触达（到期/纠纷/必要通知） |
| **预期产出** | 无续租/转介绍预期，目标是平稳退租+快速二出 |
| **精力投入** | 占老客总数20%，投入10%精力（主要用于退租流程） |

### 1.3 分层更新机制

| 更新项 | 频率 | 触发条件 |
|--------|------|----------|
| 常规更新 | 每季度1次 | 季度末统一评估，调整层级 |
| 即时更新 | 按需 | 客户发生关键行为时立即调整（转介绍成交→升级；欠租→降级） |
| 新客入层 | 入住满3个月 | 首次分层评估 |
| 客户离层 | 退租后 | 归档至"历史客户池"，保留转介绍通道 |

## 二、老客维护五步法操作流程

### 2.1 五步全景图

```
定期触达 → 需求预判 → 超值服务 → 关系深化 → 价值转化
  (每月)    (每季度)    (按需)     (每半年)     (适时)
  保持      提前发现    建立       从客户      续租+
  存在感    需求变化    亏欠感    变成朋友    转介绍
```

### 2.2 每一步操作标准

#### ① 定期触达

| 维度 | 内容 |
|------|------|
| **频率** | A级：每月2次 / B级：每月1次 / C级：每季度1次 / D级：按需 |
| **触达形式** | 优先级排序：① 语音消息（最亲切）> ② 私聊文字（有价值内容）> ③ 朋友圈互动（点赞评论）> ④ 群发消息（最差，尽量不用） |
| **触达内容** | 三类有价值信息交替发：① 市场信息（区域租金变化/新政解读/省钱技巧）；② 生活关怀（天气提醒/节日祝福/搬家纪念日）；③ 房源信息（新上好房/朋友委托出租——仅对A/B级） |
| **话术示例** | 「最近这个片区租金整体涨了200左右，还好您当时签的早，锁定了好价格。对了，最近住的还舒服吗？有需要帮忙的随时说。」 |
| **常见错误** | ❌ 每次都是"最近有没有朋友要租房"→ 让客户觉得你只想要转介绍 |
| **验收标准** | A/B级客户月度触达率 ≥90%，触达后客户回复率 ≥50% |

#### ② 需求预判

| 维度 | 内容 |
|------|------|
| **频率** | 每季度1次深度回访（A/B级），到期前45天（所有级别） |
| **预判内容** | ① 续租意向：是否续租？有什么不满？② 换房需求：想换更大的/更小的/换地段？③ 买房计划：是否有购房打算？（提前准备退租+可能转化为买卖客户）④ 生活变化：结婚/生娃/换工作/合租变动 |
| **话术示例** | 「住了一年多了，感觉怎么样？有没有什么地方住着不太方便的？或者有换房打算？你直说，我帮你提前规划。」 |
| **常见错误** | ❌ 只在快到期才问续不续租 → 客户可能已经找好下家了 |
| **验收标准** | 到期前60天完成续租意向确认率 ≥80% |

#### ③ 超值服务

| 维度 | 内容 |
|------|------|
| **定义** | 超出客户预期的、不收费的额外服务。目的是让客户产生"欠你人情"的感觉。 |
| **服务清单** | ① 代收快递/代开门（客户出差时）；② 推荐靠谱保洁/维修师傅；③ 帮忙处理物业问题（灯泡坏/漏水/邻居噪音）；④ 搬家时帮忙搭把手；⑤ 提供周边生活攻略（好吃的/好玩的/带娃的）；⑥ 帮忙对接资源（装修/家政/二手交易） |
| **话术示例** | 「你上次说热水器不太好用，我帮你联系了物业，他们明天上午过来修。你在家的话直接开门就行，不在的话我帮你盯着。」 |
| **常见错误** | ❌ 超值服务后马上提转介绍 → 显得功利，人情变交易 |
| **验收标准** | A级客户年度 ≥2次超值服务，B级客户年度 ≥1次 |

#### ④ 关系深化

| 维度 | 内容 |
|------|------|
| **频率** | A级：每季度1次线下见面 / B级：每半年1次 |
| **方式** | ① 请吃饭/喝咖啡（选客户方便的附近）；② 送小礼品（搬家纪念日/生日/节日）；③ 邀请参加活动（社区活动/行业分享）；④ 帮客户解决一个非租房问题（如帮孩子找学校信息） |
| **话术示例** | 「这周六有空吗？请你喝杯咖啡。没什么事，就是好久没见了，聊聊近况。我最近知道一个不错的兴趣班，你家小朋友可能会喜欢。」 |
| **常见错误** | ❌ 约出来就开始推销 → 客户下次不会再出来 |
| **验收标准** | A级客户年度 ≥2次线下深度互动 |

#### ⑤ 价值转化

| 维度 | 内容 |
|------|------|
| **转化类型** | ① 续租转化：到期续租；② 转介绍转化：老客介绍新客；③ 升级转化：换更大的房子/换更好的小区；④ 买卖转化：从租客变买家 |
| **转化时机** | ① 续租：到期前45天开始沟通；② 转介绍：在超值服务/关系深化后自然提出，不硬推；③ 升级：在需求预判中发现换房意向时；④ 买卖：在需求预判中发现购房意向时 |
| **话术示例** | （转介绍——自然引出）「对了，你上次说你们公司新来了好几个同事在找房子。你帮我把这个发给他们呗——我整理了一份这个片区的租房攻略，新人看了能少走很多弯路。」 |
| **常见错误** | ❌ 关系还没到位就硬要转介绍 → 透支信任 |
| **验收标准** | A级客户年度续租率 ≥90%、转介绍率 ≥50%；B级客户年度续租率 ≥70%、转介绍率 ≥30% |

## 三、二出招租五步法时间轴

### 3.1 时间轴全景

```
T-45天          T-30天          T-20天         T-10天         T-0天
  │               │               │              │              │
提前预警        精准匹配        快速带看       高效签约      交接过渡
确认续租意向    筛选意向客      集中安排       锁定客户      完成退租
启动招租准备    定向推送        批量带看       收定签约      新房客入住
```

### 3.2 每一步操作标准

#### ① 提前预警（到期前45天）

| 维度 | 内容 |
|------|------|
| **核心目标** | 确认客户续租意向，拿到明确的"续/不续"答复 |
| **操作步骤** | ① 发续租意向确认消息；② 如续租：确认续租条件（租金是否调整、租期多久）；③ 如不续租：确认退租日期，同步启动招租准备；④ 如犹豫：了解原因，针对性解决 |
| **话术示例** | 「您的合同还有一个半月到期。想跟您确认一下——是继续住，还是到期搬走？如果续的话，房东那边说租金不变。如果不续，您提前跟我说，我帮您安排。」 |
| **验收标准** | 到期前45天完成100%续租意向确认 |

#### ② 精准匹配（到期前30天）

| 维度 | 内容 |
|------|------|
| **核心目标** | 基于房源特点和历史客户数据，找到最可能成交的客户 |
| **操作步骤** | ① 更新房源档案（现有客户退租后的状态+房东新要求）；② 从客户池中筛选匹配客户（需求与房源匹配度≥80%）；③ 生成优先推送名单（TOP10）；④ 制作新的房源推广素材（实拍照片+视频+亮点介绍） |
| **话术示例** | （内部操作，无需对客户说）筛选逻辑：预算±10%、地段匹配、户型匹配、入住时间匹配 |
| **验收标准** | 生成 ≥10人的优先推送名单，推送-回复率 ≥50% |

#### ③ 快速带看（到期前20天）

| 维度 | 内容 |
|------|------|
| **核心目标** | 在退租前完成带看，缩短空置期 |
| **操作步骤** | ① 与现有租客协商带看时间（尊重现有租客，提前预约）；② 集中安排带看（同一时段安排2-3组，提高效率）；③ 执行带看标准化流程（参见《带看标准化手册》）；④ 每次带看后2小时内跟进反馈 |
| **话术示例** | （与现有租客沟通）「XX您好，下个月合同到期了，我这边开始帮房东找下一位租客。您看这周末下午方便吗？我带客户来看一下，大概15分钟就好。不方便的话您说个时间。」 |
| **验收标准** | 带看-意向转化率 ≥40%，空置期 ≤7天 |

#### ④ 高效签约（到期前10天）

| 维度 | 内容 |
|------|------|
| **核心目标** | 锁定客户，完成签约 |
| **操作步骤** | ① 意向客户→收定金锁定（"房东说了先定先得"）；② 准备合同（确认租金/押金/付款方式/入住日期/特殊条款）；③ 签约前再次确认房源状态（现有租客退租后的房屋状况）；④ 签约+收押金租金 |
| **话术示例** | 「这套房现在有三组在看，房东说谁先定就给谁。您如果确定要，先交个定金，我帮您锁住。」 |
| **验收标准** | 从意向到签约 ≤5天，签约-退租衔接无缝（新房客入住日=老客户退租日+整备完成日） |

#### ⑤ 交接过渡（到期日）

| 维度 | 内容 |
|------|------|
| **核心目标** | 老客户顺利退租，新房客顺利入住，房东满意 |
| **操作步骤** | ① 老客户退租：验房→结算押金→收回钥匙→致谢+保持联系（转介绍通道不断）；② 房屋整备：清洁→维修→美化→布置→呈现（整备五项）；③ 新房客入住：交房→签交接单→发入住指南→48小时回访 |
| **话术示例** | （老客户退租致谢）「感谢您这一年多的信任！以后有朋友要租房随时找我。您的押金会在3个工作日内到账，有问题随时联系。」 |
| **验收标准** | 退租-新房客入住间隔 ≤3天（含整备），新房客48小时回访满意率 ≥90% |

## 四、AI提醒配置指南

### 4.1 提醒系统架构

```
AI提醒系统
├── 续租类提醒（到期预警）
├── 维护类提醒（定期触达）
├── 转化类提醒（转介绍时机）
├── 风险类提醒（异常预警）
└── 整备类提醒（退租后流程）
```

### 4.2 提醒配置表

#### 续租类提醒
