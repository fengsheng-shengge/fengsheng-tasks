# 客户解码·用户体验验证方案 v1.0

> 生哥铁律：上线必须用真实用户体验验证正循环，不是"功能能跑"就行，而是"用户用完真的变强了"才算验证通过。

## 一、验证目标

验证客户解码·测学练考评产品的**正循环是否成立**：
1. 用户测出基线 → 学了能懂 → 练了会用 → 考了能过 → 评出成长 → 回到实战真的变强
2. 任何一环断裂，就是产品要改的地方

## 二、正循环验证框架

### 5个核心验证点

| 验证点 | 验证什么 | 通过标准 | 失败信号 |
|--------|---------|---------|---------|
| V1 测→学衔接 | 测完5题后，用户是否清楚自己该学什么 | 80%用户能说出自己的薄弱维度 | 用户说"不知道测完该干嘛" |
| V2 学→练转化 | 学完微课后，用户能否上手练 | 70%用户首次练习完成率>50% | 学完练不了/练完乱填 |
| V3 练→考闭环 | 练习场景和考题是否对齐 | 考题覆盖率>85%练习场景 | 考的没练过/练的考不到 |
| V4 考→评反馈 | 考完评分和摘要是否让用户知道怎么改 | 75%用户认同"评语说中了我" | 用户说"评语太泛/不准" |
| V5 评→用升级 | 拿到评价后，用户是否在实战中行为改变 | 至少3个用户反馈"我用了解码结果，客户沟通不同了" | 无人提实战变化 |

## 三、验证执行计划

### Phase 1：内测验证（6.30上线前，6.26-6.29）

**目标：** 功能链路能跑通，无阻塞性bug

**测试员：** 风声团队内部（生哥/小鱼儿/小酒窝儿/小眼镜）

**测试矩阵：**

| 角色 | 测试路径 | 必做项 |
|------|---------|--------|
| 交易线经纪人 | 测→学(3课)→练(场景01首次置业)→考(交易线20题)→评 | 走完全链路+截图卡点 |
| 租赁线客户经理 | 测→学(3课)→练(场景03学区购房)→考(租赁线20题)→评 | 走完全链路+截图卡点 |
| 生哥 | 全链路+决策人视角 | 验证正循环逻辑是否自洽 |

**输出物：**
- 内测反馈表（每人1份，模板见下方）
- 阻塞性bug清单（0个才能上线）
- 正循环5验证点初判

### Phase 2：种子用户验证（6.30上线后第1周，7.1-7.7）

**目标：** 真实用户体验验证正循环

**种子用户：** 5-10人（小鱼儿安排POC门店经纪人/客户经理）

**验证方式：**

1. **使用追踪（被动采集）**
   - localStorage行为数据3项自动记录（登录次数/完成模块/练考时长）
   - 手动导出后我方分析

2. **主动反馈（结构化采集）**
   - 每完成1个模块，弹1个单选："这个模块对你有帮助吗？✓有 ✓一般 ✓没有"
   - 全链路走完后，5问问卷（见下方）

3. **深度访谈（抽样3人）**
   - 小鱼儿或小酒窝儿电话/语音15分钟
   - 核心问题：走完测学练考评，你下次见客户会做什么不同的事？

### Phase 3：正循环闭环验证（上线后第2-4周，7.8-7.28）

**目标：** 验证"评→用→测"升级闭环

**验证方式：**
- 7天后复测：种子用户7天后重做5题快筛，对比基线变化
- 实战案例征集：至少3个"我用了解码结果谈客户"的真实案例
- NPS评分：≥30算及格，≥50算良好

## 五、验证不通过的处置规则

| 验证点不通过 | 处置 |
|-------------|------|
| V1 测→学衔接断 | 重新设计测评结果→微课推荐的映射逻辑 |
| V2 学→练转化断 | 补充练前引导/降低首次练习门槛 |
| V3 练→考闭环断 | 对齐考题和练习场景覆盖度，补缺考题 |
| V4 考→评反馈断 | 优化摘要生成规则，增加具体行为建议 |
| V5 评→用升级断 | 补充"解码→话术"实战桥接，增加话术模板 |

**原则：** 哪环断修哪环，不全盘推翻。修完后该环必须重新验证通过。
