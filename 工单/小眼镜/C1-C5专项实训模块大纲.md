## 通用设计框架

### 实训vs讲课的本质区别

| 维度 | 传统课程 | 专项实训 |
|------|----------|----------|
| 目标 | 学知识 | 用产品干活 |
| 形式 | 老师讲、学生听 | 任务驱动、实战导向 |
| 产出 | 考试通过 | 产品操作+交付物 |
| 验证 | 考试分数 | 使用次数+留存率+NPS |

### 统一实训结构（4阶段）

```
阶段一：认知启动（1-2天）
  → 建立对产品的正确认知，理解"为什么用"
        ↓
阶段二：上手实操（3-5天）
  → 在真实场景中使用产品，完成核心操作
        ↓
阶段三：实战闭环（1-2周）
  → 用产品产出实际成果，解决真实问题
        ↓
阶段四：行为固化（持续）
  → 养成使用习惯，建立反馈机制
```

### 统一验收标准（4级）

| 层级 | 标准 | 衡量方式 | 对应L等级 |
|------|------|----------|----------|
| L1 课后复述 | 能讲清楚产品是什么、能帮什么 | 3句话描述产品价值主张 | L1 入门 |
| L2 实操交付物 | 能产出规定交付物 | 交付物存在且符合模板 | L2 合格 |
| L3 产品操作 | 能独立完成核心操作 | 产品使用次数+完成率 | L2 合格 |
| L4 行为改变 | 在实际工作中常态化使用 | 30天留存率+NPS | L3 熟练 |

### 统一反馈机制

```
用户操作 → 数据落库（D1） → 计数+1 → 汇总报告
                              ↓
                       发现问题 → 产品迭代 → 课程更新
```

**核心指标**：
- 使用人数（去重计数）
- 操作完成率
- 30天留存率
- NPS净推荐值
- 反馈提交率

## C1：Agent培养师（🔴 P0 · 最优先）

### 1.1 实训定位

| 项目 | 内容 |
|------|------|
| **对应产品** | breeder（fengsheng.tech/breeder） |
| **验证目标** | ≥30人完成实训并持续使用Agent |
| **覆盖角色** | 全体服务者（经纪人/管家/店长/客户经理） |
| **实训性质** | 产品使用能力 + AI训练能力双认证 |
| **人才字典对齐** | E2 AI训练师（L1-L2入门级）、E1 生成式AI应用员（L2场景化应用） |

**核心问题**：风声项目为什么要培养Agent？

> 生哥原话："风声不是卖工具，是帮服务者建立AI协作能力。Agent培养师是风声产品的核心用户群体——他们先用产品、用出效果，才能带动更多人。"
>
> **碳基定位**：Agent培养师 = 用AI放大自身优势的人，不是学技术，是学协作。
>
> **验证逻辑**：30人验证线 = 30个"用产品干活"的人 → 30个真实反馈 → 产品迭代 → 飞轮转起来

### 1.2 实训目标（5条可验证）

| # | 目标 | 验证方式 | 对应能力 |
|---|------|----------|----------|
| T1 | 能用3句话讲清楚"Agent是什么、为什么我要用它" | 课后复述 | L1 认知 |
| T2 | 能为自己的AI设定身份、人设、规则（小鱼儿式） | 交付物评审 | L2 操作 |
| T3 | 能用Agent完成1个真实工作任务（如写方案/做分析/整理信息） | 产品操作日志 | L2 实战 |
| T4 | 能给Agent做日常调优（补充知识、优化Prompt） | 操作记录+反馈 | L2 迭代 |
| T5 | 30天内Agent使用≥10次，形成常态化协作习惯 | 留存率统计 | L3 固化 |

### 1.3 启动条件

| 条件 | 状态 | 说明 |
|------|------|------|
| A1双签完成 | ⬜ 待确认 | C1依赖A1第1-4课内容（身份设定/人设Prompt是基础） |
| breeder产品可用 | ⬜ 待确认 | 产品需具备核心功能：身份设定+知识库+对话 |
| 实训手册初稿 | ✅ 本文档 | 可立即交付 |
| 种子用户招募 | ⬜ 待启动 | 首批30人招募方案需单独设计 |
| 反馈系统D1 | ⬜ 待开发 | 产品使用数据落库，需小豆子排期 |

### 1.4 实训流程（4阶段）

│ 阶段一：认知启动（Day 1）                                     │
├─────────────────────────────────────────────────────────────┤

│                                                              │
│ 验收：                                                      │

                           ↓
┌─────────────────────────────────────────────────────────────┐
│ 阶段二：上手实操（Day 2-3）                                   │
├─────────────────────────────────────────────────────────────┤

│     2. 给Agent取名字（参考小鱼儿命名逻辑）                   │
│     3. 设定身份Prompt（我是谁、我的优势、我的铁律）           │
│     4. 上传我的知识库（工作文档/话术/案例）                   │
│   产出：一个"活的"Agent（能叫出名字、知道我的背景）          │
│                                                              │

│   输入：阶段一列出的真实任务                                 │
│   操作：                                                      │
│     1. 布置第一个任务（写一份工作汇报/整理客户信息/做方案）  │
│     2. 对比：通用AI回答 vs 我的Agent回答                      │
│     3. 调优：根据回答质量补充知识库/优化Prompt                │

│                                                              │
│ Day 3 下午：验收考核                                         │
│   操作：                                                      │
│     1. 展示你的Agent（演示设定+知识库+对话）                │

└─────────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│ 阶段三：实战闭环（Day 4-14）                                  │
├─────────────────────────────────────────────────────────────┤
│ 时长：2周（日常任务+周末复盘）                               │
│ 形式：真实工作任务 + 周反馈                                  │
│ 目标：在真实工作中用Agent产出成果                           │
│                                                              │
│ 任务设计（每周2-3个任务）：                                   │
│                                                              │
│ Week 1 任务：                                                │
│   T1：每日一问（用Agent解决一个工作中的真实问题）             │
│      输入：当天遇到的问题                                    │
│      操作：向Agent提问→记录回答→评价质量                    │
│      产出：7天日志（每天的问题+Agent回答+我的评价）           │
│                                                              │
│   T2：Agent调优日（周末1小时）                               │
│      输入：7天使用中发现的问题                               │
│      操作：根据反馈补充知识库→优化Prompt→测试效果            │
│      产出：调优记录（改了哪里、为什么改、改后效果）           │
│                                                              │
│ Week 2 任务：                                                │
│   T3：我的Agent成果展示                                      │
│      输入：2周内Agent产出的最佳成果                          │

│      产出：案例文档（500字以上，含截图）                     │
│                                                              │
│   T4：同伴互评                                                │
│      操作：交换案例→互相点评→提出改进建议                   │
│      产出：互评记录                                          │
│                                                              │
│ 验收：                                                       │
│   - 14天内Agent使用≥10次（数据验证）                        │
│   - 提交1个完整案例文档                                      │
│   - 参与同伴互评                                             │
└─────────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│ 阶段四：行为固化（Day 15-30+）                                │
├─────────────────────────────────────────────────────────────┤
│ 时长：持续（习惯养成期）                                      │
│ 形式：社群陪伴 + 月度复盘                                    │
│ 目标：Agent使用成为工作习惯，形成常态化协作                  │
│                                                              │
│ 机制设计：                                                   │
│                                                              │
│ 1. 每日打卡群                                                │
│    - 每天早上：用Agent列今日3件工作                          │
│    - 每天晚上：用Agent复盘今日工作                            │
│    - 连续7天打卡→解锁"Agent进阶技巧"资料包                  │
│                                                              │
│ 2. 周五分享会（可选参加）                                    │
│    - 分享本周Agent使用心得                                    │
│    - 讨论踩坑与解决方案                                      │
│    - 优秀案例表彰                                            │
│                                                              │

│    - 30天使用≥20次 + NPS≥50 → 获得"Agent培养师"认证         │
│    - 认证学员自动加入"Agent培养师社群"                      │
│    - 优秀学员有机会成为下一期助教                            │
