## 2. 测/学侧调整

### 2.1 微课实操环节：输入格式变更

| 项目 | 原v1.0设计 | MVP调整 | 实现方式 |
|------|-----------|---------|---------|
| 解码器输入 | 粘贴客户对话文本（5条微信对话） | 结构化表单6字段 | 前端表单组件 |
| 第1课实操 | 场景A：粘贴对话→AI四层拆解；场景B：自己写对话→四层拆解 | 场景A：填6字段表单→出三层解码；场景B：给对话→学员手动提取6字段→填表→出解码 | 场景A前端表单；场景B手动提取+表单 |
| 第2课实操 | demo客户数据→解码器一键出报告 | demo数据预填6字段→解码器出三层报告→学员验证 | JSON预填+一键解码 |
| 第3课实操 | 基于报告写服务话术 | 基于三层报告+话术扩展模块→制定服务策略 | 后端话术扩展+前端展示 |

**关键影响**：实操从"读对话→拆需求"变为"按字段填表→出结果"。学员体感从"AI帮我拆"变成"我把信息喂给AI，AI帮我理"。**需要在微课中增加"如何从客户对话中提取6字段"的教学环节**。

### 2.2 解码报告：三层结构对齐

| 项目 | 原v1.0设计 | MVP调整 | 实现方式 |
|------|-----------|---------|---------|
| 解码模型 | 四层模型（表面需求/真实需求/隐性需求/决策逻辑） | 三层结构 | 后端解码逻辑 |
| 第一层 | 表面需求 | **显性需求**（他说出来的） | 从6字段直接映射 |
| 第二层 | 真实需求+隐性需求 | **隐性需求**（他没说但重要的） | 基于字段组合推理 |
| 第三层 | 决策逻辑 | **需求优先级Top3+决策权重** | 推理+排序 |
| 第四层 | 决策逻辑 | V2再做 | — |
| 话术建议 | 解码报告标准输出 | **扩展模块**，放底部，非强制输出 | 后端可选输出 |

**三层结构输出示例**：
```
━━━ 第一层·显性需求 ━━━
• 预算200万，两居室，学区
• 老人偶尔来住，要有电梯
• 时间：明年孩子上学前

━━━ 第二层·隐性需求 ━━━
• 预算紧张但有弹性空间（"左右"=可上浮10-15%）
• 真实痛点是"老人上下楼"，不只是"有电梯"
• 学区是硬约束，面积可以妥协

━━━ 第三层·需求优先级 ━━━
P0 学区（权重35%）— 硬时限驱动，不可妥协
P1 电梯/适老（权重30%）— 老人需求=情感驱动
P2 预算控制（权重25%）— 有弹性但需算账
P3 面积（权重10%）— 最先可妥协项

━━━ 话术建议（扩展模块）━━━
• 先确认学区划片：避免白看
• 带看时重点展示电梯+老人动线：不是"有电梯"，是"老人从出门到回家全程无障碍"
• 预算对话策略：200万学区2居面积偏小，帮客户做"学区vs面积"取舍分析
```

### 3.3 评分：规则评分方案

**MVP评分方式**：对照标准答案的规则评分，非AI评分。

#### 评分通用框架

| 评分维度 | 权重 | 评什么 | 怎么评 |
|---------|------|--------|--------|
| 第一层·显性需求完整度 | 30% | 6字段信息是否完整提取 | checklist逐项对照，漏一项-5分 |
| 第二层·隐性需求准确度 | 40% | 推理是否准确，是否遗漏关键隐性需求 | 关键推理点checklist，命中得分，遗漏扣分 |
| 第三层·优先级合理性 | 30% | Top3排序+权重是否合理 | P0必须正确（错则该维度0分），P1/P2允许±1排序偏差 |

#### 10场景评分规则速查

| 场景 | 第一层checklist（30%） | 第二层关键推理点（40%） | 第三层P0硬约束（30%） |
|------|----------------------|----------------------|---------------------|
| **T1** 首次带看 | 预算/学区/电梯/老人/时间5项 | 预算弹性/老人真实痛点/面积可妥协 | P0=学区 |
| **T2** 改善型 | 当前痛点/改善动机/预算3项 | 隐性升级动力/情感驱动vs理性驱动/老人是关键推动者 | P0=老人上下楼 |
| **T3** 犹豫比价 | 3套评价/比价维度/犹豫点3项 | 真正在比什么/截胡风险/决策标准未对齐 | P0=识别真实比较维度 |
| **T4** 代际购房 | 三方诉求/出资人/使用者3项 | 支付者≠决策者≠使用者/冲突点定位/谁是一票否决 | P0=识别一票否决者 |
| **T5** 风险预警 | 房产风险/客户意向/时间3项 | 零容忍：风险识别不完整→直接0分 | P0=风险识别（零容忍） |
| **R1** 新租客匹配 | 预算/通勤/独居/阳台4项 | 预算刚性下排序/通勤是硬约束/品质期望vs预算冲突 | P0=通勤 |
| **R2** 续租谈判 | 租客诉求/房东态度/市场价3项 | 空置成本计算/优质租客价值/双赢空间 | P0=保住优质租客 |
| **R3** 押金争议 | 合同条款/双方主张/检查记录3项 | 零容忍：不依法归责→直接0分/合同优先/证据为王 | P0=依法判定（零容忍） |
| **R4** 合租矛盾 | 3方投诉/合租规则2项 | 表层冲突vs深层需求/边界问题vs习惯问题 | P0=识别根因不在表层 |
| **R5** 企业集中租赁 | 企业需求/员工需求/合规3项 | 企业vs员工需求拆分/预算人均约束/发票需求=合规硬约束 | P0=合规性 |

**零容忍场景**：T5风险预警、R3押金争议——对应维度直接0分，不参与总分计算。

#### 评分实现

| 项目 | 实现方式 |
|------|---------|
| 评分函数 | 前端JS，输入=学员答案JSON + 标准答案JSON，输出=各维度分数+总分 |
| checklist对照 | 学员答案中是否包含标准答案关键词/近似表达 |
| P0硬约束 | if P0≠标准答案 → 该场景总分=0 |
| 人工抽检 | 随机10%场景标记"需人工复核"，导出学员答案+自动评分→管理员线下核 |

### 4.1 情景判断题：规则评分方案

| 双线分化 | 交易20题+租赁20题独立 | 同左 | 两套题集 |

**规则评分实现**：
```javascript
// 情景判断题评分映射（示例）
const scoreMap = {
  'A': 3,  // 解码到第3层/决策逻辑层
  'B': 2,  // 解码到第2层/隐性需求层
  'C': 1,  // 只到第1层/显性需求层
  'D': 0   // 误读/隐瞒风险
};

// 零容忍项检查
function checkZeroTolerance(questionId, answer, line) {
  const zeroToleranceQuestions = line === 'trade' ? ['T_Q04', 'T_Q12'] : ['R_Q07', 'R_Q15'];
  if (zeroToleranceQuestions.includes(questionId) && answer === 'D') {
    return { isZeroTolerance: true, dimension: getDimension(questionId) };
  }
  return { isZeroTolerance: false };
}
```

### 4.2 实操评分：标准答案对照+人工抽检10%

## 7. 双线分化具体设计

### 7.1 小区/商圈（文本输入）

| 维度 | 交易线（A1） | 租赁线（B4） |
|------|-------------|-------------|
| 关注重点 | 学区划片、升值潜力、小区品质、物业水平 | 通勤半径、生活配套、合租友好度 |
| 输入提示 | "请输入目标小区或商圈名称" | "请输入工作地点或偏好区域" |
| 解码推理差异 | 学区→确认划片+入学年限；商圈→判断升值空间 | 通勤→计算通勤时间；区域→匹配租金水平 |
| 隐性需求挖掘 | 说"望京"可能真要"望京学区"；说"不太偏"=预算有限但不想承认 | 说"地铁附近"=通勤是硬约束；说"别太远"=有具体通勤时间上限 |

### 7.2 预算范围（选择）

#### 交易线分档

| 档位 | 选项 | 解码推理 |
|------|------|---------|
| 刚需低 | 150万以下 | 预算刚性，面积优先妥协，远郊/老破小可接受 |
| 刚需中 | 150-300万 | 学区vs面积博弈区，需帮客户做取舍分析 |
| 刚需高/改善 | 300-500万 | 有选择空间，关注品质+配套，隐性需求可能是"一步到位" |
| 改善/高端 | 500-1000万 | 品质驱动，位置+学区+品质三重约束 |
| 高端 | 1000万以上 | 个性化需求为主，解码重点在生活方式匹配 |

#### 租赁线分档

| 档位 | 选项 | 解码推理 |
|------|------|---------|
| 经济型 | 1500元/月以下 | 预算刚性，合租为主，核心需求=有地方住 |
| 性价比 | 1500-3000元/月 | 合租可接受好位置，整租需远一点，品质vs位置博弈 |
| 品质型 | 3000-5000元/月 | 整租为主，关注品质+配套，可能愿为通勤/品质付溢价 |
| 舒适型 | 5000-8000元/月 | 整租+精装，生活方式驱动，关注社区品质 |
| 高端 | 8000元/月以上 | 个性化需求为主，服务品质是关键 |

### 7.3 需求类型（多选）

#### 交易线选项

| 选项 | 解码推理 |
|------|---------|
| 首次购房 | 刚需驱动，预算敏感，信息需求大，需要"被引导"感 |
| 改善换房 | 有经验但期望值高，对增量价值敏感，卖旧买新有时间压力 |
| 学区房 | 硬约束强（入学年限），决策链含孩子因素，面积常妥协 |
| 投资房产 | ROI驱动，关注租售比/升值空间，决策理性但需数据支撑 |
| 养老房 | 适老设计+医疗配套+一楼/电梯，决策人可能含子女 |
| 婚房 | 情感驱动+预算常含父母资助，决策人复杂（夫妻+父母） |

#### 租赁线选项

| 选项 | 解码推理 |
|------|---------|
| 整租自住 | 品质+隐私需求，预算弹性相对大 |
| 合租分摊 | 预算刚性+社交需求，室友匹配度是隐性关注 |
| 企业宿舍 | 合规性+发票+集中管理，决策人=HR非居住者 |
| 短租过渡 | 灵活性优先，家具家电齐全加分，不关心长期维护 |
| 品质长租 | 生活方式驱动，关注社区服务+维修响应+续租稳定性 |
| 学区租房 | 为入学资格租房，落户/社保等政策是硬约束 |

### 7.4 核心诉求（多选）

#### 交易线选项

| 选项 | 解码推理 |
|------|---------|
| 学区教育资源 | 硬约束，入学年限=时间线驱动，常与面积冲突 |
| 通勤便利 | 每日体验，刚性需求，但地铁沿线溢价高 |
| 升值空间 | 投资属性，需数据支撑（历史成交价+规划利好） |
| 居住品质 | 改善驱动，关注物业/绿化/车位/户型 |
| 适老适幼 | 多代同住需求，电梯+医疗+学区三重约束 |
| 价格最优 | 预算驱动，需帮客户定义"最优"=性价比而非最低价 |

#### 租赁线选项

| 选项 | 解码推理 |
|------|---------|
| 通勤最短 | 硬约束，决定搜索半径，溢价可接受 |
| 预算最优 | 租金刚性，位置/面积/品质需妥协 |
| 品质居住 | 精装/家电/社区，溢价支付意愿强 |
| 灵活退租 | 预期不确定性，关注违约金/转租条款 |
| 安全合规 | 合同保障+押金安全+房东靠谱度 |
| 配套齐全 | 生活便利性（超市/医院/学校），合租人共享需求 |

### 7.5 决策人（单选）

#### 交易线选项

| 选项 | 解码推理 |
|------|---------|
| 我自己 | 个人决策，预算自主，关注点集中 |
| 夫妻共同 | 双人决策需对齐，关注点可能不同（一方重学区一方重品质） |
| 父母出资 | 支付者≠使用者，父母关注安全/保值，年轻人关注品质/位置 |
| 全家共识 | 多方博弈，需识别一票否决者，沟通成本最高 |
| 投资决策 | 理性驱动，需数据支撑（租售比/回报周期） |

#### 租赁线选项

| 选项 | 解码推理 |
|------|---------|
| 我自己 | 个人整租/合租，决策自主 |
| 合租共决 | 合租人需对齐规则，冲突预防是关键 |
| 企业决定 | HR代租，需求来自组织而非个人，合规性>舒适度 |
| 父母赞助 | 支付者关心安全/性价比，租客关心位置/自由度 |
| 房东选择 | 反向场景：房东选租客，关注租客靠谱度/租期稳定性 |

### 7.6 时间紧迫度（选择）

#### 交易线选项

| 选项 | 解码推理 |
|------|---------|
| 1个月内 | 极度紧迫，决策压力→可能冲动/被截胡，需快速锁定+风险提示 |
| 3个月内 | 紧迫，有选择空间但不可拖延，需高效筛选 |
| 半年内 | 适中，可从容看房比价，但学区有入学截止线 |
| 1年内 | 不急，观望为主，需持续维护+价值输出防流失 |
| 随缘/无明确时间 | 最低紧迫度，最易流失，需创造紧迫感（好房不等人/政策变化） |

#### 租赁线选项

| 选项 | 解码推理 |
|------|---------|
| 即住/本周 | 极度紧迫，选择面窄，需快速匹配+备选方案 |
| 2周内 | 紧迫，有一定筛选空间，效率优先 |
| 1个月内 | 适中，可看多套比价，关注性价比 |
| 季度内 | 不急，可提前锁定优惠价/新房源 |
| 续租/无明确时间 | 维系为主，续租谈判/升级推荐场景 |
