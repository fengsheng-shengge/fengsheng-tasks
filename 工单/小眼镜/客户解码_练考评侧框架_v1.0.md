## 一、练/考/评在体系中的位置

```
测（入学诊断）→ 学（3节微课）→ 练（解码器实操）→ 考（阶段测评）→ 评（成长档案）
        已完成 ✅              已完成 ✅          本文档 ←────────────────────
```

**闭环逻辑**：测告诉你"你在哪"，学告诉你"怎么做"，**练让你"做到"，考验证"做到没"，评看见"走了多远"**。

## 二、练：解码器实操

### 2.1 设计逻辑

**金句**：**"练不是练工具，是练你读客户的眼力。解码器是你的放大镜，不是你的望远镜——它放大你已经看到的，不是替你看。"**

| 项目 | 说明 |
|------|------|
| 练什么 | 输入客户信息 → 出解码报告 → 基于报告出服务策略 |
| 不是什么 | 不是练AI操作、不是练Prompt写法、不是练软件界面 |
| 本质 | 用真实客户场景练"客户洞察"能力，解码器是加速器不是替代品 |
| 与学的关系 | 学=学方法论（四层模型/解码报告/四步闭环），练=用方法论+工具做出来 |
| 与考的关系 | 练=日常训练，考=达标验收，练不等于考但必须通过练才能过考 |
| 与产品的关系 | 练的每一步就是解码器产品功能的使用流程，课程就是产品使用说明书 |

**难度递进逻辑**：

| 梯度 | 定义 | 信息完整度 | 客户复杂度 | 服务策略要求 |
|------|------|-----------|-----------|------------|
| **L0→L1入门练** | 能走通流程 | 高（结构化信息已给出） | 低（单一需求，决策链短） | 能按报告执行 |
| **L1→L2进阶练** | 能判断取舍 | 中（需自己提取关键信息） | 中（多重需求，有冲突） | 能做优先级排序 |
| **L2巩固练** | 能应变创新 | 低（模糊信息/对话片段） | 高（多方博弈/隐性需求深） | 能预判+创新方案 |

## 三、考：阶段测评

### 3.1 设计逻辑

**金句**：**"考不是问你答对了没有，是问你解码到了第几层。答对是记忆，解码到才是能力。"**

| 项目 | 说明 |
|------|------|
| 考什么 | 情景判断题（认知层）+ 实操评分（行为层），双轨制 |
| 不是什么 | 不是知识背诵、不是AI操作考试、不是标准答案匹配 |
| 核心逻辑 | 测入门槛→考出水平；考是测的升级版，维度一致但深度不同 |
| 与测的关系 | 测=5题3分钟快筛（入学的温度计），考=20题+实操（升级的体温计） |
| 与评的关系 | 考产出档位结果，评记录档位变化曲线 |

**测→考的升级关系**：

| 对比项 | 测（入学诊断） | 考（阶段测评） |
|--------|-------------|-------------|
| 目的 | 测起点 | 验进阶 |
| 时长 | 3分钟 | 20分钟 |
| 题量 | 5题快筛 | 20题情景判断+1次实操 |
| 深度 | 快速定位档位 | 精确评估各维度+验证实操能力 |
| 输出 | 档位+薄弱维度 | 档位+四维度评分+实操等级+升级建议 |
| 频次 | 入学1次 | 每完成1阶段可考1次 |

### 3.2 双轨制考卷设计

#### 轨道1：情景判断题（认知层）

**题型**：给定业务场景，选择最佳应对策略（不是选"对错"，是选"解码深度"）

**评分逻辑**：

| 选项等级 | 解码深度 | 对应档位 | 得分 |
|---------|---------|---------|------|
| A级 | 解码到第4层（决策逻辑），预判+创新 | L2 | 3分 |
| B级 | 解码到第2-3层（真实/隐性需求），有依据 | L1 | 2分 |
| C级 | 只解码到第1层（表面需求），凭感觉 | L0 | 1分 |
| D级 | 误解读/隐瞒风险 | L0强制 | 0分 |

**双线分化**：

| 维度 | 交易线题量 | 租赁线题量 | 说明 |
|------|-----------|-----------|------|
| D1客户识别 | 5题 | 5题 | 场景不同，维度一致 |
| D2需求挖掘 | 5题 | 5题 | 交易线侧重购房决策链，租赁线侧重租住匹配 |
| D3信任建立 | 5题 | 5题 | 交易线侧重风险预警建信，租赁线侧重费用透明建信 |
| D4场景应对 | 5题 | 5题 | 交易线侧重犹豫/比价/截胡，租赁线侧重争议/拖欠/续租 |
| **合计** | **20题** | **20题** | 通用题8道+分化题12道 |

**示例（交易线D2需求挖掘）**：

> 客户看了3套房，对每套的评价都是"还行吧"。你判断他真实情况最可能是？
>
> - A. 3套房都没命中他的核心需求，他不好意思直说 → 主动问"这3套最不满意的是什么"（解码到第3层·隐性需求）→ **3分**
> - B. 他在比较性价比，需要你帮他做决策 → 列对比表（解码到第2层·真实需求）→ **2分**
> - C. 他就是犹豫型，多看几套就好了 → 继续推房源（只看到第1层·表面）→ **1分**
> - D. 他说"还行"就是还行，别多想 → 忽略信号 → **0分**

**零容忍项**（与测一致，考中强化）：

| 零容忍行为 | 交易线 | 租赁线 |
|-----------|--------|--------|
| 隐瞒风险 | 发现房产风险不告知客户 | 发现合同陷阱不告知租客 |
| 误导客户 | 虚假承诺学区/升值 | 虚假承诺免押/随时退 |
| 选边站 | 只帮业主/只帮买家 | 只帮房东/只帮租客 |

**触发零容忍 → 该维度直接判0分，不参与档位计算，需补考。**

#### 轨道2：实操评分（行为层）

| 项目 | 说明 |
|------|------|
| 考法 | 给定1个综合场景 → 用解码器完成完整流程 → 提交解码报告+服务策略 |
| 时长 | 15分钟 |
| 评分 | 四维度评分表（见下） |
| 与练的区别 | 练=可重做无时限，考=限时限次（每月最多2次） |

**实操评分表**：

| 维度 | L0本能型（1分） | L1标准型（2分） | L2专业型（3分） |
|------|---------------|---------------|---------------|
| **D1客户识别** | 只看到表面信息，画像模糊 | 能提取关键信息，画像基本准确 | 画像精准，识别决策链和隐藏角色 |
| **D2需求挖掘** | 只列出客户说的需求 | 能区分表面需求和真实需求 | 识别隐性需求，排出优先级 |
| **D3信任建立** | 话术生硬或推销感强 | 话术有依据但不够自然 | 话术基于报告，预判顾虑，客户视角 |
| **D4场景应对** | 被动等客户提问 | 能回应常见顾虑 | 预判风险主动应对，有备选方案 |

### 3.3 档位升级机制

**升级条件**：

| 升级路径 | 情景判断题要求 | 实操评分要求 | 附加条件 |
|---------|-------------|-----------|---------|
| L0 → L1 | 总分≥40分（满分60）+ 无零容忍项 | 4维度均≥L1 | 完成微课1-2节+至少2次练习 |
| L1 → L2 | 总分≥50分 + 无零容忍项 + D2D4均≥A级 | 4维度均≥L2 | 完成微课3节+至少5次练习+1次真实客户应用 |
| L2 → L3 | MVP暂不开放 | — | 待v2.0定义 |

**降级保护**：考不过不降级，保持原档位，可补考。

**升级流程**：

```
学员申请升级考
    → 系统检查前置条件（课+练）
        → 不满足 → 提示补什么
        → 满足 → 进入考试
            → 轨道1：情景判断20题（5分钟）
            → 轨道2：实操1个综合场景（15分钟）
                → 双轨评分
                    → 达标 → 档位升级 → 更新成长档案
                    → 未达标 → 保持原档位 → 30天后可补考
```

### 3.4 考与产品功能映射

| 考的环节 | 产品侧功能 | 映射关系 | 待确认 |
|---------|-----------|---------|-------|
| 情景判断题 | 在线答题系统 | 题库存储+出题逻辑+自动评分 | ⚠️ 题库方案 |
| 实操场景 | 解码器综合场景 | 给定场景→学员操作解码器→提交产出 | ⚠️ 场景数据 |

| 考试限制 | 频次控制 | 每月2次，防刷分 | ⚠️ 限制逻辑 |
| 考试入口 | 课程页/个人中心 | 学完课后出现"申请升级考"按钮 | ⚠️ 入口位置 |

### 3.5 MVP方案

## 四、评：成长档案

### 4.1 设计逻辑

**金句**：**"你看不见成长，就不相信成长。成长档案就是把'我感觉好像进步了'变成'你看，你确实进步了'。"**

| 项目 | 说明 |
|------|------|
| 评什么 | 学习曲线+档位变化+真实应用记录+AI使用习惯 |
| 不是什么 | 不是成绩单、不是排行榜、不是证书展示页 |
| 本质 | 可视化学员从"凭感觉"到"专业型"的完整成长路径 |
| 可视化参照 | 健身App（Keep/Peloton）——数据驱动+情感激励+社交货币 |
| 商业化衔接 | 评=付费层的核心钩子（免费看概览，付费看详情+AI建议） |

### 4.2 成长档案四模块

#### 模块1：档位雷达图

```
        D1客户识别
           ★★★
          /    \
D4场景应对 ★★    ★★ D2需求挖掘
          \    /
           ★★★
        D3信任建立

当前档位：L1 标准型
上次档位：L0 本能型
升级时间：2026.07.15
```

| 展示项 | 免费层 | 付费层 |
|--------|-------|-------|
| 四维度雷达图 | ✅ 当前档位+维度等级 | ✅ +历史雷达图叠加对比 |
| 档位标签 | ✅ 当前档位徽章 | ✅ +升级时间线 |
| 薄弱维度提示 | ✅ "D2需求挖掘需强化" | ✅ +AI个性化练习建议 |

#### 模块2：学习曲线

| 展示项 | 免费层 | 付费层 |
|--------|-------|-------|
| 练习次数曲线 | ✅ 最近7天练习次数 | ✅ 全部历史+趋势预测 |
| 正确率趋势 | ❌ | ✅ 各维度正确率变化曲线 |
| 用时趋势 | ❌ | ✅ 同类场景用时变化（越做越快=越熟） |
| 里程碑标记 | ✅ "首次完成L1练习"等关键节点 | ✅ +AI分析"你最擅长/最需提升的场景类型" |

**学习曲线示例**：

```
             1-2课       L1→L2
```

#### 模块3：真实应用记录

| 记录项 | 来源 | 免费层 | 付费层 |
|--------|------|-------|-------|
| 解码器使用次数 | 解码器使用日志 | ✅ 总次数 | ✅ 按日/周/月+使用时段分布 |
| 解码报告产出 | 解码器输出 | ✅ 最近1份 | ✅ 全部历史+搜索 |
| 服务策略采纳 | 学员自评 | ❌ | ✅ 记录"用了/没用"+原因 |
| 客户反馈 | 学员手动录入 | ❌ | ✅ "客户说'你真懂我'"等关键反馈 |
| 成交关联 | 学员手动关联 | ❌ | ✅ 解码→服务→成交的完整链路 |

**⏳ 数据回流前提**：解码器使用日志能回流到成长档案——**待产品侧确认数据接口**。

#### 模块4：AI使用习惯画像

| 画像维度 | 测什么 | 为什么重要 |
|---------|--------|-----------|
| 使用频率 | 每周用解码器几次 | 频率=习惯养成进度（切入期→习惯期的核心指标） |
| 使用深度 | 每次只看报告还是看完报告+话术建议 | 深度=是否从"看结果"进化到"用结果行动" |
| 场景覆盖 | 5种场景都练了还是只练1种 | 覆盖=能力广度（习惯期→伙伴期的关键） |
| 人工修正率 | 解码报告出来后你改了多少 | 修正率低=AI还没懂你，修正率过高=你可能没读懂报告 |

**三阶段对应画像特征**：

| 阶段 | 使用频率 | 使用深度 | 场景覆盖 | 人工修正率 |
|------|---------|---------|---------|-----------|
| 切入期 | 偶尔用（<3次/周） | 只看报告 | 1-2种场景 | 高（>50%）或极低（<5%，盲信AI） |
| 习惯期 | 经常用（3-5次/ week） | 报告+话术 | 3-4种场景 | 适中（20-40%，有判断力） |
| 伙伴期 | 每个客户都用 | 全链路 | 5种+自创场景 | 低且精准（10-20%，只改关键点） |

## 五、测→学→练→考→评 全链路闭环

### 5.1 闭环流程

```
入学诊断（测）
    → 档位L0/L1/L2 + 薄弱维度
    → 推送学习路径
        ↓
微课学习（学）
    → 第1课：四层模型（读客户）
    → 第2课：解码报告（解码客户）
    → 第3课：四步闭环（用解码结果服务客户）
        ↓
解码器实操（练）
    → 5个场景梯度练习（L0→L1→L2）
    → 每次练习产出：解码报告+服务策略
    → 练习记录→回流成长档案
        ↓
阶段测评（考）
    → 轨道1：情景判断20题（认知层）
    → 轨道2：实操1个综合场景（行为层）
    → 双轨评分→档位更新→回流成长档案
        ↓
成长档案（评）
    → 可视化：雷达图+学习曲线+应用记录+AI习惯
    → 输出：档位变化+薄弱维度+AI成长建议
    → 衔接：评的结果→指导下一步学/练/考
        ↓
    【闭环回到"学"或"练"——评告诉你还差什么，继续学/练/考】
```

### 5.2 各环节档位流转

| 起始档位 | 测后推送 | 学后应达 | 练后应达 | 考后验证 | 评后反馈 |
|---------|---------|---------|---------|---------|---------|
| L0 | 微课1→2→3全部 | L1认知 | L1实操 | L1认证 | "你在进步，D2D4需强化，继续练" |
| L1 | 微课2→3（跳过1） | L1→L2认知 | L1→L2实操 | L1或L2认证 | "D1D3已达L2，D2D4还在L1，针对性练" |
| L2 | 直接进练 | L2巩固 | L2熟练 | L2认证 | "基础扎实，建议真实客户实战+申请L3" |

### 5.3 三阶递进闭环

| 阶段 | 测 | 学 | 练 | 考 | 评 | 体感 |
|------|---|---|---|---|---|------|
| **切入期** | 5题快筛 | 3节微课 | L0→L1场景（2个） | 5题复用 | 档位标签 | "原来可以这样读客户" |
| **习惯期** | — | 回顾微课 | L1→L2场景（5个） | 20题+实操 | 完整档案 | "每个客户我都先用解码器" |
| **伙伴期** | — | — | L2场景+真实客户 | L2认证+实战案例 | AI使用习惯画像 | "AI是我的搭档，不只是工具" |
