## 一、整体架构

```
用户进入 fengsheng.tech
    → 身份选择（经纪人 / 客户经理）
    → 入学诊断（测）—— 5题3分钟 → 档位+薄弱维度+学习路径
    → 微课学习（学）—— 3节×30min → 每节有实操+产品衔接
    → 解码器实操（练）—— 真实客户场景 → 解码报告 → 话术输出
    → 阶段测评（考）—— 情景题+实操评分 → 档位更新
    → 成长档案（评）—— 学习曲线+档位变化+真实应用记录
```

**测/学侧先行，练/考/评侧等产品功能边界确认后对齐。**

## 二、测：入学诊断

### 2.1 诊断逻辑

| 项目 | 说明 |
|------|------|
| 目标 | 测"你读懂客户的能力在哪个档"，不是测AI能力 |
| 时长 | 3分钟（5题快筛） |
| 输入 | 身份选择 + 5道情景判断题 |
| 输出 | 客户洞察档位（L0/L1/L2）+ 薄弱维度 + 推荐学习路径 |
| 底层 | 人才字典A1/A2/B1/B4"客户洞察"相关技能点 |

### 2.2 四大诊断维度

| 维度 | 测什么 | 交易线重点 | 租赁线重点 |
|------|--------|-----------|-----------|
| **D1 客户识别** | 快速判断客户类型和真实需求 | 购房动机（刚需/改善/投资）、家庭阶段 | 租客画像（通勤/学区/性价比）、租期预判 |
| **D2 需求挖掘** | 从模糊表达中挖出真实诉求 | 预算弹性、决策权判断、隐性需求 | 合租/整租偏好、特殊需求、续租意愿 |
| **D3 信任建立** | 首次接触建立专业信任 | 风险主动提醒、不推销先诊断、价格透明 | 费用说明透明、维修响应承诺、退租流程提前说 |
| **D4 场景应对** | 典型业务场景正确判断 | 犹豫/比价/截胡/家属意见不一 | 押金争议/拖欠/涨租/提前退租 |

### 2.3 档位定义

| 档位 | 特征 | 学完应达到 |
|------|------|-----------|
| **L0 本能型** | 凭感觉做事，没有方法论 | → L1 |
| **L1 标准型** | 能按流程做，但不灵活 | → L2 |
| **L2 专业型** | 能独立判断，会应变 | → 巩固+进练 |
| **L3 专家型** | 能教别人，能创新 | MVP暂不覆盖 |

### 2.4 MVP题库（5题快筛）

**通用题2题**（双线共用）：
- Q1 [D1客户识别]：客户说"找个不错的房子"，最佳第一反应？
  - ✓ 先诊断再推荐（3分）| △ 只问硬条件/二选一（2分）| ✗ 上来就推（1分）
- Q2 [D3信任建立]：客户嫌价格高，怎么回应最建信任？
  - ✓ 承认感受+给数据+给方案（3分）| △ 有服务意识没依据（2分）| ✗ 否定感受/交定价权（1分）

**交易线3题**（A1经纪人）：
- Q3 [D2需求挖掘]：夫妻看房关注点不同，怎么判断？
  - ✓ 识别双人决策不同关注点（3分）| △ 有拆解但优先级可能错（2分）| ✗ 只看表面/粗暴试探（1分）
- Q4 [D4场景应对]：发现房屋有抵押未解清，怎么做？
  - ✓ 主动提醒风险+安全路径（3分）| △ 方向对但信息不完整（2分）| ✗ **零容忍**：隐瞒风险直接判L0
- Q5 [D1+D2综合]：代际购房看了一个月没定，真实情况？
  - ✓ 三重确认叠加（支付者≠决策者≠使用者）（3分）| △ 部分正确（2分）| ✗ 简单归因（1分）

**租赁线3题**（B4客户经理）：
- Q3' [D1客户识别]：低预算+地铁附近，第一反应？
  - ✓ 预算约束下先精准定位筛选条件（3分）| △ 细化了交通但忽略其他维度（2分）| ✗ 直接匹配/否定预期（1分）
- Q4' [D4场景应对]：退租押金争议，怎么处理？
  - ✓ 合同+证据+中立（3分）| △ 有调解意识缺依据（2分）| ✗ 推卸/选边站（1分）
- Q5' [D2需求挖掘]：优质租客续租要求降价，怎么判断？
  - ✓ 用数据帮房东算账（降vs空置哪个亏更多）（3分）| △ 没查数据直接判断（2分）| ✗ 传话筒/忽略空置成本（1分）

### 2.5 评分与输出

**快筛评分**：

| 总分 | 档位 | 推送策略 |
|------|------|---------|
| 13-15 | L2 专业型 | 直接进「练」——你有方法论，实战巩固 |
| 9-12 | L1 标准型 | 推微课2+3——会做但不灵活，补方法论+工具 |
| 5-8 | L0 本能型 | 从微课1开始——需要建立方法论底子 |

**薄弱维度输出**：每维度单独计分，低于2分标记"需强化"，学习路径优先推送对应模块。

**零容忍项**：隐瞒风险（Q4选A/D）→ 直接判L0，无论总分多少。

【v1.1调整】基于8项MVP决策，原6项待定问题全部已确认：

| # | 需求 | 课程侧要求 | v1.1确认方案 |
|---|------|-----------|-------------|
| T1 | 题库存储 | 10题（5通用+5双线分化），支持后续扩到40题 | JS硬编码嵌入前端（D6决策：JSON文件嵌入） |
| T2 | 出题逻辑 | MVP固定5题；v2自适应 | MVP固定5题，前端按身份切换后3题 |
| T3 | 评分计算 | 前端即可（简单加权） | 前端JS函数，规则评分（D5决策） |
| T4 | 档位展示 | 页面展示档位+维度雷达图 | 页面展示：Canvas/ECharts雷达图，数据读localStorage（D4决策） |

## 三、学：3节微课

### 3.1 课程设计底层

【v1.1调整】解码模型从四层调整为三层，课程主线同步更新：

```
课程主线 = 业务流程（读客户 → 解码客户 → 用解码结果服务客户）
AI = 交互过程中的自然能力（嵌入每个环节的提效工具）
解码器 = 业务流程中的一个环节（不是"学工具"，是"做业务时顺手用"）
每节课 = 想明白时刻(2min) + 方法论(10min) + 实操(15min) + 课后(3min) = 30min
```

**课程风格**：借鉴得到/罗振宇/刘润——知识转译（复杂东西大白话讲透）+ 底层逻辑（案例→模型→公式）+ 金句驱动认知升级

### 3.2 第1课：客户到底在说什么？

| 项目 | 内容 |
|------|------|
| **学习目标** | 能从客户的信息中判断出他真正要什么 |
| **想明白时刻** | "经纪人老张带看20次才成交，小李3次就签了。区别不是小李嘴甜，是小李会'读人'。" |
| **核心模型** | **客户解码三层模型** 【v1.1调整：原四层→三层】 |
| **验收标准** | L1能说出三层分别是什么；L2能对客户信息做三层拆解 |

【v1.1调整】**客户解码三层模型**（原四层→三层，对齐X3输出Schema）：

| 层级 | 含义 | 输出字段 | 示例 |
|------|------|---------|------|
| 第一层·显性需求 | 他说出来的 | items[].content + items[].source | "预算200万，两居室，学区" |
| 第二层·隐性需求 | 他没说但重要的 | items[].content + items[].signal + items[].confidence | "预算有弹性，真实痛点是老人上下楼" |
| 第三层·需求优先级 | 哪个最关键 | top3[].rank + .concern + .weight + .explanation | "P0学区(35%)—硬时限驱动" |
| — | — | decisionScore.value + .label + .interpretation | "72分·中等决策力—需强化隐性需求识别" |

**金句**：**"客户说的不是需求，是他以为的需求。你的工作是找到他不知道自己知道的那个需求。"**

【v1.1调整】**实操环节**（15min）——输入方式从"粘贴对话"改为"6字段表单"：

- 场景A：用解码器填写6字段表单 → AI输出三层解码 → 学员对比自己的判断
  - 新增教学环节："如何从客户对话中提取6字段"——给一段客户对话，学员手动提取6字段信息，再填表出解码
- 场景B：给一段客户对话 → 学员先手动提取6字段 → 填表 → 出三层解码 → 对比自己的提取和AI的解码
- **AI嵌入**：解码器的三层输出 = 三层模型的放大镜，不是"学用工具"，是"读客户时有个更快帮手"

【v1.1调整】**课后作业**（适配手动导出）：
```
📝 第1课课后作业
任务：下次接待客户，按6字段提取信息，用解码器出三层报告
交付：
1. 你提取的6字段内容（复制粘贴）
2. 解码器三层输出（点"保存到成长档案"→复制摘要→粘贴到成长档案-学习记录）
3. 你的自评：解码报告准不准？哪里需要修正？
→ 粘贴到成长档案-学习记录
```

### 3.3 第2课：3秒读懂客户——解码报告实战

| 项目 | 内容 |
|------|------|
| **学习目标** | 能用解码器3秒出客户解码报告，基于报告制定服务策略 |
| **想明白时刻** | "同一个客户，A说'预算有限偏刚需'，B说'他是帮孩子买的改善型，决策权在孩子'。谁签了？B。因为B读对了。" |
| **核心模型** | **客户分类解码表 + 解码报告三层结构** 【v1.1调整：原"三要素"→"三层结构+话术扩展"】 |
| **验收标准** | L1能读懂三层解码报告；L2能基于报告制定差异化服务策略 |

**客户分类解码表**（双线分化）：

| 维度 | 交易线（A1） | 租赁线（B4） |
|------|-------------|-------------|
| 决策类型 | 独立/家庭/代际决策 | 个人/合租/企业决策 |
| 时间敏感度 | 紧急（置换）/不急（观望） | 即住/月内/季内 |
| 价格弹性 | 刚需低弹性/改善有弹性/投资看ROI | 预算刚性/品质溢价/合租分摊 |
| 信息需求 | 学区/税费/贷款/升值空间 | 通勤/配套/合租规则/退租条件 |
| 风险关注 | 查封/抵押/学区政策变化 | 押金安全/房东靠谱度/合同陷阱 |

【v1.1调整】**解码报告三层结构**（原"三要素"→三层+话术扩展，对齐X3 Schema）：

| 层级 | 输出内容 | 字段定义 | 说明 |
|------|---------|---------|------|
| 第一层·显性需求 | 客户说出来的 | items[].content, items[].source | 从6字段直接映射 |
| 第二层·隐性需求 | 客户没说但重要的 | items[].content, items[].signal, items[].confidence | 基于字段组合推理 |
| 第三层·需求优先级 | Top3+权重+解释 | top3[].rank, .concern, .weight, .explanation | 推理+排序 |
| 决策评分 | 客户决策力评估 | decisionScore.value, .label, .interpretation | 综合评估 |
| 话术建议（扩展模块） | 推荐说什么+怎么说 | 底部可折叠区域，非强制输出 | D2决策：V2再做第四层 |

**金句**：**"3秒不是AI快，是你平时30分钟做的事，现在有人帮你3秒出初稿。你的专业判断才是终稿。"**

【v1.1调整】**实操环节**（15min）——输入改6字段表单、输出对齐三层Schema：

- 场景A：demo数据预填6字段 → 解码器一键出三层报告 → 学员验证报告准不准
- 场景B：拿到三层报告 → 制定服务策略（推荐哪套+先说什么话+预判什么顾虑）
- 场景C（租赁线）：租客"预算2000地铁附近"预填6字段 → 报告告诉你真实需求
- **AI嵌入**：解码器出报告 = 客户分析过程被加速；你判断准不准 = 专业能力在升级

【v1.1调整】**课后作业**（适配手动导出）：
```
📝 第2课课后作业
任务：用解码器做一个真实客户的解码报告，写下你的服务策略
交付：
1. 6字段表单填写内容（截图或复制）
2. 解码器三层输出（点"保存到成长档案"→复制摘要→粘贴到成长档案-学习记录）
3. 基于报告的服务策略：推荐什么+先说什么+预判什么顾虑
→ 粘贴到成长档案-学习记录
```

### 3.4 第3课：从解码到成交——让每个客户都感觉"你真懂我"

| 项目 | 内容 |
|------|------|
| **学习目标** | 能把解码报告转化成客户愿意买单的服务方案，实现读客户→服务客户闭环 |
| **想明白时刻** | "解码报告不是给自己看的，是帮客户'被读懂'。客户说'你真懂我'的那一刻，成交就是时间问题。" |
| **核心模型** | **四步闭环 + 信任飞轮** |
| **验收标准** | L1能说出四步闭环；L2能完成从三层解码报告到服务方案的完整输出 |

**从解码到成交四步闭环**：
```
三层解码报告（知道客户要什么）
    ↓
匹配方案（推荐+为什么推荐这套）
    ↓
预判顾虑（客户还没问你就先回答了）
    ↓
促成决策（不是催签，是帮客户下决心）
```

**"你真懂我"的三个对比**：

| 场景 | 普通经纪人 | 懂客户的经纪人 |
|------|-----------|--------------|
| "再看看" | "好的您再考虑" | "您是不是对XX还有顾虑？我帮您确认一下" |
| 嫌贵 | "这个价真不贵" | "理解，咱们看看同小区成交价对比，您心里就有数了" |
| 被截胡 | "那套确实不错" | "您要是喜欢那套，我帮您查查有没有抵押查封风险" |

**信任飞轮**：
```
读对客户 → 说对话 → 解决真问题 → 客户信任你
    ↑                                    ↓
    ← 更多转介绍 ← 更多成交 ← 客户觉得你靠谱
```

**金句**：**"成交不是说服，是帮客户做他本就想做的决定。解码报告告诉你他在犹豫什么，你帮他跨过去。"**

【v1.1调整】**实操环节**（15min）——基于三层报告+话术扩展模块：

- 场景A：基于第2课的三层解码报告 → 写一段服务话术（理解客户后的自然表达，不是模板背诵）
- 场景B：模拟客户3个常见顾虑 → 用三层报告数据逐一回应
- 场景C（租赁线）：续租谈判场景 → 用数据帮房东算账 → 从"降不降"变成"降vs空置哪个亏"
- **AI嵌入**：三层解码报告→话术扩展模块 = AI帮你把"读到的"变成"说出来的"，不是替你说，是帮你组织好

【v1.1调整】**课后作业**（适配手动导出）：

### 8.3 P0场景示例（场景01）

```
场景01：首次置业·刚需上车 [P0] [交易线]
━━━━━━━━━━━━━━━━━━━━━━━━
📋 基本信息
  小区/商圈：朝阳区，望京/花家地一带
  预算范围：150-300万
  需求类型：首次置业
  核心诉求：预算控制(高权重) > 通勤便利(高权重) > 居住品质(中权重)
  决策人：夫妻共同
  时间紧迫度：3个月内

👤 典型画像
  90后小夫妻，首套房，对流程完全不熟
  关注点分散（什么都想看），容易被打折吸引

🔍 解码要点
  第一层·显性需求：
    • 预算150-300万两居室（source=预算范围）
    • 望京一带通勤便利（source=小区/商圈+核心诉求）
    • 3个月内要定房（source=时间紧迫度）

  第二层·隐性需求：
    • 预算上限实际可到280万看性价比（signal="150-300万"上限留余地，confidence=0.8）
    • 首次置业信息焦虑，需要"被引导"而非"被推销"（signal=首次置业+夫妻共同，confidence=0.85）
    • 妻子更看重品质，丈夫更看重总价（signal=夫妻共同决策+预算控制，confidence=0.7）

  第三层·需求优先级Top3：
    rank 1: 预算控制（weight=35%, explanation="首次置业预算敏感度最高，超预算=直接放弃"）
    rank 2: 通勤便利（weight=30%, explanation="刚需群体通勤是每日体验，不可妥协"）
    rank 3: 居住品质（weight=20%, explanation="有改善空间但非硬约束，可妥协面积换位置"）

  决策评分：
    value: 58
    label: "初级决策力"
    interpretation: "首次置业+夫妻共同=决策易受外部影响，需要专业引导减少犹豫"

💬 推荐话术（扩展模块）
  开场："首次买房最重要的是不踩坑，我先帮您理清哪些必须满足、哪些可以灵活，咱们效率最高地看。"
  顾虑回应："300万在望京两居选择确实不多，但往花家地方向性价比更高，我先带您看两套对比一下。"
  促成："这套虽然面积比您预期小5平，但省下的钱正好把装修解决了，而且学区直接锁定。"
```

### 9.2 Schema与微课三层模型对齐

| 微课教学 | Schema字段 | 学员应理解 |

| 第一层·显性需求 | layer1_explicit.items[].content + .source | 客户说出来的需求，对应哪个输入字段 |
| 第二层·隐性需求 | layer2_implicit.items[].content + .signal + .confidence | 客户没说但重要的，推理信号是什么，有多确定 |
