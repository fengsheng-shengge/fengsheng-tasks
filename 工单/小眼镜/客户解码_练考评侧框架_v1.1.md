## 二、练：解码器实操

### 2.1 设计逻辑

**金句**：**"练不是练工具，是练你读客户的眼力。解码器是你的放大镜，不是你的望远镜——它放大你已经看到的，不是替你看。"**

| 项目 | 说明 |
|------|------|
| 练什么 | 输入客户信息 → 出解码报告 → 基于报告出服务策略 |
| 不是什么 | 不是练AI操作、不是练Prompt写法、不是练软件界面 |
| 本质 | 用真实客户场景练"客户洞察"能力，解码器是加速器不是替代品 |
| 与学的关系 | 学=学方法论（四层模型/解码报告/四步闭环），练=用方法论+工具做出来 |
| 与考的关系 | 练=日常训练，考=达标验收，练不等于考但必须通过练才能过考 |
| 与产品的关系 | 练的每一步就是解码器产品功能的使用流程，课程就是产品使用说明书 |

**难度递进逻辑**：【v1.1调整】难度梯度与X2场景优先级映射

| 梯度 | 定义 | 信息完整度 | 客户复杂度 | 服务策略要求 | 对应X2优先级 |
|------|------|-----------|-----------|------------|-------------|
| **L0→L1入门练** | 能走通流程 | 高（6字段结构化信息已给出） | 低（单一需求，决策链短） | 能按报告执行 | P0高频场景 |
| **L1→L2进阶练** | 能判断取舍 | 中（需自己提取关键信息填入6字段） | 中（多重需求，有冲突） | 能做优先级排序 | P1中频场景 |
| **L2巩固练** | 能应变创新 | 低（模糊信息/多角色博弈） | 高（多方博弈/隐性需求深） | 能预判+创新方案 | P2-P3复杂场景 |

### 3.1 设计逻辑

**金句**：**"考不是问你答对了没有，是问你解码到了第几层。答对是记忆，解码到才是能力。"**

| 项目 | 说明 |
|------|------|
| 考什么 | 情景判断题（认知层）+ 实操评分（行为层），双轨制 |
| 不是什么 | 不是知识背诵、不是AI操作考试、不是标准答案匹配 |
| 核心逻辑 | 测入门槛→考出水平；考是测的升级版，维度一致但深度不同 |
| 与测的关系 | 测=5题3分钟快筛（入学的温度计），考=20题+实操（升级的体温计） |
| 与评的关系 | 考产出档位结果，评记录档位变化曲线 |

**测→考的升级关系**：

| 对比项 | 测（入学诊断） | 考（阶段测评） |
|--------|-------------|-------------|
| 目的 | 测起点 | 验进阶 |
| 时长 | 3分钟 | 20分钟 |
| 题量 | 5题快筛 | 20题情景判断+1次实操 |
| 深度 | 快速定位档位 | 精确评估各维度+验证实操能力 |
| 输出 | 档位+薄弱维度 | 档位+四维度评分+实操等级+升级建议 |
| 频次 | 入学1次 | 每完成1阶段可考1次 |

### 3.2 双轨制考卷设计

#### 轨道1：情景判断题（认知层）

**题型**：给定业务场景，选择最佳应对策略（不是选"对错"，是选"解码深度"）

**评分逻辑**：【v1.1调整】明确规则评分映射

| 选项等级 | 解码深度 | 对应档位 | 得分 |
|---------|---------|---------|------|
| A级 | 解码到第3层（需求优先级），预判+创新 | L2 | 3分 |
| B级 | 解码到第2层（隐性需求），有依据 | L1 | 2分 |
| C级 | 只解码到第1层（显性需求），凭感觉 | L0 | 1分 |
| D级 | 误解读/隐瞒风险 | L0强制 | 0分 |

**规则评分实现**（前端JS）：
```javascript
const scoreMap = { 'A': 3, 'B': 2, 'C': 1, 'D': 0 };
```

**双线分化**：

| 维度 | 交易线题量 | 租赁线题量 | 说明 |
|------|-----------|-----------|------|
| D1客户识别 | 5题 | 5题 | 场景不同，维度一致 |
| D2需求挖掘 | 5题 | 5题 | 交易线侧重购房决策链，租赁线侧重租住匹配 |
| D3信任建立 | 5题 | 5题 | 交易线侧重风险预警建信，租赁线侧重费用透明建信 |
| D4场景应对 | 5题 | 5题 | 交易线侧重犹豫/比价/截胡，租赁线侧重争议/拖欠/续租 |
| **合计** | **20题** | **20题** | 通用题8道+分化题12道 |

**示例（交易线D2需求挖掘，对齐X2场景01首次置业）**：

> 年轻夫妻首次看房，说"200万左右，离公司近就行"。你判断最该先确认的是？
>
> - A. 两人分别最看重什么——"如果只能满足一个，你选通勤还是品质？"（解码到第3层·需求优先级）→ **3分**
> - B. "200万左右"的弹性空间——"最高能接受多少？"（解码到第2层·隐性需求）→ **2分**
> - C. 帮他们搜望京200万房源（只到第1层·显性需求）→ **1分**
> - D. "200万能买到不错的"——不追问直接推盘 → 忽略信号 → **0分**

**零容忍项**（与测一致，考中强化）：

| 零容忍行为 | 交易线 | 租赁线 |
|-----------|--------|--------|
| 隐瞒风险 | 发现房产风险不告知客户（法拍/抵押/查封） | 发现合同陷阱不告知租客 |
| 误导客户 | 虚假承诺学区/升值 | 虚假承诺免押/随时退 |
| 选边站 | 只帮业主/只帮买家 | 只帮房东/只帮租客 |

**触发零容忍 → 该维度直接判0分，不参与档位计算，需补考。**

#### 轨道2：实操评分（行为层）

【v1.1调整】评分方式改为规则评分+人工抽检10%

| 项目 | 说明 |
|------|------|
| 考法 | 给定1个综合场景 → 用解码器完成完整流程 → 提交三层解码输出+自评 |
| 时长 | 15分钟 |
| 评分 | 规则评分（对照标准答案，同练模块评分函数）+人工抽检10% |
| 与练的区别 | 练=可重做无时限，考=限时限次（30天冷却期） |

**实操评分表**（对齐X3三层Schema）：

| 维度 | L0本能型（1分） | L1标准型（2分） | L2专业型（3分） |
|------|---------------|---------------|---------------|
| **D1客户识别** | 只看到表单字段表面含义，画像模糊 | 能提取6字段关键信息，画像基本准确 | 画像精准，识别决策链和隐藏角色 |
| **D2需求挖掘** | 只列出客户填写的需求 | 能区分显性需求和隐性需求 | 识别隐性需求+信号+置信度，排出优先级 |
| **D3信任建立** | 话术生硬或推销感强 | 话术有依据但不够自然 | 话术基于报告，预判顾虑，客户视角 |
| **D4场景应对** | 被动等客户提问 | 能回应常见顾虑 | 预判风险主动应对，有备选方案 |

**人工抽检10%方案**：【v1.1调整】具体化

| 步骤 | 操作 | 实现 |
|------|------|------|
| 1 | 系统随机标记10%考试记录 | 前端Math.random()，标记到localStorage `needsManualReview: true` |
| 2 | 被标记的记录导出 | 管理员手动触发导出按钮→生成JSON/CSV文件下载 |
| 3 | 管理员线下复核 | 比对学员答案vs标准答案vs自动评分，校准偏差 |
| 4 | 反馈结果 | 人工确认/修正评分，更新localStorage对应记录 |

### 3.3 档位升级机制

【v1.1调整】档位存储改为localStorage，30天冷却期用localStorage记录

**升级条件**：

| 升级路径 | 情景判断题要求 | 实操评分要求 | 附加条件 |
|---------|-------------|-----------|---------|
| L0 → L1 | 总分≥40分（满分60）+ 无零容忍项 | 4维度均≥L1 | 完成微课1-2节+至少2次练习 |
| L1 → L2 | 总分≥50分 + 无零容忍项 + D2D4均≥A级 | 4维度均≥L2 | 完成微课3节+至少5次练习+1次真实客户应用 |
| L2 → L3 | MVP暂不开放 | — | 待v2.0定义 |

**降级保护**：考不过不降级，保持原档位，可补考。

**30天冷却期实现**：

```javascript
function canTakeExam() {
  const lastExam = localStorage.getItem('lastExamDate');
  if (!lastExam) return true;
  const daysSince = (Date.now() - new Date(lastExam).getTime()) / (1000*60*60*24);
  return daysSince >= 30;
}
```

**升级流程**：

```

    → 前端检查localStorage前置条件（课+练+冷却期）
        → 不满足 → 提示补什么/剩余冷却天数
        → 满足 → 进入考试
            → 轨道1：情景判断20题（5分钟）
            → 轨道2：实操1个综合场景（15分钟）

                    → 未达标 → 保持原档位 → 30天后可补考
```

```

        ↓

    → 10个场景梯度练习（L0→L1→L2），对齐X2
    → 输入：6字段结构化表单（X1）
    → 输出：三层结构+决策评分+话术扩展（X3）

        ↓

    → 轨道2：实操1个综合场景（规则评分）
    → 双轨评分→档位更新→localStorage
    → 30天冷却期
        ↓
成长档案（评）
    → 三模块：雷达图+历史列表+学习记录自填
    → 数据流：手动触发导出（"保存到成长档案"→摘要→复制粘贴）

        ↓

```
