## 一、为什么需要课程页面

课程体系是110人验证线的**流量入口**。每门课程至少引导5人到 fengsheng.tech 完成核心操作。

```
课程学习 → 课中引导"用风声工具试试" → 学员到fengsheng.tech
    → 完成核心操作 → 反馈落库 → 验证计数+1
```

没有课程页面，这个链路断在第一步。

## 二、最小可用功能（MVP）

### 2.1 课程列表页

| 功能 | 说明 |
|------|------|
| 课程目录 | 展示通识课（A1-A4）+ 专业课（B1-B4）的模块列表 |
| 课程状态标记 | 每门课标注：已上线 / 即将上线 / 敬请期待 |
| 进入入口 | 点击课程 → 进入课程详情页 |

### 2.2 课程详情页

| 功能 | 说明 |
|------|------|
| 课程大纲 | 展示每周主题 + 核心能力 + 交付物 |
| 学习路径 | 推荐学习顺序（A4→A1→B模块→C模块） |
| 产品衔接入口 | 每课对应的fengsheng.tech产品操作入口（关键！） |
| 验收标准 | 4级验收标准展示 |

### 2.3 产品衔接（核心）

每个课程页面必须包含：
1. **产品入口引导**（1句话）：说明这个能力用XX工具可以快速上手
2. **实操任务按钮**：点击跳转到对应产品页面（/breeder、/decoder、/report、/assessment、/quality）
3. **反馈收集**：操作完成后提交反馈，进入D1数据库

## 三、技术要求

### 3.1 页面结构

```
fengsheng.tech/courses          ← 课程列表页
fengsheng.tech/courses/a1       ← A1课程详情
fengsheng.tech/courses/a1/w1   ← 第1课详情（含产品衔接入口）
fengsheng.tech/courses/a4       ← A4课程详情
fengsheng.tech/courses/b4       ← B4课程详情
```

### 3.2 数据需求

| 数据 | 来源 | 用途 |
|------|------|------|
| 验证计数 | 课程页面产品入口点击 → 产品页面操作完成 → 计数+1 | 跟踪每门课引流到产品的人数 |
| 反馈数据 | 课程实操后提交的反馈 | 进入D1数据库，驱动课程迭代 |
| 学习进度 | 用户完成每课交付物后标记 | 后续做学习路径追踪（v2再做） |

### 3.3 与现有产品的衔接

- 课程页面的"产品入口"按钮直接跳转现有5产品页面
- 跳转时带上课程标识参数（如 `?from=course_a1_w1`），便于追踪来源
- 产品页面完成操作后，反馈数据回传时带上课程标识
