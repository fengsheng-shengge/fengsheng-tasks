# 第1课 · 客户解码，你缺的不是AI，是眼力

> **阶段**：切入期 | **时长**：30分钟 | **对应场景**：场景01"首次置业·刚需上车"
> **学习目标**：能从客户的信息中判断出他真正要什么
> **验收标准**：L1能说出四维模型D1-D4分别是什么；L2能对客户信息做四维拆解

## 开场金句

你有没有这种感觉——客户说了一堆，你听了一堆，推荐了十套房，客户一句"再看看"就走了？

然后你怪AI不好用，怪解码器不准，怪客户不靠谱。

但你想过没有：**垃圾进，垃圾出。**

你喂给AI的是"客户说想要两居室，望京附近"，出来的当然是"推荐望京两居室"——这叫信息搬运，不叫客户解码。

真正决定解码质量的，不是AI有多聪明，是你往里输的信息有多准。你缺的不是AI，是眼力——读懂客户的眼力。

今天这节课，我帮你把"眼力"拆成四个维度，以后每个客户进来，你知道该看什么、该问什么、该往解码器里填什么。

---

## 核心概念：客户洞察四维模型 D1-D4

别被"四维模型"吓到，其实就是四个问题，每个客户你都得回答：

### D1 客户识别：这个客户到底是谁？

不是问名字，是问——他的真实身份是什么。

同样说"我想买房"，28岁互联网小夫妻和45岁为孩子上学的家长，是完全不同的客户。你如果只看到"想买房"三个字，后面全错。

**交易线重点看**：购房动机（刚需/改善/投资）+ 家庭阶段（新婚/有娃/养老）
**租赁线重点看**：租客画像（通勤/学区/性价比）+ 租期预判（短租过渡/长租安定）

**怎么练**：下次客户说"我想找个房子"，别急着推荐，先问一句"您是给自己找还是帮家人看的？"——这一句话就能分出至少三种客户类型。

### D2 需求挖掘：他没说的是不是比说了的更重要？

客户说的叫"显性需求"，客户没说但影响决策的叫"隐性需求"。

客户说"预算300万"，这是显性。但"300万是死线还是可以上浮10%看性价比"，这是隐性。你不知道这个，推荐就踩雷。

**交易线重点挖**：预算弹性 + 决策权（谁拍板）+ 隐性偏好（嘴上说通勤重要，实际最在意品质）
**租赁线重点挖**：合租/整租偏好 + 特殊需求（养宠/办公/短租）+ 续租意愿

**怎么练**：客户说一个条件，你追问一个"为什么"。他说"望京附近"，你问"是工作在那边还是喜欢那个片区？"——答案直接决定推荐逻辑。

### D3 信任建立：怎么让客户3分钟内觉得"这人靠谱"？

信任不是靠嘴甜，是靠**你比他先想到他没想到的事**。

客户嫌贵，你说"这个价真不贵"——你在否定他的感受，信任归零。
客户嫌贵，你说"理解，我帮您看看同小区最近成交价，您心里有数"——你在帮他做判断，信任+1。

**交易线重点**：风险主动提醒 + 不推销先诊断 + 价格透明
**租赁线重点**：费用说明透明 + 维修响应承诺 + 退租流程提前说

**怎么练**：下次带看前，先说一句"我先帮您理清哪些必须满足、哪些可以灵活，咱们效率最高地看"——这句话就是在建立"我不是推销员，我是顾问"的信任。

### D4 场景应对：关键时刻你扛不扛得住？

客户犹豫不决、被截胡、押金争议、房东涨租——这些关键时刻，你能不能接住，直接决定成交还是流失。

**交易线重点**：比价犹豫 + 家属意见不一 + 房屋风险
**租赁线重点**：押金争议 + 拖欠租金 + 提前退租 + 涨租

**怎么练**：想一个你最怕遇到的客户场景，写下来你会怎么说——如果你写不出来，那个场景就是你的D4短板。

## 场景演示：场景01"首次置业·刚需上车"

现在我们拿一个真实场景走一遍，看看四维模型怎么用。

### 场景背景

小王和小李，90后小夫妻，首套房。小王在望京上班，小李在国贸。两人看了两周房，越看越迷糊——什么户型都有，什么价格都看，根本不知道怎么选。

你接到他们的需求，怎么用四维模型解码？

### 第一步：填6字段——把客户信息结构化

别凭脑子记，用解码器，先填6个字段：

| 字段 | 填什么 | 为什么这么填 |
|------|--------|------------|
| **小区/商圈** | 朝阳区望京/花家地一带 | 小王上班在望京，通勤是核心约束 |
| **预算范围** | 150-300万 | 首次置业，预算有限但非刚性 |
| **需求类型** | 首次置业 | 没买过房，信息焦虑 |
| **核心诉求** | 预算控制(高) > 通勤便利(高) > 居住品质(中) | 预算卡着，通勤不能妥协，品质能忍 |
| **决策人** | 夫妻共同 | 两人都得住，意见得统一 |
| **时间紧迫度** | 3个月内 | 不是急到明天，但也不想拖半年 |

这6个字段填完，你其实已经做了D1客户识别——你知道这是一对首套刚需、预算有限、通勤驱动、双人决策的年轻夫妻。

### 第二步：看三层输出——解码器帮你放大眼力

6字段填完，解码器3秒出三层报告。我们来看：

**━━━ 第一层·显性需求 ━━━**

客户说出来的，6字段直接映射：

- 预算150-300万两居室（来源：预算范围）
- 望京一带通勤便利（来源：小区/商圈+核心诉求）
- 3个月内要定房（来源：时间紧迫度）
- 首次购房无经验（来源：需求类型）
- 夫妻共同决策（来源：决策人）

这一层不需要AI，你自己也能列。但问题是——只看到这些，你推荐什么？推荐300万以内的望京两居室？所有人都在推荐这个，你凭什么赢？

**━━━ 第二层·隐性需求 ━━━**

客户没说但重要的，解码器帮你推理：

- **预算有弹性**：300万是心理上限，但实际可到280万看性价比（信号：预算选了150-300万"中段"+首次购房不熟悉市场，置信度0.85）
- **信息焦虑，需要"被引导"**：首套房+夫妻共同决策=信息量大到不知道怎么筛选，不是不想买，是不知道怎么选（信号：首次置业+夫妻共同+看了两周越看越迷糊，置信度0.90）
- **妻子更重品质，丈夫更重总价**：夫妻共同决策+预算控制+通勤便利=两人关注点不同（信号：核心诉求排序暗示两人偏好差异，置信度0.75）

这三条隐性需求，才是你能赢的关键。别人推荐房子，你帮客户做选择——区别就在这里。

**━━━ 第三层·需求优先级Top3 ━━━**

哪个最关键，解码器帮你排：

| 排名 | 关注点 | 权重 | 理由 |
|------|--------|------|------|
| P0 | 预算控制 | 35% | 首次置业预算敏感度最高，超预算=直接放弃 |
| P1 | 通勤便利 | 30% | 刚需群体通勤是每日体验，不可妥协 |
| P2 | 居住品质 | 20% | 有改善空间但非硬约束，可妥协面积换位置 |

**━━━ 决策评分 ━━━**

决策力58分·初级——首次置业+夫妻共同=决策易受外部影响，需要专业引导减少犹豫。

### 第三步：四维对照——你看到了什么？

| 维度 | 你读到了什么 |
|------|------------|
| **D1客户识别** | 首套刚需小夫妻，通勤驱动型，预算有限但有弹性 |
| **D2需求挖掘** | 预算弹性+信息焦虑+夫妻关注点不同=三个隐性需求别人看不到 |
| **D3信任建立** | 他们需要的不是更多房源，是"帮我做选择"——你先帮他们理清优先级 |
| **D4场景应对** | 3个月时限+预算有限=最怕冲动买错或犹豫错过 |

四维都看清了，你的推荐才有准星。

## 四维模型和三层输出的关系

别搞混了——四维模型是你的**眼力框架**（看什么），三层输出是解码器的**放大镜**（帮你看得更清）。

```
D1客户识别 → 决定你往6字段里填什么（填错=全错）
D2需求挖掘 → 决定你能不能看懂第二层隐性需求
D3信任建立 → 决定你怎么跟客户沟通解码结果
D4场景应对 → 决定关键时刻你能不能用解码结果帮客户做决策
```

**垃圾进垃圾出**——D1做不好，6字段填不准，解码器再智能也白搭。所以我说，你缺的不是AI，是眼力。

## 从客户对话中提取6字段

很多人问：客户不会一个字段一个字段告诉我啊，我怎么填6字段？

这就是关键能力——从客户零散的对话中，提取结构化信息。

举个例子，客户说：

> "我和女朋友打算结婚，在望京上班，想买个两居室，别太贵，最好300万以内能搞定。我俩都觉得离公司近比较重要。估计3个月内得定吧，不然来不及。"

你提取出来就是：

| 字段 | 提取内容 | 提取依据 |
|------|---------|---------|
| 小区/商圈 | 望京 | "在望京上班"+"离公司近" |
| 预算范围 | 150-300万 | "300万以内" |
| 需求类型 | 首次置业 | "打算结婚"+"别太贵"=首套 |
| 核心诉求 | 通勤便利>预算控制 | "离公司近比较重要"+"别太贵" |
| 决策人 | 夫妻共同 | "我俩都觉得" |
| 时间紧迫度 | 3个月内 | "3个月内得定" |

看到了吗？一段60个字的对话，6字段全提取出来了。这就是D1客户识别的落地——不是背模型，是练提取。
