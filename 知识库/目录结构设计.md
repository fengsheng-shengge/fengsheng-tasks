## 命名规范

| 类别 | 格式 | 示例 |
|------|------|------|
| 词条 | `{业务线}-{阶段}-{层级}-{ID}` | `购房线-首套-物质-BUY-001` |
| 产品文档 | `{产品}-{类型}-{版本}` | `解码器-使用指南-v1.0` |
| SOP | `{流程名}-SOP` | `知识萃取-SOP` |
| 复盘 | `{类型}复盘-{日期}` | `日复盘-20260627` |
| 决策 | `决策-{主题}-{日期}` | `决策-定价方案-20260626` |

## 双链规则

- 每个词条 → 链接关联词条 + 链接所属SOP + 链接验证记录
- 每个产品文档 → 链接需求文档 + 链接使用指南 + 链接FAQ
- 每个SOP → 链接使用场景 + 链接案例
- 每个复盘 → 链接相关项目 + 链接决策记录
- 每个决策 → 链接前置讨论 + 链接后续影响

## 日常闭环

```
捕捉(碎片→Inbox)
  → 整理(AI摘要/打标签/命名)
    → 入库(按模板写入对应目录)
      → 回顾复用(旧笔记串起来)
```

## 进化机制

1. 越用越懂你：AI记住风声的命名习惯和写作风格
2. 自动补全结构：摘要、标签、分类越来越顺手
3. 旧笔记被激活：新内容把旧内容重新连起来
4. 知识图谱长出来：不是一堆文档，而是一张会生长的网
